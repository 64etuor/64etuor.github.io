Core Rules

Frontmatter Standard

---
title: ""
description: ""
created: YYYY-MM-DD
modified: YYYY-MM-DD
type: fleeting | literature | permanent | reference | project | log | output
status: inbox | draft | active | review | done | archive
author: ""
tags: []
---

description: optional, 한 줄(~100자) 요약. AI 인덱싱·검색·Dataview 목록 표시용. 비워 둬도 무방.

Tag System (summary)

3 dimensions: domain(1-3) + source(1) + action(0-1), 3-8 tags per note

  • Domain: dev/, data/, ai/, infra/, biz/, career/, lang/, write/, finance/, life/
  • Source: src/book, src/article, src/lecture, src/video, src/course, src/paper, src/doc, src/experience
  • Action (temporary): action/todo, action/review, action/practice, action/followup

Wikilinks & Backlinks

  • Internal vault references always use [[wikilink]]. External URLs only use [text](url).
  • 연관 노트 연결은 반드시 본문 wikilink로 한다 (YAML related: 속성 사용 금지)
    • 인라인: 본문 문맥 속에서 자연스럽게 [[노트명]] 언급
    • ## Related 섹션: 본문 하단에 리스트로 정리
    • 두 방식 혼용 가능
  • wikilink를 백틱으로 감싸지 말 것 — `[[MBO]]`처럼 백틱 안에 넣으면 Obsidian이 인라인 코드로 해석해 링크·백링크·그래프가 전부 깨진다. 본문 어디서든 [[MBO]] 그대로 쓴다. 코드 폰트 강조가 필요하면 wikilink 밖에서만 쓴다 (예: `description` 필드는 [[tag-system]]에서 정의된다).

File Name & Wikilink Safety

파일명과 wikilink 텍스트에는 다음 문자를 넣지 않는다. Obsidian 파서와 OS 파일 시스템에서 링크가 깨지거나 백링크·검색 인덱스가 빠진다.

문자 문제 대체 방법
' 아포스트로피 위키링크 파싱 실패 (예: Porter's → 링크 깨짐) 제거하거나 aliases:로 이동
: 콜론 Windows 파일명 금지, YAML 구문 충돌 - 또는 로 대체
/ \ 경로 구분자로 해석 제거
` ` 파이프 wikilink alias 구분자와 충돌
[ ] 중첩 wikilink로 해석 제거
# 해시 block/heading 참조로 해석 제거
^ 캐럿 block ID 참조로 해석 제거
? * < > " Windows 파일명 금지 제거

원칙: 파일명은 최대한 단순하게(영문·한글·숫자·공백·-·_·.), 디스플레이에 필요한 특수표현은 title: 또는 aliases:에 넣는다.

예: Porter's Five Forces.md ❌ → Five Forces.md + aliases: [Porter Five Forces, 포터의 5가지 경쟁요인] ✅

Exploration & Loading Rules

Before starting any vault task, follow this procedure:

  1. Read this file (CLAUDE.md) — already loaded.
  2. Analyze the user's request and identify the task type.
  3. Scan the routing table below. Match the task to the most relevant file(s) using the Description column.
  4. Load only the matched file(s). Maximum 3 on-demand files per task.
  5. If no file matches, proceed without and inform the user.

Principles:

  • Fewer precise files > many loosely related files. When in doubt, don't load.
  • Check the routing table before every task — the system files contain use_when fields in their frontmatter for precise matching.
  • After loading, follow the instructions in the loaded file (Maps for classification, Programs for step-by-step workflows).
