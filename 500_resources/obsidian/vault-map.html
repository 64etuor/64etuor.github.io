Vault Map

Folder Structure

This vault uses a modified PARA structure. Each folder has a single role.

Folder Role type value Placement criteria
100_Inbox/ Inbox fleeting Anything not yet classified
200_Books/ Book notes literature Notes based on books
300_Projects/ Project docs project Work with deadlines and deliverables
400_Areas/ Ongoing areas permanent Areas maintained without deadlines
500_Resources/ References reference External materials, system docs, tool configs
600_Tech Stack/ Tech knowledge permanent Learning and reference notes by tech stack
700_Studies/ Study notes literature Lectures, courses, bootcamp notes
900_Writings/ Creative works & journals output, log Essays, fiction, poetry, diaries, retrospectives
Clippings/ Web scraps reference Browser clippings (unprocessed)
Templates/ Templates - Templater only
Settings/ Settings - Obsidian app settings

Placement Decision Tree

New note -> "Has deadline and deliverables?"
            |-- Yes -> 300_Projects/
            +-- No -> "Summarizing an external source?"
                       |-- Yes -> "From a book?"
                       |           |-- Yes -> 200_Books/
                       |           +-- No -> "From a lecture/course?"
                       |                      |-- Yes -> 700_Studies/
                       |                      +-- No -> 500_Resources/ or Clippings/
                       +-- No -> "Self-authored tech notes?"
                                  |-- Yes -> 600_Tech Stack/
                                  +-- No -> "Creative writing?"
                                             |-- Yes -> 900_Writings/
                                             +-- No -> "Journal or log?"
                                                        |-- Yes -> 900_Writings/
                                                        +-- No -> 400_Areas/ or 100_Inbox/

Inbox Processing (100_Inbox/, Clippings/)

Both 100_Inbox/ and Clippings/ are inboxes. Self-written memos go to 100_Inbox/, browser clippings go to Clippings/. Processing logic is the same.

Inbox item -> "Worth processing?"
  |-- No -> Keep as-is (or delete)
  +-- Yes -> "What level of processing?"
              |-- Preserve original (highlight only) -> 500_Resources/{topic}/
              |-- Rewrite in own words             -> 600_Tech Stack/{topic}/ or 400_Areas/{topic}/
              +-- Develop into article/creative    -> Run the appropriate Program
Processing level type Placement Example
Unprocessed fleeting / reference Keep in 100_Inbox/ or Clippings/ Quick memo, link for later
Original preserved + notes reference 500_Resources/{topic}/ API docs, official guides
Tech topic rewritten permanent 600_Tech Stack/{topic}/ Tech blog -> own summary
Non-tech topic rewritten literature 400_Areas/{topic}/ Self-help, finance articles
Book-based literature 200_Books/ Book summary, review
Developed into creative work output 900_Writings/{genre}/ Essay, poetry

Stale items in 100_Inbox/ and Clippings/ are reviewed periodically via the vault-maintenance program.

File Naming

Type Pattern Example
General note Descriptive title SQL JOIN Complete Guide.md
Date-based YYMMDD - Title 260331 - On Kindness.md
Tech article Descriptive English+Korean Decision Tree Architect - Claude Code Harness.md

Forbidden characters in filenames and wikilinks

Obsidian의 wikilink 파서와 OS 파일시스템이 깨지는 문자는 파일명·wikilink 텍스트에서 제외한다.

  • 절대 금지: ' (아포스트로피), : / \ | [ ] # ^ ? * < > "
  • 권장 문자만: 영문·한글·숫자·공백·-·_·.
  • 특수표현 우회: Porter's Five Forces 같은 디스플레이 이름이 필요하면 파일명은 Five Forces.md로 쓰고 aliases: [Porter Five Forces, 포터의 5가지 경쟁요인]에 넣는다. Obsidian 검색·자동완성이 alias를 인식한다.
