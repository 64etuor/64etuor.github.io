Obsidian Webpage Export 플러그인 가이드



OBSIDIAN

Webpage Export + GitHub Pages

나만의 디지털 가든을 무료로 웹에 배포하는 가이드

2026.03 · v1.1



이 가이드에 대해

이 문서는 Obsidian의 Webpage Export 플러그인으로 볼트(Vault)를 HTML로 변환한 뒤, GitHub Pages를 통해 무료로 웹 호스팅하는 전체 과정을 단계별로 안내합니다.

💡 이런 분을 위한 가이드입니다

개인 노트, 블로그, 포트폴리오 등을 무료로 웹에 공개하고 싶은 분. Git이나 터미널 사용이 처음이어도 괜찮습니다.

필요한 것들

  • Obsidian (설치됨)
  • GitHub 계정 (github.com)
  • Git (이 가이드에서 설치 안내)
  • 인터넷 연결

STEP 1 · Git 설치하기

Git은 파일의 변경 이력을 관리하고, GitHub에 파일을 업로드하는 데 필요한 도구입니다.

Windows

  1. https://git-scm.com 접속
  2. "Download for Windows" 클릭 후 설치 파일 실행
  3. 설치 옵션은 모두 기본값 그대로 → Next → Install
  4. 설치 완료 후 터미널(CMD 또는 PowerShell)에서 확인:
git --version

macOS

Homebrew 설치

  1. 터미널을 열고 아래 명령어 입력:
### Homebrew
Install [homebrew](https://brew.sh/) if you don't already have it, then:  
`$ brew install git`

혹은

Xcode Command Line Tools

  1. 터미널을 열고 아래 명령어 입력:
### Xcode Command Line Tools
xcode-select --install
  1. 팝업이 나타나면 "설치" 클릭
  2. 설치 완료 후 확인:
git --version

Git 초기 설정

설치 후 터미널에서 본인 정보를 등록합니다:

git config --global user.name "본인이름"
git config --global user.email "본인@email.com"

💡 팁 — GitHub 계정에 등록한 이메일과 동일한 이메일을 사용하세요.

STEP 2 · Obsidian Webpage Export 플러그인 설치

방법 A, B, C 중 하나를 선택해서 진행

방법 A: 커뮤니티 플러그인으로 설치 (권장)

  1. Obsidian 설정(⚙️) → 커뮤니티 플러그인
  2. "제한된 모드 해제" 토글 켜기
  3. "Webpage HTML Export" 검색 → 설치 → 활성화

방법 B: BRAT으로 베타 버전 설치

BRAT은 GitHub 저장소 주소를 입력해 베타 버전 플러그인을 설치할 수 있는 플러그인입니다. 정식 배포 전 최신 기능을 미리 사용하고 싶을 때 유용합니다.

  1. 커뮤니티 플러그인에서 "BRAT" 검색 → 설치 → 활성화
  2. Ctrl(cmd)+P 혹은 플러그인 설정 패널에서 → "BRAT: Add a beta plugin" 실행
  3. 저장소 주소 입력: 
    https://github.com/KosmosisDire/obsidian-webpage-export)
  4. 설치 완료 후 설정 → 플러그인에서 활성화

방법 C: 수동 설치

커뮤니티 플러그인이나 BRAT을 사용할 수 없는 경우의 수동 설치 방법입니다.

  1. Latest Release 에서 .zip 파일 다운로드
  2. 볼트 디렉토리 하위에 압축 풀기: {VaultFolder}/.obsidian/plugins/
  3. 옵시디언 재시동

STEP 3 · GitHub 저장소 만들고 클론하기

HTML을 내보내기 전에, 먼저 내보낼 대상 폴더(저장소)를 만들어야 합니다.

3-1. GitHub에서 저장소 생성

  1. github.com 로그인
  2. 오른쪽 상단 + 버튼 → New repository
  3. 설정:

    ⭐ 권장: 아이디.github.io 저장소명 사용

    저장소 이름을 본인아이디.github.io로 설정하면 별도 Pages 활성화 없이 자동으로 GitHub Pages가 배포됩니다. 주소도 https://본인아이디.github.io로 깔끔합니다.

  • Repository name: 본인아이디.github.io (권장) 또는 원하는 이름
  • Public 선택 (무료 GitHub Pages는 Public만 가능)
  1. Create repository 클릭

3-2. 로컬에 클론(clone)

  1. 내 레포지토리 주소를 복사합니다.
  2. 터미널을 열고, 저장소를 내 컴퓨터에 복제합니다:
cd ~/{원하는 폴더 주소}
git clone https://github.com/{본인아이디}/{본인아이디}.github.io.git

💡 팁 — Windows라면, 원하는 폴더로 이동 후 주소표시줄에 cmd 입력하면 바로 해당 폴더에서 터미널을 진입할 수 있습니다.

다른 이름으로 저장소를 만들었다면:

git clone https://github.com/{본인아이디}/my-digital-garden(만든 레포지토리 이름).git

💡 팁 — 클론한 폴더 경로를 기억해두세요. 다음 단계에서 내보내기 대상 폴더로 설정합니다.

STEP 4 · HTML로 내보내기

플러그인으로 볼트를 HTML로 변환하고, 바로 클론한 저장소 폴더에 저장합니다.

내보내기 실행

  1. Export as HTML 플러그인 선택 혹은 커맨트 팔레트Ctrl(cmd)+P에서 Webpage HTML Export: Set html export settings 선택

    OR
  2. Obsidian 왼쪽 파일 트리에서 내보낼 파일 또는 폴더를 클릭하여 체크 한 후 Save 선택
  3. 내보내기 폴더 선택: 내보내기(Export) 폴더를 Github을 클론한 저장소 폴더로 지정
  4. Export 선택

⚠️ 중요

내보내기 폴더를 반드시 클론한 GitHub 저장소 폴더로 지정하세요. 별도 폴더에 내보낸 후 복사하는 것보다, 처음부터 저장소 폴더에 직접 내보내는 것이 훨씬 편리합니다.

# 예시: 클론 폴더 경로
# Windows: C:\Users\본인이름\Documents\본인아이디.github.io
# Mac: /Users/본인이름/Documents/본인아이디.github.io

💡 팁 — 볼트 루트에 index.md를 생성하고 내보내면 index.html으로 자동 변환됩니다. GitHub Pages의 기본 페이지로 사용되며 편집 가능합니다.

STEP 5 · GitHub에 업로드 (Push)

내보내기가 완료되면, 터미널에서 아래 명령어를 순서대로 실행합니다:

cd ~/{내보내기 폴더}
git add .
git commit -m "Add obsidian html export"
git push

처음 실행시 GitHub 로그인 창이 나타날 수 있습니다. 브라우저로 로그인하면 됩니다.

STEP 6 · GitHub Pages 활성화

아이디.github.io로 저장소를 만든 경우

별도 설정이 필요 없습니다.

저장소 이름이 본인아이디.github.io인 경우, push하면 자동으로 GitHub Pages가 활성화됩니다.

접속 주소:

https://본인아이디.github.io

다른 이름으로 저장소를 만든 경우

저장소 이름이 본인아이디.github.io가 아니라면 수동으로 Pages를 활성화해야 합니다:

  1. GitHub에서 해당 저장소 페이지로 이동
  2. Settings 탭 클릭
  3. 왼쪽 메뉴에서 Pages 클릭
  4. Source 항목을 "Deploy from a branch"로 설정
  5. Branch: main / 폴더: / (root) 선택 → Save

1~2분 후 사이트가 배포됩니다.

접속 주소:

https://본인아이디.github.io/저장소이름/

💡 팁 — Actions 탭에서 배포 상태를 확인할 수 있습니다. 녹색 체크표시가 나타나면 성공!

업데이트 방법

노트를 수정한 후 웹사이트에 반영하는 방법입니다. 매번 이 3단계만 반복하면 됩니다:

1. 내보내기 — STEP4에서처럼 내보낼 파일이나 폴더를 변경하거나 혹은 그대로 Export

2. 커밋 & 푸시 — 터미널에서:

cd ~/{내보내기 폴더}
git add .
git commit -m "업데이트 내용 메모"
git push

3. 확인 — 배포가 되기 까지 충분히 기다린 후 사이트 접속하여 반영 확인

Commits에 가시면 배포 상황을 확인할 수 있습니다.

커스텀 도메인 연결 (선택)

본인만의 도메인을 사용하고 싶다면:

  1. 도메인 DNS 설정에서 CNAME 레코드 추가: 본인아이디.github.io
  2. 저장소 루트에 CNAME 파일 생성 (내용: 본인도메인.com)
  3. GitHub → Settings → Pages → Custom domain에 도메인 입력
  4. "Enforce HTTPS" 체크

문제 해결

증상 해결 방법
페이지가 404로 나옴 index.html이 저장소 루트에 있는지 확인. 하위 폴더 안에 있으면 안 됩니다.
CSS/이미지가 깨짐 플러그인 설정에서 Inline/Local Assets 켜져 있는지 확인
git push 오류 GitHub 로그인 확인. 브라우저 인증 팝업이 나오면 허용 클릭
Pages 설정이 안 보임 저장소가 Public인지 확인. Private면 무료 플랜에서 Pages 불가
배포 후 변경 반영 안 됨 브라우저 캐시 문제. Ctrl+Shift+R로 강제 새로고침
아이디.github.io가 안 됨 Repository 이름이 정확히 아이디.github.io인지 확인 (대소문자 주의)
