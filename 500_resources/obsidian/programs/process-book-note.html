Program: Process Book Note

Create or organize a book note from book information or reading notes.

Trigger

  • User requests a note for a book
  • User asks to organize post-reading notes
  • User requests a book review

Inputs

  • Book title or ISBN
  • User's memos, highlights, notable passages (if any)
  • Reading period (if any)

Steps

1. Gather

  • Confirm basic book info (author, publish date, page count)
  • Organize user-provided memos/highlights

2. Template

  • Detailed: Templates/Booknote.md
  • Simple: Templates/simple-reference-books.md
  • Default to Booknote unless user specifies otherwise

3. Metadata

Known issue — tag key: The Booknote plugin generates tag: (singular, inline) instead of the standard tags: (plural, YAML array). Always fix this after template generation: rename tag to tags and convert to array format.

Known issue — status values: The Booknote plugin and existing book notes use Korean status values. The dashboard accepts both. Use either set:
| Korean (plugin default) | English (system standard) |
|------------------------|--------------------------|
| 계획 | inbox |
| 진행 | active |
| 완료 | done |

---
title: "Book Title"
created: {today}
modified: {today}
type: literature
status: done
author: "Author Name"
tags:                       # MUST be "tags" (plural), YAML array — never "tag"
  - src/book
  - {1-2 domain tags}
category: "domestic" | "foreign"
total_page: N
publish_date: YYYY-MM-DD
start_read_date: YYYY-MM-DD
finish_read_date: YYYY-MM-DD
my_rate: 1-10
book_note: true
---

4. Write

  • Notes: Rewrite key content in own words (interpretation, not quotation)
  • Quotes: Notable original passages
  • Source: Source information

5. Backlink (wikilink)

본문에서 강하게 연관된 노트가 있으면 반드시 [[wikilink]]를 남긴다.

방식 사용 시점 예시
인라인 링크 본문 문맥 속에서 자연스럽게 언급될 때 저자는 [[Antifragile System Design 1 Optionality]]에서 말한 옵셔널리티와 같은 논지를 펼친다
## Related 섹션 직접 언급하긴 어렵지만 주제적으로 연관될 때 본문 하단에 리스트로 정리
  • 두 방식을 혼용해도 좋다 (인라인 + Related 섹션)
  • YAML frontmatter에 related: 속성을 넣지 않는다 — 본문 wikilink만 사용

6. Place

  • Place under 200_Books/

Output

200_Books/{Book Title}.md — Book note

Quality Gate

  • Property key is tags (plural, YAML array) — not tag (singular/inline)
  • type: literature and src/book tag present
  • Domain tags match the book's subject
  • Notes section uses own words, not mere quotation
  • my_rate included (when provided by user)
