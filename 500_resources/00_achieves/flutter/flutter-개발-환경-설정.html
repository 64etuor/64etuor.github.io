Flutter 개발 환경 설정

아래는 Flutter 개발 환경 설정을 위한 완성된 문서입니다. 이 문서는 Flutter SDK 설치부터 Android SDK 설정, flutter doctor를 통한 환경 설정 확인까지 포함하고 있습니다.

Flutter 개발 환경 설정 가이드

Flutter는 하나의 코드베이스로 iOS와 Android 앱을 개발할 수 있는 강력한 프레임워크입니다. 이 문서에서는 Flutter 개발 환경을 설정하는 방법을 단계별로 자세히 설명하겠습니다.

목차

  1. Flutter SDK 설치
  2. IDE 설치 및 설정
  3. Android SDK 설정
  4. flutter doctor로 환경 설정 확인
  5. 모바일 에뮬레이터 설정
  6. Flutter 프로젝트 생성 및 실행

1. Flutter SDK 설치

1.1. Flutter SDK 다운로드

  • Flutter 공식 웹사이트(https://flutter.dev/)에서 운영체제에 맞는 Flutter SDK를 다운로드합니다.
    • Windows: .zip 파일
    • macOS: .dmg 파일
    • Linux: .tar.xz 파일

1.2. Flutter SDK 압축 해제

  • 다운로드한 파일의 압축을 원하는 디렉터리로 해제합니다.
    • 예시: C:\src\flutter (Windows), ~/flutter (macOS/Linux)

1.3. 시스템 PATH 설정

Flutter SDK를 시스템 PATH에 추가하여 터미널에서 Flutter 명령어를 쉽게 실행할 수 있도록 합니다.

Windows:

  1. 시스템 속성 -> 고급 시스템 설정 -> 환경 변수를 엽니다.
  2. 사용자 변수 또는 시스템 변수의 Path를 선택하고 편집을 클릭합니다.
  3. Flutter SDK의 bin 폴더 경로를 추가합니다.
    • 예시: C:\src\flutter\bin

macOS/Linux:

터미널에 다음 명령어를 입력하여 .bashrc, .bash_profile, 또는 .zshrc 파일에 Flutter 경로를 추가합니다.

export PATH="$PATH:`pwd`/flutter/bin"

변경 사항을 적용하기 위해 터미널을 재시작하거나 source ~/.bashrc를 실행합니다.

1.4. Flutter 설치 확인

Flutter가 올바르게 설치되었는지 확인하기 위해 flutter doctor 명령어를 사용합니다.

flutter doctor

이 명령어는 Flutter 설치 상태를 진단하고 필요한 추가 설치 항목을 안내합니다.

2. IDE 설치 및 설정

2.1. IDE 선택

Flutter 개발을 위해 주로 사용하는 IDE는 다음과 같습니다:

  • Visual Studio Code: 가볍고 빠르며, Flutter 플러그인을 통해 강력한 지원을 제공합니다.
  • Android Studio: 전체적인 Android 개발 도구와 통합되어 있으며, 강력한 디버깅 기능과 에뮬레이터 지원을 제공합니다.

2.2. Visual Studio Code 설정

  1. Visual Studio Code 다운로드 및 설치: https://code.visualstudio.com/ 에서 다운로드합니다.
  2. Flutter 플러그인 설치:
    • VS Code를 실행한 후, 왼쪽 사이드바에서 Extensions 아이콘을 클릭합니다.
    • "Flutter"와 "Dart" 플러그인을 검색하여 설치합니다.
  3. Flutter 프로젝트 생성:
    • View -> Command Palette -> Flutter: New Project를 선택하여 새로운 Flutter 프로젝트를 생성합니다.

2.3. Android Studio 설정

  1. Android Studio 다운로드 및 설치: https://developer.android.com/studio 에서 다운로드합니다.
  2. Flutter 플러그인 설치:
    • Android Studio를 실행하고, Plugins 메뉴로 이동하여 "Flutter" 플러그인을 설치합니다. 이는 Dart 플러그인을 자동으로 설치해줍니다.
  3. Android SDK 설치:
    • Android Studio에서 SDK Manager를 열어 Android SDK, SDK Platform, 그리고 Android Virtual Device (AVD)를 설치합니다.
  4. Flutter 프로젝트 생성:
    • File -> New -> New Flutter Project를 선택하여 새로운 Flutter 프로젝트를 생성합니다.

3. Android SDK 설정

3.1. Android SDK Command-line tools 설치

Flutter는 Android SDK Command-line tools가 필요합니다. 이를 설치하기 위해 다음 단계를 따릅니다.

  1. Android Studio 실행:

    • Android Studio를 실행합니다.

  2. SDK Manager 열기:

    • File -> Settings (macOS에서는 Preferences)로 이동한 후, Appearance & Behavior -> System Settings -> Android SDK를 선택합니다.

  3. SDK Tools 탭 선택:

    • SDK Tools 탭으로 이동합니다.

  4. Android SDK Command-line tools 체크:

    • 목록에서 Android SDK Command-line tools (latest)를 선택합니다.
    • 체크박스에 체크가 되어 있지 않다면, 체크하고 Apply 버튼을 눌러 설치를 진행합니다.

  5. 설치 확인:

    • 설치가 완료되면 OK 버튼을 눌러 설정을 저장합니다.

4. flutter doctor로 환경 설정 확인

4.1. Flutter Doctor 실행

Flutter 개발 환경이 올바르게 설정되었는지 확인하려면, 터미널에서 다음 명령어를 실행합니다.

flutter doctor

4.2. Flutter Doctor 결과 확인

flutter doctor는 다음과 같은 사항들을 점검합니다:

  • Flutter SDK 설치 여부
  • Android SDK 설치 여부
  • Xcode 설치 여부 (macOS에서만)
  • Android Studio 및 Flutter/Dart 플러그인 설치 여부
  • 연결된 디바이스(에뮬레이터 또는 실제 디바이스) 여부

각 항목의 상태가 출력되며, X 마크가 있는 경우 해당 항목에 문제가 있음을 나타냅니다. 문제가 발생한 경우 Flutter는 그에 따른 해결 방법을 제안합니다.

4.3. Android SDK Command-line tools 문제 해결

만약 flutter doctor 출력에서 Android SDK Command-line tools와 관련된 문제가 발생한 경우, 위에서 설명한 대로 SDK Tools 탭에서 Android SDK Command-line tools (latest)를 설치했는지 확인하고, 설치 후 flutter doctor를 다시 실행합니다.

5. 모바일 에뮬레이터 설정

5.1. Android 에뮬레이터 설정

  1. AVD Manager 실행:
    • Android Studio에서 AVD Manager를 엽니다 (Tools -> Device Manager).
  2. 새로운 가상 장치 생성:
    • Create Virtual Device 버튼을 클릭하고, 원하는 기기 유형과 Android 버전을 선택하여 새로운 가상 장치를 만듭니다.
  3. 에뮬레이터 실행:
    • 생성한 가상 장치를 선택하고 Play 버튼을 눌러 에뮬레이터를 실행합니다.

5.2. iOS 에뮬레이터 (macOS 전용)

  1. Xcode 설치:
    • Mac App Store에서 Xcode를 다운로드하고 설치합니다.
  2. Xcode 설정:
    • Xcode를 실행하고 Preferences -> Locations 탭에서 Command Line Tools를 설정합니다.
  3. iOS 시뮬레이터 실행:
    • Xcode의 Window -> Devices and Simulators에서 iOS 시뮬레이터를 실행할 수 있습니다.

6. Flutter 프로젝트 생성 및 실행

6.1. Flutter 프로젝트 생성

Flutter 프로젝트를 생성하여 환경 설정이 제대로 동작하는지 테스트합니다.

flutter create my_flutter_app

6.2. 프로젝트 실행

생성한 Flutter 프로젝트 디렉터리로 이동한 후, 앱을 실행합니다.

cd my_flutter_app
flutter run

flutter doctor를 통해 모든 환경이 제대로 설정되었음을 확인한 후, 앱이 정상적으로 실행되면 Flutter 개발 환경이 성공적으로 설정된 것입니다.

6.3. Hot Reload 및 Hot Restart

  • Hot Reload: 코드 변경 사항을 빠르게 적용하기 위해 터미널에서 r 키를 누릅니다.
  • Hot Restart: 전체 앱을 재시작하기 위해 R 키를 누릅니다.

이 문서에서 설명한 단계를 따라 Flutter 개발 환경을 성공적으로 설정할 수 있습니다. flutter doctor 명령어를 통해 환경 설정이 올바르게 되었는지 확인하고, 필요한 경우 추가 설정을 적용하세요. 이제 Flutter의 다양한 위젯과 플러그인을 활용하여 본격적으로 앱 개발을 시작할 수 있습니다.
