DBMS_MySQL에서의 비트연산 활용 예시

원문 링크

학교나 또는 개발하시다 보면 한번쯤은 비트연산에 관해 들어보셨을듯 합니다.
그런데 mysql을 통한 실무단에서는 잘 사용하지 않는듯 하여
실무에 사용하는 방법을 간단하게 적어봅니다.
사실은 이미 Tip&Tech에서도 나와 있지만 Remind 차원에서 적어봅니다.

쿼리에서 속도가 많이 걸리는 부분이 일반적으로 like 문과 or 같은 것입니다.
그중 가장 안좋은 것이 like와 or를 같이 써야 하는 것이겠지요.

QA 질문을 답하다 보면 필드 값으로 1|2|3|4|5|... 와 같은 구조가 있는데
해당 값중 2와 5값이 들어 있는 row를 찾고 싶다라는 질문을 종종 받습니다.

이런 값을 찾을 때는 대체적으로 아래와 같은 쿼리로 실행을 하지요.
where 필드 like '%2|%' or  필드 like '%5|%'
그런데 이를 비트연산으로 바꾸면 속도가 상당히 개선됩니다.

우선 비교 연산의 속도부터 알아보자면 빠른순서대로 비트연산 > 숫자 > 문자 의 순입니다.

선택항목이 다음과 같이 A~P까지 있고 다중선택 가능한 항목있다고 하겠습니다.
아래에서 0000 0000 0000 0000 이런 부분은 16자리(16bit)의 2진수 입니다.
그리고 1, 2, 4, 8 ... 은 그 2진수를 10진수로 변환한 값이구요.
이 값은 2의0승부터 시작하여 2의 제곱수로 되어 있는걸 아실수 있을겁니다.
그리고 A, B, C, D ... 은 각 10진수에 해당되는 화면에 표시되는 우리가 선택할 항목이구요.
비트 연산은 NOT OR XOR AND 가 있는데 그중 자주 사용하는것은 AND 연산입니다.

0000 0000 0000 0001 1 A
0000 0000 0000 0010 2 B
0000 0000 0000 0100 4 C
0000 0000 0000 1000 8 D
0000 0000 0001 0000 16 E
0000 0000 0010 0000 32 F
0000 0000 0100 0000 64 G
0000 0000 1000 0000 128 H
0000 0001 0000 0000 256 I
0000 0010 0000 0000 512 J
0000 0100 0000 0000 1024 K
0000 1000 0000 0000 2048 L
0001 0000 0000 0000 4096 M
0010 0000 0000 0000 8192 N
0100 0000 0000 0000 16384 O
1000 0000 0000 0000 32768 P

우리가 A 와 B 를 선택한다고 하면 더하기를 해서
0000 0000 0000 0001 1 A
0000 0000 0000 0010 2 B

0000 0000 0000 0011 3 A+B
이런 결과 값이 되는 것입니다.
이때 DB에 저장할때는 3이란 숫자를 저장하게 됩니다.

위의 모든 값을 저장한다고 하면 모든값을 더한다는 말과 같기에
1111 1111 1111 1111 65535 (A+B+C+D+E+F+G+H+I+J+K+L+M+N+O+P) 가 됩니다.
이와 같은 구조는 2진수의 자리수가 늘어날때마다 선택사항을 추가, 확인할수 있는 구조가 됩니다.
A+B = 3, D+F+K = 1064 ....
이런식으로 16개의 선택항목중 어떤것이 선택되었고 선택되지 않았는지를
10진수의 숫자로 구분이 가능하다는 것이죠.

그럼 실제 예제로 사용해보겠습니다.

다중선택 페이지 1.php 

<?php  
  
$aMultiSelect = Array (  
"1"=>"A","2"=>"B","4"=>"C","8"=>"D",  
"16"=>"E","32"=>"F","64"=>"G","128"=>"H",  
"256"=>"I","512"=>"J","1024"=>"K","2048"=>"L",  
"4096"=>"M","8192"=>"N","16384"=>"O","32768"=>"P"  
);  
echo "  
<form method=post action='2.php'>";  
foreach ($aMultiSelect as $sKey => $sVal) {  
echo "  
<input type='checkbox' name='multiselect[]' value='$sKey'>$sVal<br />";  
}  
echo "  
<input type='submit' value='submit'>  
</form>";  
?>

저는 여기서 A D E를 선택했습니다.

선택되어진 값을 + 하여 DB에 저장하는 페이지 2.php 

<?php  
  
$sSum = 0;  
foreach ($_POST[multiselect] as $sKey=>$sVal) {  
  $sSum += $sVal;  
}  
  
//$sSum 을 DB에 저장, A D E를 선택했으니 25겠죠  
?>

DB에서 값을 찾을때...
A(1), B(2), C(4)중 하나라도 포함된것을 찾으려고 합니다.
1+2+4 = 7 이므로
select * from 테이블 where 필드&7>0;
AND(&) 연산은 각 자릿수를 비교해, 두값에 모두 1이 있을 때에만 1을 반환합니다.
0000 0000 0001 1001 25 <== DB에 저장되어 있는 형태
0000 0000 0000 0111 7 <== 비교값

0000 0000 0000 0001 1

결과값 1은 0 보다 크거나 같기에 좀전에 저장했던 데이터가 출력될것입니다.
그러면 D, E 두개다 저장된것을 찾고 싶을 때도 잇을겁니다.
이때는 8+16 = 24 이므로
select * from 테이블 where bit_count(필드&24)=2;
0000 0000 0001 1001 25 <== DB에 저장되어 있는 형태
0000 0000 0001 1000 24 <== 비교값

0000 0000 0001 1000 24
여기서 bit_count는 2진수의 1의 갯수를 세었다고 생각하시면 됩니다.
D, E를 And 연산하여 참인것이 2개인것을 찾아라라는 말입니다.

그럼 And 와 or 가 같이 들어가 있는걸 검색해보겠습니다.
쿼리고 where 필드 like '%A%' or (필드 like '%D%' and 필드 like '%E%')
보기만 해도 시간이 많이 걸릴 쿼리 같으시죠...
이걸 비트연산으로 바꾸면
select * from 테이블 where 필드&1>0 or bit_count(필드&24)=2;
이와 같이 바뀌게 됩니다.

이와같은 비트연산은 속도 뿐만 아니라 쿼리도 간단히 할수 있고 구조적으로도 관리하기가 편합니다.
그래서 비트연산을 사용하는것이죠

간만에 적으니 두서없이 생각나는대로 막 적은것 같습니다.
그래도 도움이 되었으면 좋겠네요. ^^
