Power BI_통합 대시보드 구현

블로그 계획:

예전에 Power BI로 대시보드를 만들 때, 내가 원하는 수준의 기능을 구현하기 위해 참고할 만한 국문 자료가 없어서 대부분 영문 자료를 참조하거나 Microsoft Power BI Documentation을 참조했다. 구축 이후 수정할 일이 크게 없어 방치하다 보니 세세한 요청 사항이 생겼을 때, 나도 어떻게 만들었는지 기억이 안 나서(…) 다시 자료 찾아보는 일이 생겼다. 그런데 아무래도 한국에서 많이 사용을 하는 툴이 아니다 보니 여전히 국문 자료가 부족한 듯 보인다. 그래서 리포트를 만들었던 과정을 복기하는 겸, 리포트 내에 적용했던 여러 사례를 이 블로그를 통해 소개하고자 한다.
기본적인 Power BI의 기능에 대한 강의나 블로그, 영상 자료 등은 이미 있으니 이 블로그에서는 다른 사람들이 다루지 않은 부분으로 판단되는 것들만 적도록 하겠다.

배경:

파편화되어 있던 데이터들을 한 곳에서 관리하고 데이터의 일관성과 무결성을 확보하기 위해 엑셀로 정리하던 기존 자료들을 모두 없애고, 데이터를 Sharepoint 서버에 등록하는 방식으로 업무의 형태를 완전히 바꾸었다. 비구조적인 데이터를 정규화화여 구조화했으므로 데이터 분석 및 시각화가 용이해졌다.

목적:

  1. 업무 효율 극대화: raw data 원본만 서버에 등록하고 나머지 자료는 모두 raw data로부터 자동으로 파생되게 함으로써 기존에 보고 자료 등을 작성하기 위해 데이터를 2차 가공하던 일련의 과정을 모두 없앤다.
  2. 모든 개별 엑셀 보고 자료를 없애고 하나의 대시보드에서 모든 내용을 확인할 수 있도록 한다.
  3. 엔드유저의 필요에 맞게 데이터를 필터링 할 수 있는 동적 보고서를 구현한다.
  4. 기존에 집계하거나 분석하지 않던 데이터 또한 필요에 맞게 분석하여 이용할 수 있도록 한다.

병행 업무:

  1. 엑셀 VBA로 작성한 데이터 입력 솔루션을 없애고(Oauto2.0 인증 문제로 데이터셋 자동 업데이트가 불가능한 문제 발생) 간이 ERP 제작(데이터 입력 시 무결성을 확보하기 위해 솔루션을 직접 제작)
  2. 데이터베이스 아키텍쳐 작성
  3. 데이터 정규화

대시보드 시각화(회사 정보가 나오는 곳은 가림 처리)

1. Overall Dashboard

  • 차트 막대에 마우스 커서를 올려서 생산/계획 세부 내용 확인(툴팁)

2. Dashboard(a week)

3. Line Dashboard1

4. Line Dashboard2

5. Production Summary

6. Defects Summary

  • Defect Cost 분석
  • 차트 막대에 마우스 커서를 올려서 불량 사진 및 내역 확인(툴팁)

7. Defects Details

8. KPIs Overview

  • Target 값은 분기별 목표 실적과 날짜에 동적 연동

9. Time Tracking

10. OEE Summary / Downtime History

11. OPEX Summary

  • 금액의 단위를 KRW와 USD로 출력 가능하게끔 구현(본사 보고용도)

12. HR Summary

13. 일일 근태 현황

14. Process Tracking

15. Production History

16. ERD Diagram
