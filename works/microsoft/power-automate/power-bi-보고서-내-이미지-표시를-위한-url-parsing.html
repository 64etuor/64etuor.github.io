Power BI 보고서 내 이미지 표시를 위한 URL Parsing

목적:

Power BI로 만든 우리 팀의 통합 대시보드의 Defect 탭에는 생산 불량과 FPY 등이 기록된다.
그런데 텍스트로만 표현된 불량 내역은 직관적이지 못하다.
그래서 대시보드에서 바로 불량 사진을 확인할 수 있도록 구현하고자 했다.
우선 구현하고자 하는 기능과 Power BI, Sharepoint List의 기능을 점검했다.

기본 정보:

  • 대시보드 구현 어플리케이션: Power BI
  • 데이터베이스: Sharepoint List (Dataverse나 SQL 등과 다르게 추가 비용이 발생하지 않음. 다만 데이터베이스로 사용하기엔 한계가 명확.)

구현 기능:

  • Power BI 대시보드에서 불량 사진을 바로 확인할 수 있는 기능 구현
    (Power BI에서는 URL이 저장된 열이 있다면 Data Category를 변경하여 사진으로 출력하는 기능 있는 것으로 확인)

한계:

  • Sharepoint List에 Image를 등록하면 레코드 형태로 저장되기 때문에 Power BI에서 바로 출력이 불가능

접근 방법:

  • Sharepoint List에 Image가 등록/수정되면 Power Automate로 JSON을 파싱하여 이미지 URL 추출하여 imgURL이라는 열에 저장

URL 추출 과정:

Sharepoint 커넥터의 Get Attachments 액션을 활용하면 AbsoluteUri를 간단하게 추출할 수 있다. 아래는 커넥터에서 자체 지원하지 않는 경우(이 경우가 훨씬 많다.)를 위해 기록을 남겨둔다.

1. Power Automate의 새로운 Flow를 만들어준다. 사진의 Instant cloud flow 클릭

2. 이름을 정하고 Manually trigger a flow 선택 후 Create

3. Send an HTTP request to Sharepoint 액션 추가 후,

  • Site Address: 파싱 대상 Sharepoint List의 주소 선택
  • Method: GET
  • Uri:
//'listName'에 Sharepoint List의 Internal Name을 넣어준다.
//items다음의 $orderby=Created desc&$top=1 는 리스트의 항목 생성시간을 내림차순으로 정렬하고, 가장 상위의 값 1개를 가져오는 명령이다.

/_api/web/lists/getbytitle('listName')/items?$orderby=Created desc&$top=1

4. Flow Test를 진행하여, Outputs의 body 확인


5. 절대 uri 값 찾기 (VS CODE 같은 외부 IDE에서 확인하면 편하다.)

  • 값을 살펴보면 “_metadata”: 내에 “uri”가 저장되어 있다.
  • 그리고 “image”에 리스트 형태로 “fileName”이 저장되어 있다.
    그 아래로 스크롤을 내리다 보면 fileName을 확인할 수 있다.
    이제 Parse JSON Flow를 추가하여 위 body 데이터에서 필요한 값만 추출해서 imgURL이라는 변수를 생성하고 값을 할당하면 된다.
    그런데 작업 도중에 Sharepoint 커넥터의 Get attachments를 사용하면 더 간단하게 원하는 결과물을 도출할 수 있다는 걸 알게 됐다.
    (위의 Send HTTP request 액션은 너무나 자주 사용하는 액션이므로 그대로 두겠다.)

6. Sharepoint 커넥터의 Get attachments 액션 활용

  • Sharepoint 커넥터의 Get attachments 액션을 사용하면 Sharepoint List에 등록된 항목의 첨부파일 정보 중 “Id”, “AbsoluteUri”, “Display Name”을 가져올 수 있다. 따라서 Parse JSON 액션 등의 과정을 생략할 수 있다.

결과:

1. Power Automate 플로우 완성 및 설명

※ For each는 배열이나 여러 항목 등을 반환하는 액션이 있거나 액션에서 필요로 할 경우, 자동 생성됨

  1. When an item is created or modified: 트리거로 설정한다. 이미지가 등록될 때뿐만 아니라 이미지 수정이 이루어질 경우에도 대응되도록 한다.
  2. Get attachments: 등록/수정된 이미지의 AbsoluteUri를 읽는다.
  3. Condition: imgURL 열에 이미 등록된 값이 있다면 읽어온 AbsoluteUri와 비교하여 서로가 다를 경우에만 항목이 업데이트 되도록 한다.
  4. Update: 불러온 이미지의 AbsoluteUri를 imgURL열에 업데이트한다.
  5. Post message in a chat or channel(옵션사항): Update 액션이 실패할 경우, 알람 메세지를 특정인에게 보낸다.

2. Power BI로 등록된 URL로 이미지 출력

  1. 새로운 imgURL 열이 추가되었으므로 데이터셋을 한 번 새로고침 해준다.
  2. Table View에 가서 imgURL 열의 Data Category를 “Image URL”로 수정해준다.
  3. Power BI 데스크탑 앱으로 확인할 경우 기본 디폴트 이미지만 보이고 실제 사진이 보이지 않는다. 실제 사진은 웹으로 접속해야만 볼 수 있다. 확인해 본다.
    이미지가 정상적으로 출력된다.
  4. 이제 차트에 마우스 커서를 올리면 툴팁으로 사진을 확인할 수 있게 구현해본다. 우선 새로운 Power BI 보고서에 새로운 페이지를 추가한다.
  5. Page의 Format에서 Page type을 Tooltip으로 설정한다. 그리고 캔버스의 크기를 필요한 만큼 조절한다.

  6. 시각적 개체 Table을 삽입하고 필요한 데이터들과 imgURL 열을 넣어준다.
  7. 테이블의 크기, 이미지 크기를 적절하게 조절해준다.
  8. Tooltip 페이지 완성
  9. 이미지가 Tooltip으로 표시되길 원하는 차트로 이동해서 Format > Properties로 이동한다.
    • Options의 Type을 Report page로 설정
    • Page를 위에서 만든 Page로 설정
  10. 최종 확인을 위해 Power BI 리포트를 저장하고 웹에서 확인한다.
    툴팁에서도 이미지가 잘 출력된다.
