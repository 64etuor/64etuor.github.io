PIVOT, UNPIVOT

ğŸ§  Understanding PIVOT and UNPIVOT in SQL

ğŸ“Œ Introduction

  • PIVOT and UNPIVOT are SQL operations used to transform data between row-oriented and column-oriented formats.
  • PIVOT turns unique values from one column into multiple columns, restructuring rows into a cross-tabular format.
  • UNPIVOT does the opposite: it converts columns into rows, flattening a cross-tabular structure back into a normalized format.

ğŸ”„ PIVOT in SQL

ğŸ“ Definition

The PIVOT operator transforms rows into columns, aggregating data based on specified criteria.

ğŸ“œ Syntax

SELECT column1, column2, [pivoted_column1], [pivoted_column2], ...
FROM (source_query)
PIVOT (
    aggregation_function(value_column)
    FOR pivot_column IN ([pivoted_value1], [pivoted_value2], ...)
) AS alias;

ğŸš€ Example of PIVOT

Suppose we have the following Sales table:

Employee Month Sales
Alice January 100
Alice February 150
Bob January 200
Bob February 250

We want to pivot the Month values so that each month becomes a separate column.

PIVOT Query

SELECT Employee, [January], [February]
FROM (
    SELECT Employee, Month, Sales
    FROM Sales
) AS SourceTable
PIVOT (
    SUM(Sales)
    FOR Month IN ([January], [February])
) AS PivotTable;

Result

Employee January February
Alice 100 150
Bob 200 250

ğŸ” Explanation

  1. Source Query:
    The inner query retrieves the data to be pivoted.
  2. Aggregation:
    The SUM(Sales) function aggregates the sales data.
  3. Pivot Column:
    The FOR Month IN ([January], [February]) clause specifies the unique values to pivot.

ğŸ”„ UNPIVOT in SQL

ğŸ“ Definition

The UNPIVOT operator converts columns into rows, effectively reversing a PIVOT operation.

ğŸ“œ Syntax

SELECT column1, column2, pivot_column, value_column
FROM (source_query)
UNPIVOT (
    value_column FOR pivot_column IN ([pivoted_column1], [pivoted_column2], ...)
) AS alias;

ğŸš€ Example of UNPIVOT

Suppose we have the following SalesSummary table (which is the result of the previous PIVOT):

Employee January February
Alice 100 150
Bob 200 250

We want to unpivot the January and February columns back into rows.

UNPIVOT Query

SELECT Employee, Month, Sales
FROM SalesSummary
UNPIVOT (
    Sales FOR Month IN ([January], [February])
) AS UnpivotTable;

Result

Employee Month Sales
Alice January 100
Alice February 150
Bob January 200
Bob February 250

ğŸ” Explanation

  1. Source Table:
    The SalesSummary table is the starting point for the UNPIVOT.
  2. Sales:
    The Sales column holds the values from the unpivoted columns.
  3. FOR Month IN ([January], [February]):
    The column names January and February are converted into row values under the Month column.

ğŸ“Š Comparison of PIVOT and UNPIVOT

Aspect PIVOT UNPIVOT
Purpose Convert rows into columns Convert columns into rows
Use Case Creating cross-tabular summaries Flattening cross-tabular data
Operation Aggregates data into new columns Transforms multiple columns into rows
Result Format Wide table (more columns) Tall table (more rows)

ğŸš€ Practical Use Cases

When to Use PIVOT

  • Generating Reports:
    • Creating summary reports where categories (e.g., months, products) become columns.
  • Data Analysis:
    • Comparing data across multiple dimensions (e.g., sales per region).

When to Use UNPIVOT

  • Normalizing Data:
    • Transforming wide tables into a normalized form suitable for analysis.
  • Preparing Data for Visualization:
    • Many visualization tools work better with long-format data.

ğŸ“ Key Points to Remember

  1. Unique Values:

    • For PIVOT, the values specified in the IN clause should be unique in the pivot column.

  2. Aggregation:

    • PIVOT requires an aggregation function like SUM, COUNT, or AVG.

  3. Aliases:

    • Always provide aliases for subqueries and the resulting PIVOT or UNPIVOT tables.

  4. SQL Dialect:

    • PIVOT and UNPIVOT are supported in SQL Server and Oracle. In MySQL and PostgreSQL, you can achieve similar results using conditional aggregation or custom queries.

ğŸ“š Conclusion

  • PIVOT and UNPIVOT are powerful SQL operations for reshaping data.
  • PIVOT transforms rows into columns, while UNPIVOT does the opposite.
  • Knowing when and how to use these operations helps you manage data effectively for reporting and analysis.
