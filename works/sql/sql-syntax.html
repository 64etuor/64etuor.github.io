SQL Syntax

A Comprehensive Guide to SQL Syntax

Introduction

  • SQL (Structured Query Language) is the standard language used to communicate and manipulate relational databases.
  • It allows you to create, read, update, and delete data within database tables, as well as define their structure and relationships.
  • Understanding SQL syntax is crucial for any work involving data analysis, backend development, or database administration.

Basic Concepts

What is a Relational Database?

  • A relational database organizes data into tables (relations), which consist of rows and columns.
  • Each table represents an entity (e.g., "Customers," "Orders") and each row represents a record in that table. Columns define attributes of these records (e.g., Customer Name, Order Date).

SQL Dialects

  • While SQL is standardized by ANSI/ISO, vendors often implement dialect-specific features. Common dialects include:
    • MySQL: Popular open-source database.
    • PostgreSQL: Advanced open-source database with rich features.
    • Microsoft SQL Server: Enterprise-level database management system from Microsoft.
    • Oracle Database: Robust enterprise-level database system known for scalability.
  • Core syntax is consistent across dialects, but certain functions, data types, and keywords may differ.

Data Definition Language (DDL)

Overview of DDL

  • DDL commands define or alter the structure of the database.
  • They include commands like CREATE, ALTER, and DROP.

Creating Databases and Tables

  • CREATE DATABASE:
    • Syntax: 
      CREATE DATABASE database_name;
  • CREATE TABLE:
    • Syntax: 
      CREATE TABLE table_name (
  column_name1 data_type constraints,
  column_name2 data_type constraints,
  ...
);
    • Example: 
      CREATE TABLE Employees (
  EmployeeID INT PRIMARY KEY,
  FirstName VARCHAR(50),
  LastName VARCHAR(50),
  HireDate DATE
);

Altering Tables

  • ALTER TABLE: Modify an existing table structure.
    • Add a column: 
      ALTER TABLE Employees
ADD Email VARCHAR(100);
    • Modify a column: 
      ALTER TABLE Employees
MODIFY Email VARCHAR(150);
    • Drop a column: 
      ALTER TABLE Employees
DROP COLUMN Email;

Dropping Databases and Tables

  • DROP TABLE: Deletes a table and all its data permanently. 
    DROP TABLE Employees;
  • DROP DATABASE: Removes a database entirely. 
    DROP DATABASE company_db;

Data Manipulation Language (DML)

Overview of DML

  • DML commands manipulate the data stored in tables.
  • Common DML commands include INSERT, UPDATE, and DELETE.

Inserting Data

  • INSERT: Add new rows to a table.
    • Syntax: 
      INSERT INTO table_name (column1, column2, ...)
VALUES (value1, value2, ...);
    • Example: 
      INSERT INTO Employees (EmployeeID, FirstName, LastName, HireDate)
VALUES (1, 'John', 'Doe', '2024-01-15');

Updating Data

  • UPDATE: Modify existing records.
    • Syntax: 
      UPDATE table_name
SET column1 = value1, column2 = value2
WHERE condition;
    • Example: 
      UPDATE Employees
SET Email = 'john.doe@example.com'
WHERE EmployeeID = 1;

Deleting Data

  • DELETE: Remove rows that meet a specific condition.
    • Syntax: 
      DELETE FROM table_name
WHERE condition;
    • Example: 
      DELETE FROM Employees
WHERE EmployeeID = 1;

Data Querying (SELECT Statements)

SELECT Statement Overview

  • SELECT: The most common SQL command used to query data from one or more tables.
  • Syntax: 
    SELECT column1, column2, ...
FROM table_name
WHERE conditions
GROUP BY grouping_columns
HAVING conditions_on_aggregates
ORDER BY sorting_columns;

Selecting Specific Columns

  • Retrieve certain columns: 
    SELECT FirstName, LastName
FROM Employees;
  • Retrieve all columns: 
    SELECT *
FROM Employees;

Filtering Rows (WHERE Clause)

  • WHERE: Limits which rows are returned. 
    SELECT FirstName, LastName
FROM Employees
WHERE HireDate > '2024-01-01';

Sorting Results (ORDER BY)

  • ORDER BY: Sorts results ascending (ASC) or descending (DESC). 
    SELECT FirstName, LastName, HireDate
FROM Employees
ORDER BY HireDate DESC;

Aggregation Functions

  • COUNT, SUM, AVG, MAX, MIN 
    SELECT COUNT(*) AS TotalEmployees
FROM Employees;
  • GROUP BY: Aggregate functions often used with GROUP BY. 
    SELECT HireDate, COUNT(*) AS HiresOnDate
FROM Employees
GROUP BY HireDate;
  • HAVING: Filters groups after aggregation. 
    SELECT HireDate, COUNT(*) AS HiresOnDate
FROM Employees
GROUP BY HireDate
HAVING COUNT(*) > 1;

Joining Tables

  • JOIN: Combine rows from multiple tables based on related columns.
    • INNER JOIN: Returns matching rows in both tables. 
      SELECT e.FirstName, e.LastName, d.DepartmentName
FROM Employees e
INNER JOIN Departments d ON e.DepartmentID = d.DepartmentID;
    • LEFT JOIN: Returns all rows from left table, and matched rows from right table.
    • RIGHT JOIN: Returns all rows from right table, and matched rows from left table.
    • FULL OUTER JOIN: Returns all rows from both tables, matched or not.

Subqueries

  • A subquery is a query nested inside another query. 
    SELECT FirstName, LastName
FROM Employees
WHERE EmployeeID IN (
  SELECT EmployeeID
  FROM Projects
  WHERE ProjectStatus = 'Active'
);

Data Control Language (DCL) and Transaction Control

Grants and Revokes (DCL)

  • GRANT: Gives permissions to a user or role. 
    GRANT SELECT ON Employees TO analyst;
  • REVOKE: Removes permissions. 
    REVOKE SELECT ON Employees FROM analyst;

Transactions

  • A transaction is a sequence of SQL operations treated as a single unit of work.
  • COMMIT: Saves changes made during the transaction.
  • ROLLBACK: Undoes all changes made in the transaction if an error occurs. 
    START TRANSACTION;
UPDATE Employees SET FirstName = 'Jane' WHERE EmployeeID = 2;
COMMIT;

Constraints and Keys

Primary Keys

  • Uniquely identifies each row in a table.
  • Only one primary key per table, which can consist of one or multiple columns.

Foreign Keys

  • Establish relationships between tables.
  • A foreign key in one table references a primary key in another table.

Other Constraints

  • NOT NULL: Prevents a column from having NULL values.
  • UNIQUE: Ensures all values in a column are distinct.
  • CHECK: Ensures that values meet a specific condition.
  • DEFAULT: Assigns a default value if no value is provided.

Indexes and Optimization

What is an Index?

  • An index speeds up queries by allowing the database engine to find data faster.
  • Trade-off: Faster reads, but slower writes (INSERT/UPDATE/DELETE) due to maintenance of the index.

Creating Indexes

CREATE INDEX idx_employees_lastname
ON Employees (LastName);

Advanced Topics

Views

  • A view is a virtual table based on the result of a query. 
    CREATE VIEW ActiveEmployees AS
SELECT FirstName, LastName
FROM Employees
WHERE Status = 'Active';

Stored Procedures and Functions

  • Stored Procedures: Precompiled collections of SQL statements stored in the database.
  • Functions: Similar to procedures but return a value.
  • Syntax varies by SQL dialect.

Window Functions

  • Perform calculations across sets of rows related to the current row without collapsing them into a single output row. 
    SELECT FirstName, LastName,
  RANK() OVER (ORDER BY HireDate) AS HireOrder
FROM Employees;

Conclusion

Key Takeaways

  • SQL syntax provides a powerful and intuitive way to interact with relational databases.
  • By mastering DDL, DML, and SELECT queries, you can handle most database operations.
  • Advanced features like indexing, views, and stored procedures enhance performance and maintainability.
  • Regular practice, along with consulting official documentation and reputable guides, will solidify your SQL skills.
