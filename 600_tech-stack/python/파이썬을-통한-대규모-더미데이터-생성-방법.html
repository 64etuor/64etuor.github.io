파이썬을 통한 대규모 더미데이터 생성 방법

Seanlee님의 블로그 원글

더미데이터

머신러닝 및 딥러닝 모델 학습 연습이나 SQL 기반으로 분석 연습을 할때, 더미 데이터가 필요 하기 마련입니다. 하지만 현실에서 양질의 데이터 를 얻는것은 결코 쉬운일이 아닙니다. 파이썬의 Faker를 통해 부담없고 안전한 더미 데이터를 생성하고, MySQL에 입력하는 방법을 알아보겠습니다.

1. 테이블 및 데이터 이해

아래 회원 테이블에 데이터를 10,000개 생성해보도록 하겠습니다. 입력이 필요한 항목은 9개 입니다.

  • name: 이름 (문자)
  • phone: 휴대전화 (문자, 010- **** - **** 형식)
  • user_status: 유저상태 (문자, ['active', 'inactive', 'dormancy'])
  • user_class: 유저등급 (문자, ['일반','Family', 'VIP', 'VVIP'])
  • email: 이메일 주소 (문자)
  • marketing_agree: 마케팅 수신동의 (참거짓)
  • social_login: 소셜 로그인 (문자, ['google','facebook','kakao','naver'])
  • last_logged_at: 마지막 접속일 (날짜)
  • created_at: 생성일 및 가입일 (날짜)

2. 파이썬 라이브러리 설치

파이썬 라이브러리인 faker와 sqlalchemy를 이용해서 대량의 더미 데이터를 만들어보도록 하겠습니다.

pip install Faker
pip install mysqlclient
pip install sqlalchemy

Faker: https://pypi.org/project/Faker/
sqlalchemy: https://pypi.org/project/SQLAlchemy/

3. Faker를 이용해서 더미 데이터 입력

Faker 공식문서 에서 다양한 종류의 더미데이터 확인이 가능합니다.

3-1. 한국인 이름 생성 연습

from faker import Faker
fake = Faker('ko_KR') # locale 정보 설정
Faker.seed() # 초기 seed 설정

name = fake.name()
print(name)

3-2. 무료 이메일 주소 생성 연습

from faker import Faker
fake = Faker('ko_KR') # locale 정보 설정
Faker.seed() # 초기 seed 설정

name = fake.free_email()
print(name)

3-3. 대규모 더미 데이터 생성 연습

from faker import Faker
fake = Faker('ko_KR') # locale 정보 설정
Faker.seed() # 초기 seed 설정

repeat_count = 1000 # 1000 회 반복하며 더미 데이터 생성
email = [fake.unique.free_email() for i in range(repeat_count)] # unique 추가시 고유값 생성
print(email)

3-4. 입력에 필요한 더미 데이터 생성!

앞서 정의한 9개의 항목에 대해 10,000 개의 레코드를 생성합니다.

  • 핸드폰 번호 의 경우는 010 이후에 4자리, 4자리 랜덤 숫자 조합을 생성했으며, zfill을 통해 4자리 문자열 앞에 자리를 0으로 채워줍니다. (가령, 15 => 0015, 835 => 0835)
  • 회원 등급 은 4가지 등급 ['일반', 'Family', 'VIP', 'VVIP'] 중 하나를 랜덤으로 선택합니다.
  • 이메일 은 10,000번이 반복되어 생성되는 동안 각각은 고유한 이메일 주소를 갖습니다.
  • 마케팅 수신동의 는 0,1 둘 중에 하나를 랜덤으로 선택합니다.
  • 소셜로그인 은 4가지 종류 ['google', 'facebook', 'kakao', 'naver'] 중 하나를 랜덤으로 선택합니다.
  • 가입일 과 마지막 로그인 정보는 오늘 기준으로 지난 1년 사이의 일자를 랜덤으로 선택합니다.

생성된 9개의 리스트를 바탕으로 판다스의 데이터프레임에 넣어주는 작업을 합니다. 이는 후에 데이터베이스에 입력을 편하게 하기 위함입니다.

from faker import Faker
import random
import pandas as pd

fake = Faker('ko_KR')
Faker.seed(1)

repeat_count = 10000

name = [fake.name() for i in range(repeat_count)]
phone = [
    ('010-'+str(random.randint(1, 9999)).zfill(4)
        +'-'+str(random.randint(1, 9999)).zfill(4))
    for i in range(repeat_count)
]
user_status     = ['active' for i in range(repeat_count)]
user_class         = [random.choice(['일반','Family', 'VIP', 'VVIP']) for i in range(repeat_count)]
email             = [fake.unique.free_email() for i in range(repeat_count)]
marketing_agree = [random.choice(['0','1']) for i in range(repeat_count)]
social_login     = [random.choice(['google','facebook','kakao','naver']) for i in range(repeat_count)]
last_logged_at     = [fake.date_between(start_date = '-1y', end_date ='today') for i in range(repeat_count)]
created_at         = [fake.date_between(start_date = '-1y', end_date ='today') for i in range(repeat_count)]

df = pd.DataFrame()
df['name'] = name
df['phone'] = phone
df['user_status'] = user_status
df['user_class'] = user_class
df['email'] = email
df['marketing_agree'] = marketing_agree
df['social_login'] = social_login
df['last_logged_at'] = last_logged_at
df['created_at'] = created_at

print(df)

4. sqlalchemy를 통해 데이터베이스에 입력

4-1. 데이터프레임을 딕셔너리 변환

앞서 생성된 데이터프레임 (df)를 ".to_dict"를 통해 키-벨류 형식으로 바꿔줍니다.

records = df.to_dict(orient='records')

4-2. 데이터베이스 접속 정보 입력 (MySQL)

from sqlalchemy import create_engine
import sqlalchemy as db

username     = '{user_name}'
password     = '{password}'
host         = '{database_host}'
port         = 3306
db             = '{db_name}'

engine = create_engine(f"mysql://{username}:{password}@{host}:{port}/{db}?charset=utf8mb4")

4-3. 연결 및 데이터 입력

입력할 테이블 정보 ('users') 를 정의합니다.
INSERT를 진행할 값에 대한 정보로 앞서 4-1에서 변환한 키-벨류인 딕셔너리를 전달합니다.

with engine.connect() as conn:
    
    metadata = db.MetaData()
    table = db.Table('users', metadata, autoload=True, autoload_with=engine)

    query = db.insert(table).values(records) 
    result_proxy = conn.execute(query)

결과 확인

10,000 건의 데이터가 성공적으로 입력된 것을 확인할 수 있습니다.

데이터의 사용 목적에 따라 더미 데이터의 논리적 결함이 문제가 될 수 있습니다. 가령, 최근 접속일이 가입일 이전에 발생하는 케이스라던지, 고객 등급에서 최고 등급의 비중이 유난히 높은 케이스 등 현실의 구조를 반영하지 못할 수 있습니다. 이 점을 잘 참고해서 데이터를 확인할 필요가 있습니다.
