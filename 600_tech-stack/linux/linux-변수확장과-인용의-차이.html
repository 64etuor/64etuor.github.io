Linux 변수확장과 인용의 차이

쉘 스크립트에서 $변수와 "${변수}"의 차이를 이해하려면 변수 확장(Variable Expansion)과 인용(Quoting)의 역할을 알아야 합니다.

1. $변수와 "${변수}"의 차이

쉘에서 변수를 참조할 때:

  • $변수: 공백이 있을 경우 문제가 발생할 수 있음
  • "${변수}": 안전하게 변수 값을 감싸서 처리 가능

2. 차이가 발생하는 이유

❌ $변수 사용 시 문제 예제 (공백 포함)

FILENAME="my file.txt"
echo $FILENAME

출력:

my file.txt
  • 쉘은 공백을 구분자로 인식하기 때문에 "my"와 "file.txt"를 두 개의 인자로 인식하여 문제가 발생할 수 있음.
cp $FILENAME /backup/

이 경우, cp 명령어는 my와 file.txt를 두 개의 인자로 해석하여 오류가 발생할 수 있음.

✅ "${변수}" 사용 시 안전한 예제

FILENAME="my file.txt"
echo "${FILENAME}"

출력:

my file.txt

cp "${FILENAME}" /backup/
  • 변수 전체를 하나의 인자로 유지하므로 공백이 있어도 안전하게 처리됨.

3. " 안과 밖에서 $() 사용 차이

쉘에서 $(명령어 실행)은 명령어 치환(Command Substitution)을 수행합니다.

📌 $()를 "" 안에서 사용할 때

TIMESTAMP="$(date +"%Y%m%d_%H%M%S")"
  • "$(date ...)"를 감싸면 전체를 하나의 문자열로 처리.
  • date 명령어의 결과를 안전하게 TIMESTAMP 변수에 저장할 수 있음.

❌ $()를 "" 없이 사용할 때 문제

TIMESTAMP=$(date +"%Y %m %d")
  • 공백이 포함된 결과(2025 02 12)가 나오면, 쉘이 2025, 02, 12를 각각 다른 인자로 인식하여 예상치 못한 동작을 유발할 수 있음.

✅ 따라서, "$(명령어)"로 감싸는 것이 안전함.

4. 스크립트에서 적용된 부분

BACKUP_FILE="${BACKUP_DIR}/important_${TIMESTAMP}.bak"
  • BACKUP_DIR, TIMESTAMP에 공백이 포함될 가능성을 방지.
  • ${변수}를 쓰면 변수 이름을 명확하게 구분할 수 있음.
if [ ! -d "$BACKUP_DIR" ]; then
  • "${BACKUP_DIR}" 없이 $BACKUP_DIR을 사용해도 되지만, 폴더 경로에 공백이 포함될 가능성을 방지하기 위해 "${BACKUP_DIR}"을 사용하는 것이 안전함.

결론

💡 변수 사용 시 "로 감싸는 것이 안전함

  • 공백, 특수 문자 문제 방지
  • 변수 전체를 하나의 인자로 인식
  • 예상치 못한 쉘 해석 오류 방지

🚀 Best Practice

변수를 사용할 때는 항상 "$변수" 또는 "${변수}" 형태로 감싸는 것이 좋다!
