Shell Alias 설정 - PowerShell & macOS

Shell Alias 설정: PowerShell & macOS

한 줄 요약

gs를 치면 git status가 실행되게 만드는 방법. PowerShell과 macOS(zsh) 양쪽 다.

도입: 같은 목적, 다른 메커니즘

# macOS (zsh)
alias gs="git status"

# PowerShell
Set-Alias -Name gs -Value git   # ← 이건 안 된다. 인자를 못 넘긴다.

Bash/Zsh의 alias는 문자열 치환이라 인자 포함이 자연스럽지만, PowerShell의 Set-Alias는 명령어 이름만 매핑한다. git status처럼 인자가 붙은 명령은 alias로 안 되고 함수를 써야 한다. 이 차이가 모든 혼란의 원인이다.

핵심 구조: Alias가 로드되는 방식

macOS (zsh) PowerShell
설정 파일 ~/.zshrc $PROFILE
파일 위치 /Users/{user}/.zshrc C:\Users\{user}\Documents\PowerShell\Microsoft.PowerShell_profile.ps1
로드 시점 터미널 새 세션 시작 시 PowerShell 새 세션 시작 시
alias 메커니즘 문자열 치환 명령어 → 명령어 매핑 (함수 필요)
적용 명령 source ~/.zshrc . $PROFILE

macOS (zsh) 설정법

1. 기본 alias

.zshrc에 한 줄 추가하면 끝이다.

# ~/.zshrc
alias gs="git status"
alias ga="git add ."
alias gc="git commit -m"
alias gp="git push"
alias ll="ls -alF"
alias ..="cd .."
alias ...="cd ../.."

적용:

source ~/.zshrc

2. 인자를 받는 alias (함수형)

alias는 뒤에 인자를 붙이는 게 제한적이다. 인자를 중간에 넣어야 하면 함수를 쓴다.

# ~/.zshrc

# "mkcd dirname" → mkdir + cd를 한 번에
mkcd() {
	mkdir -p "$1" && cd "$1"
}

# "gcom 메시지" → git commit -m "메시지"
gcom() {
	git commit -m "$1"
}

# "port 3000" → 해당 포트 사용 프로세스 확인
port() {
	lsof -i :"$1"
}

3. 파일 분리 (alias가 많아지면)

# ~/.zshrc 끝에 추가
[ -f ~/.aliases ] && source ~/.aliases

# ~/.aliases (별도 파일)
alias gs="git status"
alias ga="git add ."
# ... 나머지 alias

PowerShell 설정법

1. Profile 파일 확인 및 생성

# Profile 경로 확인
echo $PROFILE
# → C:\Users\{user}\Documents\PowerShell\Microsoft.PowerShell_profile.ps1

# 파일이 없으면 생성
if (!(Test-Path -Path $PROFILE)) {
	New-Item -ItemType File -Path $PROFILE -Force
}

# 편집
notepad $PROFILE

2. 단순 alias (인자 없는 경우만)

Set-Alias는 명령어 이름 → 명령어 이름 매핑만 된다. 인자 포함 불가.

# $PROFILE
Set-Alias -Name np -Value notepad
Set-Alias -Name ex -Value explorer
Set-Alias -Name c -Value cls

np file.txt → notepad file.txt로 치환된다. 여기까지는 단순하다.

3. 인자가 필요한 alias → 함수로 작성

PowerShell에서 git status를 gs로 줄이려면 함수가 유일한 방법이다.

# $PROFILE

function gs { git status }
function ga { git add . }
function gp { git push }
function gl { git log --oneline -20 }

function gc {
	param([string]$message)
	git commit -m $message
}

# mkdir + cd 한 번에
function mkcd {
	param([string]$dir)
	New-Item -ItemType Directory -Path $dir -Force | Out-Null
	Set-Location $dir
}

# 특정 포트 사용 프로세스 확인
function port {
	param([int]$p)
	Get-NetTCPConnection -LocalPort $p -ErrorAction SilentlyContinue |
		Select-Object LocalPort, OwningProcess, @{
			Name='ProcessName';
			Expression={(Get-Process -Id $_.OwningProcess).ProcessName}
		}
}

4. 적용

. $PROFILE

실행 정책 에러가 나면:

Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser

비교 테이블: 같은 작업, 다른 구문

작업 macOS (zsh) PowerShell
git status alias gs="git status" function gs { git status }
git commit -m gcom() { git commit -m "$1" } function gc { param($m); git commit -m $m }
mkdir + cd mkcd() { mkdir -p "$1" && cd "$1" } function mkcd { param($d); mkdir $d; cd $d }
포트 확인 port() { lsof -i :"$1" } function port { param($p); Get-NetTCPConnection -LocalPort $p }
적용 source ~/.zshrc . $PROFILE
파일 위치 ~/.zshrc $PROFILE

실전 레시피: 양쪽에서 쓸만한 alias 세트

Git 작업 (가장 타이핑을 줄여주는 것들)

zsh:

alias gs="git status"
alias ga="git add ."
alias gp="git push"
alias gl="git log --oneline -20"
alias gd="git diff"
alias gb="git branch"
gcom() { git commit -m "$1"; }
gcam() { git add . && git commit -m "$1"; }

PowerShell:

function gs  { git status }
function ga  { git add . }
function gp  { git push }
function gl  { git log --oneline -20 }
function gd  { git diff }
function gb  { git branch }
function gc  { param($m) git commit -m $m }
function gca { param($m) git add .; git commit -m $m }

AI 코딩 에이전트 (권한 자동 승인)

매번 권한 프롬프트를 확인하기 귀찮을 때. 각 CLI의 자동 승인 플래그를 함수로 감싼다.

PowerShell:

function cc     { claude.exe --dangerously-skip-permissions @args }
function claude  { claude.exe --dangerously-skip-permissions @args }
function codex   { codex.exe --yolo @args }
function gemini  { gemini.cmd --yolo @args }

zsh:

alias cc="claude --dangerously-skip-permissions"
alias claude="claude --dangerously-skip-permissions"
alias codex="codex --yolo"
alias gemini="gemini --yolo"
--dangerously-skip-permissions와 --yolo는 모든 도구 실행을 자동 승인한다.

신뢰할 수 있는 프로젝트에서만 사용할 것. @args(PowerShell) / "$@"(zsh 함수)는 뒤에 붙는 인자를 그대로 전달한다.

디렉토리 탐색

zsh:

alias ll="ls -alFh"
alias ..="cd .."
alias ...="cd ../.."
alias dt="cd ~/Desktop"

PowerShell:

function ll  { Get-ChildItem -Force | Format-Table Mode, Length, LastWriteTime, Name }
function ..  { Set-Location .. }
function ... { Set-Location ../.. }
function dt  { Set-Location ~/Desktop }

주의할 점

PowerShell 함수 이름 충돌

PowerShell에는 기본 alias가 이미 많이 잡혀 있다. gc는 Get-Content의 기본 alias다.

# 충돌 확인
Get-Alias gc
# → gc -> Get-Content

# 기존 alias를 제거하고 함수로 덮어씌우기
Remove-Alias -Name gc -Force -ErrorAction SilentlyContinue
function gc { param($m) git commit -m $m }

자주 충돌하는 이름: gc (Get-Content), gl (Get-Location), gp (Get-ItemProperty), sort (Sort-Object)

zsh: alias vs function 판단 기준

조건 사용
고정된 명령어 문자열 alias
인자를 중간에 삽입 function
조건 분기나 로직 필요 function
단순히 뒤에 인자 붙이기 alias로 충분 (alias gc="git commit -m" → gc "message")
한 줄 요약

macOS는 ~/.zshrc에 alias로, PowerShell은 $PROFILE에 function으로. 인자가 필요한 순간 양쪽 다 함수를 쓰게 된다.
