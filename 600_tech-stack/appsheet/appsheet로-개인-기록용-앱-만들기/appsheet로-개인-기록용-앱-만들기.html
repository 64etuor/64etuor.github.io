3. 스키마 정의 ( 데이터 타입 및 속성 지정 )

[Tip] Snipaste를 활용한 스키마 정의

https://www.snipaste.com/

좋은 툴을 공유해 주신 더베러 ‘한돌’ 님께 감사의 말씀을 전하고 싶습니다. 😊

  1. F1을 눌러서 원하는 영역을 선택합니다.

  1. F3을 눌러서 선택한 영역을 화면에 Floating 시킵니다.
  2. 화면을 옆으로 스크롤해서 열의 정보를 참조하며 열의 속성을 작성합니다.

속성명 세부 설명

속성명 (Name) 설명 (Description)
Name 컬럼의 내부 식별자로 사용되는 이름. 데이터베이스 또는 스프레드시트의 열(header)과 동일하거나 변경 가능.
Type 컬럼의 데이터 유형을 지정하는 속성 (예: Text, Number, Date, Enum 등).
Key? 해당 컬럼을 테이블의 고유 식별자로 설정하는 옵션. 중복되지 않는 값이어야 하며, 한 테이블당 하나만 설정 가능.
Label? 앱에서 데이터를 표시할 때 기본적으로 사용되는 컬럼. Key 컬럼과 함께 사용되며, 여러 개 설정 가능.
Formula (App Formula) 특정 수식을 기반으로 값을 자동으로 계산하는 필드. 값이 자동으로 업데이트됨. (예: [Price] * [Quantity])
Show? 컬럼이 앱 화면에서 보이도록 설정하는 옵션. FALSE로 설정하면 숨겨짐.
Editable? 사용자가 필드를 직접 수정할 수 있는지 여부를 설정. 조건을 지정하여 특정 경우에만 수정 가능하도록 설정할 수도 있음.
Required? 필수 입력 필드 여부를 설정. TRUE로 설정하면 사용자가 값을 입력하지 않으면 저장할 수 없음.
Initial Value 데이터 입력 시 기본적으로 설정되는 초기값. (예: NOW()를 사용하면 현재 날짜/시간 자동 입력)
Display Name 앱 화면에서 해당 컬럼이 표시될 때의 사용자 친화적인 이름. 원래 컬럼 이름과 다르게 표시 가능. (예: Full Name 대신 Customer Name)
Description 필드에 대한 설명을 제공하는 옵션. 사용자 가이드 역할을 하며, 입력 시 참고할 수 있도록 도움을 줌.
Search? 해당 컬럼의 값이 검색 기능에서 사용될 수 있도록 설정하는 옵션. TRUE로 설정하면 검색 가능.
Scan? 바코드 또는 QR 코드를 스캔하여 값을 입력할 수 있도록 설정. 모바일 앱에서 유용하게 사용됨.
NFC? NFC (Near Field Communication) 태그를 스캔하여 데이터를 입력할 수 있도록 설정하는 옵션.
PII? 개인 식별 정보(PII, Personally Identifiable Information) 여부를 설정. TRUE로 설정하면 데이터 보호를 위해 추가적인 보안 조치가 적용될 수 있음.

추가 질문이 있으면 알려주세요! 🚀

뷰의 속성 설명

속성명 (Name) 설명 (Description)
View name 해당 뷰의 고유한 이름을 설정. 앱 내에서 다른 뷰와 구별하기 위한 용도로 사용됨.
For this data 어떤 테이블 또는 슬라이스(Slice)를 이 뷰에서 표시할지를 지정.
View type 뷰의 형태를 선택 (예: table, deck, gallery, form, chart, map, dashboard 등).
Position 앱 내에서 이 뷰에 접근하는 버튼의 위치를 설정 (first, middle, menu, ref 등).
Sort by 데이터를 정렬할 기준 컬럼을 지정 (예: started_at을 Descending 정렬).
Group by 특정 컬럼을 기준으로 데이터를 그룹화하여 표시.
Group aggregate 그룹별 데이터를 요약하여 표시하는 방식 (COUNT, SUM, AVERAGE, MAX, MIN 등).
Main image 각 행(Row)에 표시할 대표 이미지를 지정하는 컬럼.
Image shape 대표 이미지의 모양을 설정 (Square Image, Round Image, Full Image).
Primary header 각 행의 메인 헤더(제목)로 사용할 컬럼을 선택.
Secondary header 각 행의 서브 헤더(보조 텍스트)로 사용할 컬럼을 선택.
Summary column 각 행의 우측 상단 요약 정보로 표시할 컬럼을 선택.
Nested table column 참조 데이터(Ref Values)를 중첩 테이블 형태로 보여줄 컬럼을 지정.
Show action bar 각 행의 하단에 액션 버튼(예: 삭제, 수정, 맵 보기 등)을 표시할지 여부 (ON/OFF).
Actions 해당 뷰에서 사용할 액션 버튼을 설정 (Delete, Edit, View Map 등).

따라서 상황에 맞는 적절한 조건을 설정하여 줍니다. 아래는 plan을 complete처리 하는 액션을 비활성화 하는 예제 입니다.

9. Virtual Column 생성하기

Virtual Column이란?
  • 데이터베이스에 실제로 존재하지 않는 열
  • 수식으로 계산된 결과를 실시간으로 보여줌(동적 데이터)
  • 앱에서만 존재하고, Google Sheets 등 원본 데이터에는 저장되지 않음

예: 마감기한까지 남은 일수를 만약 정적으로 db에 저장한다면?

→ 한 번 저장된 값은 해당 레코드에 접근하기 전까지 변경되지 않음(Formular 또한 해당 레코드에 접근했을 때 작동)

task, plan, todo 테이블 - (테이블명)_description

FK_(테이블명)_id로 가져온 값을 사용자가 식별할 수 있도록 (테이블명)_description을 표시하는 가상 열 추가

IF(ISBLANK([FK_plan_id]), "",
LOOKUP([FK_plan_id], "plan", "PK_plan_id", "plan_description"))

todo, project 테이블 - days_left (virtual column)

동적으로 표시되는 데이터(todo 기한까지 남은 일수 등) 구현을 위해 가상 열 추가

IFS(
  [status] = "Completed", "Completed",
  [status] = "Pending", "Pending",
  ISNOTBLANK([end_date]),
    IFS(
      FLOOR(HOUR([end_date] - TODAY()) / 24) > 0, CONCATENATE(FLOOR(HOUR([end_date] - TODAY()) / 24), "d left"),
      FLOOR(HOUR([end_date] - TODAY()) / 24) = 0, "Due today",
      FLOOR(HOUR([end_date] - TODAY()) / 24) < 0, CONCATENATE(ABS(FLOOR(HOUR([end_date] - TODAY()) / 24)), "d overdue")
    )
)
