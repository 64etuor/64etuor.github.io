✅ 1.1 일일 작업(Task) 기록

항목 설명
기능명 Task 기록 추가
설명 사용자는 하루 동안 수행한 작업을 기록할 수 있으며, 소요 시간이 자동 계산됨. 이미지 첨부 및 위치 기록 가능
입력값 task_type, task_description, image(선택), location(선택)
처리 과정 1. 사용자가 작업을 시작하면 started_at 자동 기록
2. 사용자가 작업을 완료하면 ended_at 자동 기록 및 duration_min(소요 시간) 계산
3. 작업이 plan에서 파생된 경우, FK_plan_id 저장
출력값 Task 상세 정보 (소요 시간, 첨부 이미지, 위치 포함)
예외 처리 - started_at 없이 ended_at이 입력될 경우 오류 처리

✅ 1.2 일일 계획(Plan) 작성

항목 설명
기능명 Plan 생성
설명 사용자는 하루 계획을 작성할 수 있으며, 해당 계획에서 Task를 생성할 수 있음
입력값 plan_description, task_type, time_period, FK_todo_id(선택)
처리 과정 1. 사용자가 plan_description 입력 후 계획을 생성
2. FK_todo_id가 입력된 경우, 해당 todo에서 파생된 계획으로 저장
3. 사용자가 계획을 실행하면 status가 진행 중으로 변경됨
4. 완료 시 completed_at, completed_date가 기록됨. status는 completed로 변경.
5. 취소 시 cancelled_at, cancelled_date가 기록되며 논리적 삭제 처리. status는 cancelled로 변경.
출력값 Plan 상세 정보 (계획 내용, 상태, 연결된 Task)
예외 처리 - 동일한 plan_description이 중복되지 않도록 처리

✅ 1.3 중기 계획(Todo) 관리

항목 설명
기능명 Todo 생성 및 관리
설명 사용자는 중기 계획 또는 일회성 계획을 등록하고 진행 상태를 관리할 수 있음
입력값 todo_description, task_type, start_date, end_date, is_one_time, FK_project_id(선택)
처리 과정 1. 사용자가 todo_description 및 기타 정보 입력 후 등록
2. FK_project_id가 입력된 경우, project에서 파생된 Todo로 저장
3. 사용자가 todo에서 파생된 plan을 생성하면 status가 In-progress로 변경
4. 완료 시 completed_at, completed_date가 기록됨. status는 completed로 변경.
5. 취소 시 cancelled_at, cancelled_date가 기록되며 논리적 삭제 처리. status는 cancelled로 변경.
6. remaining_days는 end_date - today()로 동적으로 자동 계산(Virtual Column)
출력값 Todo 상세 정보 (진행 상태, 남은 일 수)
예외 처리 - start_date > end_date인 경우 오류 처리

✅ 1.4 장기 프로젝트(Project) 관리

항목 설명
기능명 Project 생성 및 관리
설명 사용자는 장기적인 프로젝트를 생성하고 관련된 Todo를 관리할 수 있음
입력값 project_description, task_type, start_date, end_date
처리 과정 1. 사용자가 project_description 입력 후 프로젝트 생성
2. 사용자가 project에서 파생된 todo 를 생성하면 status가 In-progress로 변경
3. 완료 시 completed_at, completed_date가 기록됨. status는 completed로 변경.
4. 취소 시 cancelled_at, cancelled_date가 기록되며 논리적 삭제 처리. status는 cancelled로 변경.
5. remaining_days는 end_date - today()로 동적으로 자동 계산(Virtual Column)
출력값 Project 상세 정보 (진행 상태, 남은 일 수)
예외 처리 - start_date > end_date인 경우 오류 처리

✅ 1.5 빠른 메모(Quick) 기록

항목 설명
기능명 Quick Note 작성
설명 사용자는 빠르게 기록할 메모를 작성할 수 있음
입력값 note, image(선택), location(선택)
처리 과정 1. 사용자가 메모 입력 후 저장
2. image 및 location이 있는 경우 함께 저장
출력값 Quick Note 상세 정보
예외 처리 - note 입력값 없이 저장 불가

2. 시스템 기능 요구사항

✅ 2.1 자동 상태 업데이트

항목 설명
기능명 Task, Plan, Todo, Project 자동 상태 업데이트
설명 completed_at, cancelled_at 등의 상태 변경 시 자동 업데이트
처리 과정 1. completed_at이 입력되면 status를 완료로 변경
2. cancelled_at이 입력되면 status를 취소로 변경
3. remaining_days 값은 end_date - today()로 자동 계산
출력값 업데이트된 상태 및 남은 일 수
예외 처리 - 상태 변경 시 날짜 필드가 올바르게 입력되었는지 검증

✅ 2.2 Google Task 연동(선택)

항목 설명
기능명 Google Task API 연동
설명 사용자는 Todo 및 Project를 Google Task와 동기화 가능
입력값 google_task_id
처리 과정 1. todo, project에 등록된 시트 행을 기반으로 동기화 요청 (Apps Script)
2. Google Task와 동기화될 경우, google_task_id를 retrieve 후 todo, project 테이블의 google_task_id 열에 저장.
2. google_task_id를 기반으로 동기화
출력값 동기화된 Google Task
예외 처리 - API 오류 발생 시 재시도 처리
