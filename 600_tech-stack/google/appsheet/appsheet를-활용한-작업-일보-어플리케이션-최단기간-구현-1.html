Appsheet를 활용한 작업 일보 어플리케이션 최단기간 구현 - 1

목적:

DV 단계인 배터리 모듈 팩의 레이저 용접 공정의 수기 작업 일보를 보니 날짜, 모듈번호, 불량수량만 적고 있었다. 그 수기 문서를 바탕으로 불량내역을 또 엑셀로 정리하고 있었는데 이는 유의미한 데이터로 생각할 수가 없었다. 임시 작업이라지만 DV 단계에서 불량의 원인과 현상 등을 좀 더 체계적으로 정리할 필요를 느꼈고, 종이 문서를 굳이 전산화하는 작업도 없애고자 Prototype의 작업일보 앱을 만들어 보고자 했다. Prototype이라곤 했지만 충분히 Small Business나 소규모 생산현장 등에서는 Appsheet를 메인 툴로 활용도 가능할 것 같아서 블로그를 남긴다. 내가 작업한 것은 하나의 예시로 본인들이 원하는 앱을 구현해 보시길 바란다.

필요 기능 명세:

  1. 생산 데이터 입력 기능
  2. Unique Key (ModuleKey) 중복 방지
  3. 재작업 데이터 입력 기능
  4. 실시간 대시보드 구현 가능 여부
  5. Rework 작업 시간 계산
  6. 작업 History 확인 기능

1단계: 개발 구상

가장 빠른 기간 내(하루)에 앱을 구현할 수 있는 방법을 찾다 보니 Appsheet가 현재 목적에 딱 어울리는 것 같았다.(예상 개발 시간 4시간 내외) 또한 구글 Spreadsheet에 연동하여 실시간 대시보드를 구현하는 것도 어렵지 않았기 때문에 Appsheet에 구글 Spreadsheet를 연동하여 구현하는 것으로 가닥을 잡았다.

2단계: 테이블 Entity 정의

테이블의 Entity를 정의한다.

Entity 이름 설명
ID 각 레코드의 고유 식별자 (Primary Key)
status 용접 작업의 현재 상태 (완료, 수리 등)
date 용접 작업이 수행된 일시
operator 용접 작업을 수행한 작업자 이름 또는 ID
moduleNo 용접된 모듈의 번호
sideNo 용접된 모듈의 사이드 번호
ModuleKey 모듈을 식별하는 고유 키
isDefect 용접이 불량이 발생했는지 여부 (TRUE/FALSE)
defectCU 구리(Copper) 용접부 불량 여부
defectAL 알루미늄(Aluminum) 용접부 불량 여부
sumDefect 용접 불량 개수의 총합
defectReason 불량의 원인 (예: 재료 문제, 작업 실수 등)
defectDetail 불량에 대한 구체적인 설명
Remark 추가적인 비고 사항
reworkMethod 재작업 방법 (불량 발생 시 재작업 방식)
isReworkStarted 재작업이 시작되었는지 여부 (TRUE/FALSE)
reworkStartTime 재작업이 시작된 시간
isReworkEnded 재작업이 완료되었는지 여부 (TRUE/FALSE)
reworkEndTime 재작업이 완료된 시간
isReworked 재작업이 완료되었는지 여부 (TRUE/FALSE)
isPermantDefect 영구적인 불량인지 여부 (TRUE/FALSE)
reworkedDate 재작업이 완료된 날짜
reworkDuration(sec) 재작업에 소요된 시간 (초 단위)
date_group 날짜별로 그룹화된 정보 (예: 일자별 데이터 그룹핑을 위한 필드)

일단은 간단한 기능만 구현하기 때문에 이정도로 정해두고 필요한 사항은 추가적으로 등록하여 스키마를 업데이트하도록 한다.

구글 Spreadsheet에 테이블을 만들고 해당 Entity 이름이 헤더로 구성된 테이블을 작성한다.

3단계: Appsheet와 Spreadsheet 연동

1. AppSheet 앱 Create에서 Start with existing data를 선택한다.

2. 앱의 이름과 카테고리를 정한다.

3. 이후 데이터 소스를 입력하는 화면에서 미리 만들어 둔 Google Spreadsheet를 선택한다.

4. 이제 데이터 탭을 보면 Spreadsheet에 등록된 Entity의 이름을 기반으로 Data의 Type을 추정해서 아래와 같이 나열된다. 잘못 입력된 것이 없는지 확인하고 수정해준다.

5. 데이터 속성에 대한 설명

속성 설명
Name 해당 필드(열)의 이름
Type 필드의 데이터 유형 (Text, Number, Date 등)
Key? 각 행을 고유하게 식별하는 필드인지 여부 (Primary Key)
Label? 해당 필드가 레코드의 이름으로 사용되는지 여부 (사용자에게 보이는 주요 필드)
Formula 필드의 값을 계산할 수 있는 수식
Show? 해당 필드를 화면에 표시할지 여부를 결정
Editable? 사용자가 해당 필드를 수정할 수 있는지 여부
Require? 필수로 입력해야 하는 필드인지 여부
Initial Value 해당 필드의 초기값을 설정하는 값
Display Name 사용자 인터페이스에 표시되는 필드 이름
Description 필드에 대한 설명 (도움말 등 추가 설명 제공)
Search? 필드를 검색 가능하게 할지 여부
Scan? 해당 필드가 바코드 스캔 기능을 사용할지 여부
NFC? 필드가 NFC 스캔을 지원할지 여부
PII? 해당 필드가 개인정보 식별 정보(PII)를 포함하는지 여부

NAME열에는 Spreadsheet의 열 이름이 입력된다. Appsheet는 열 이름을 기준으로 참조하기 때문에 해당 NAME을 수정하거나 Spreadsheet의 열 이름을 바꾸면 오류가 발생할 수 있다.
Label? 의 경우, 특히나 차트 기능을 구현할 경우 축의 기준으로 적용하기도 하니 참고.
Formula는 사용자로부터 데이터를 입력 받는 것이 아니라 수식으로 자동 계산되게 하고 싶은 경우에 사용한다.
Show? 는 엔드유저에게 해당 열이 보이게 할 지 선택하는 항목이다. 수식을 넣어 다양하게 활용할 수 있으며, 만약 FALSE로 세팅된 경우는 Initial Value도 적용되지 않으니 주의한다.
Editable? 엔드유저가 해당 정보를 수정하게 할 지 여부이다.
Require? 필수 정보로 등록하여 해당 값이 누락될 경우, 누락 메세지와 함께 데이터 저장이 불가능하게 된다.
NULL 값을 피하고 싶은 경우, Initial Value에 디폴트 값을 입력해주면 된다.
나머지는 필요에 따라 적절하게 적용한다.

4단계: Formula 열 정의

우선 ID는 자동으로 Key 속성이 설정되었으니 필요에 따라 나머지 속성 값들을 바꿔주면 된다.
우선 Formula로 값이 입력되게 하고 싶은 열을 정의한다.

a) status: 제품 상태에 관한 열은 위에 나열된 Boolean 값들(isDefect/isReworked/isPermantDefect)을 참조하여 값이 정해지게 한다. 아래의 코드를 Fomula에 넣어준다.

IF(
AND([isDefect] = TRUE, [isReworked] = TRUE), 
"수리완료", 
IF(
   [isDefect] = FALSE, 
   "양품생산", 
   IF(
      AND([isDefect] = TRUE, [isReworked] = FALSE), 
      "수리필요", 
      IF([isPermantDefect] = TRUE, "불량판정", "")
   )

b) sumDefect: defectCU와 defectAL의 합산으로 정의한다.

c) reworkStartTime: Rework 작업시간을 입력 항목이다. 이 [isReworkStarted]값이 TRUE가 되었을 때의 시간 함수 NOW() 가 이 곳에 입력되게 한다. 이미 값이 입력되어 있을 땐 다른 변수로 인해 값이 동적으로 변경되지 않도록 추가적인 수식(ISBLANK)을 추가해준다. 코드 입력창이 따로 indent 기능을 지원하지 않기 때문에 수식 indent는 좀 보기 싫지만 이대로 두겠다.

IF(
AND([isReworkStarted] = TRUE, ISBLANK([reworkStartTime])),
NOW(),
IF(
[isReworkStarted] = FALSE, "", [reworkStartTime]))

d) reworkEndTime은 위와 완전히 동일한 논리로 작성한다.

e) reworkedDate: [isReworked] 값이 TRUE가 되었을 때의 시간을 기록한다.

IF(
AND([isReworked] = TRUE,ISBLANK([reworkedDate])),
NOW(),
[reworkedDate])

f) reworkDuration: [reworkEndTime]에서 [reworkStartTime]을 뺀 값을 Duration으로 리턴한다. 단순히 아래와 같이 수식을 적을 경우, reworkEndTime 값이 Null일 땐 음수가 리턴된다.

[reworkEndTime] - [reworkStartTime]

따라서 Duration이 엔드유저에게 표시되는 시점은 [reworkEndTime]이 입력된 시점, 즉 값이 양수로 변한 상태인 게 바람직하다. Show if?에 추가적인 수식을 적는다. Duration으로 리턴된 값은 직접적으로 숫자와 비교가 불가능하기 때문에 초(TOTALSECONDS)로 변환해 주었다.

TOTALSECONDS([reworkDuration]) > 0

g) EDITABLE? / REQUIRE?: Formula로 값이 수정되는 값은 자동적으로 Non-editable로 작동한다. NOT NULL 속성을 주기 위해 Inital value를 정해두거나, REQUIRE에 체크함으로써 입력 누락을 방지한다.

h) DISPLAY NAME: 기본적으로 어플리케이션에 표시되는 열 이름은 Spreadsheet에서 설정한 열 이름이다. 코드 작성을 용이하게 하고 쿼리문의 오류를 방지하기 위해 모두 영어로 작성되었기 때문에 작업을 하는 사람이 보기에 좋은 형태는 아니다. 따라서 Display Name을 한글로 설정하여 엔드유저가 보기 쉽게 만들어 준다.

i) DESCRIPTION: 열에 대한 설명을 작성한다. 일단 빈 값으로 두었다. 그외 속성들은 바코드 스캔도 필요 없고 굳이 SEARCH 기능을 활용할 필요도, 개인정보도 없기 때문에 모두 FALSE값으로 두거나 기본값으로 두었다.

j) 추가로 moduleNo를 사람이 입력하지 않아도 되게 할 수 있도록 유저 친화적인 기능을 넣어준다. 현재 테이블의 moduleNo의 MAX 값을 계산하고 그 moduleNo가 2개 미만이면(moduleNo 한 개당 2개의 sideNo가 매칭된다.) 그 moduleNo를 리턴하고, 그렇지 않다면 moduleNo + 1 한 값을 리턴한다.
