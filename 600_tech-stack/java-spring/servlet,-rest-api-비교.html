Servlet, REST API 비교

Servlet vs REST API: 비교 및 실무 활용 사례

1. 개요

Servlet과 REST API는 Java 기반의 웹 개발에서 많이 사용되는 기술입니다. Servlet은 전통적인 Java 웹 애플리케이션 개발 방식이며, REST API는 클라이언트-서버 구조에서 데이터를 주고받기 위한 현대적인 아키텍처입니다. 이 문서에서는 두 기술의 차이점과 실무 활용 예를 비교하여 설명합니다.

2. Servlet과 REST API 개념 비교

구분 Servlet REST API
주요 개념 Java EE 기반 웹 요청 처리 기술 HTTP 기반으로 데이터 주고받는 서비스
역할 동적 웹페이지 생성 및 웹 요청 처리 JSON, XML 등의 데이터를 반환하는 API
기본 구조 HttpServlet 클래스를 상속받아 요청을 처리 @RestController를 이용하여 API를 구현
응답 데이터 HTML, 텍스트, JSON 가능 JSON, XML 등의 데이터만 반환
사용 환경 JSP, Java 웹 애플리케이션 웹 및 모바일 앱과의 데이터 통신
요청 처리 방식 doGet(), doPost() 오버라이드 @GetMapping(), @PostMapping() 등의 어노테이션 사용
상태 관리 세션 기반 (HttpSession) 가능 무상태(Stateless) 원칙
배포 방식 WAR 파일로 Tomcat 등에 배포 단독 실행 가능 (Spring Boot 내장 Tomcat)

3. 코드 비교

1) Servlet 예제

@WebServlet("/hello")
public class HelloServlet extends HttpServlet {
    protected void doGet(HttpServletRequest request, HttpServletResponse response)
            throws ServletException, IOException {
        response.setContentType("text/html");
        PrintWriter out = response.getWriter();
        out.println("<h1>Hello, Servlet!</h1>");
    }
}
  • Servlet에서는 HTML을 직접 생성하여 응답
  • Tomcat 같은 WAS에서 실행 필요

2) REST API 예제 (Spring Boot)

@RestController
@RequestMapping("/api")
public class HelloController {
    @GetMapping("/hello")
    public String hello() {
        return "Hello, API!";
    }
}
  • JSON 또는 일반 문자열 응답
  • 프론트엔드(React, Vue) 또는 모바일 앱과 쉽게 연동 가능

4. 실무 활용 예

1) Servlet 실무 활용 예

✅ JSP와 함께 사용하여 동적 웹페이지 생성 ✅ 세션 관리 및 로그인 기능 제공 (HttpSession) ✅ 서블릿 필터(Filter)와 리스너(Listener)를 활용한 보안 및 로깅 ✅ 웹 애플리케이션의 Form 기반 데이터 처리

실무 예제:

  • 전통적인 기업 내부 웹 애플리케이션 (예: 사내 인트라넷, 내부 관리 시스템)
  • 레거시 시스템에서 HTML을 직접 렌더링하는 웹페이지

2) REST API 실무 활용 예

✅ 프론트엔드(React, Vue, Angular)와 통신하는 백엔드 API 제공 ✅ 모바일 앱과 연동하여 JSON 데이터 반환 ✅ OAuth, JWT를 활용한 인증 및 보안 처리 ✅ 마이크로서비스 아키텍처에서 서비스 간 데이터 교환

실무 예제:

  • 쇼핑몰, 금융 서비스의 API 서버
  • IoT 디바이스와 통신하는 RESTful 서비스
  • 클라우드 기반 서비스 (예: AWS Lambda에서 REST API 제공)

5. 결론

선택 기준 Servlet 사용 REST API 사용
웹페이지 렌더링 필요 ✅ 사용 ❌ 사용 안 함
데이터만 반환 (JSON, XML) ❌ 사용 안 함 ✅ 사용
프론트엔드와 분리된 백엔드 ❌ 비효율적 ✅ 적합
상태 유지 (Session 필요) ✅ 사용 가능 ❌ Stateless 방식
마이크로서비스 환경 ❌ 적합하지 않음 ✅ 적합
  • Servlet은 HTML을 직접 렌더링하고 세션을 관리할 때 유용합니다.
  • REST API는 프론트엔드와 분리된 구조에서 데이터를 주고받을 때 적합합니다.
  • 현대적인 웹 개발에서는 REST API를 활용하여 프론트엔드와 백엔드를 분리하는 방식이 더 선호됩니다.

📌 최신 트렌드에서는 REST API(Spring Boot 기반)가 더 널리 사용되며, Servlet은 레거시 시스템이나 특정 목적에서 여전히 활용되고 있음.
