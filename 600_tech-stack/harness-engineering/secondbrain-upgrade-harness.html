수집된 요구사항 (날것 그대로)

발언 요약

  • "AI가 인식하고 찾고 관리하기 편하게 하고 싶은건데"
  • "내가 생각한 것과 외부에서 퍼온 자료를 구분하고 싶거든"
  • "폴더 구조도 좀 마음에 안들긴하거든. 특히 memo과 contents, 그리고 TIL"
  • "이걸 건들면 다른 프로젝트까지 다 변경을 해줘야 해"
  • "에세이와 아티클이 좀 겹쳐" → Published로 통합, type으로 구분
  • "아웃풋 종류가 많은데 정리를 못한거야"
  • "완료조건: 자동분류 에이전트 + tarelog/openclaw 수정까지"
  • "카테고리 도입도 괜찮을거 같은데"
  • "프로젝트가 템플릿을 참조해서 만들게 하는거지. 지금은 프롬프트 안에 들어있어서"
  • "현재 볼트 백업을 꼭 넣고"
  • "백업볼트도 깃을 걸면 좋겠어"
  • "심링크 만들고 안정되면 실제로 옮기자" → 실제로는 새 폴더로 이동 후 옛 경로에 심링크

2-2. CPS 문서

CPS 1회차

Context

사용자는 4,000+ 노트의 Obsidian 볼트(SecondBrain)를 운영 중. 6개 자동화 프로젝트가 볼트에 노트를 생성함. CMDS 시스템(요한님 프로젝트)을 참고해 업그레이드하고자 함.

Problem

  1. "내 생각"과 "외부 자료"가 폴더 곳곳에 혼재
  2. 산출물이 에세이 외에는 정리되지 않음
  3. 프론트매터가 프로젝트마다 제각각
  4. 템플릿이 각 프로젝트 코드에 하드코딩
  5. 폴더 변경 시 6개 프로젝트 경로가 동시에 깨짐

Solution

  • 00-92 번호 기반 폴더 구조 (내것/외부 분리)
  • TARE 프론트매터 6필드 표준
  • 심링크 전략으로 프로젝트 순차 전환
  • 91 Templates/agents/에 템플릿 통합 (Single Source of Truth)
  • 5 Phase 로드맵

2-3. PRD

개요

SecondBrain Obsidian 볼트를 AI 친화적 구조로 전환하고, 연동된 6개 프로젝트의 경로·프론트매터·템플릿을 통합 표준화하며, 최종적으로 자동 분류 에이전트를 구축한다.

3단계: 설계 분석

3-1. 원칙 (Principles)

코어 철학

"호기심이 지식을 모으고, 연결이 나를 만들며, 그 경험이 누군가의 등대가 된다."

지식 프로세스: TARE 4C

단계 이름 의미 폴더 대응
1 Collect 호기심으로 모은다 00번대
2 Connect 연결하여 나를 만든다 20-30번대
3 Create 경험을 결과물로 만든다 40번대
4 Cast 세상에 비춘다 (등대) 발행/공유

시스템 원칙 (5대 원칙)

# 원칙 설명
1 내것과 남의것은 섞지 않는다 내 생각(30번대)과 외부 자료(20번대)는 폴더 레벨에서 분리
2 구조가 먼저, AI는 따라온다 폴더·프론트매터·템플릿이 일관되면 AI는 자동으로 잘 작동
3 하나를 고치면 전체가 따라온다 템플릿이 Single Source of Truth. 중복 정의 금지
4 연결을 끊지 않으면서 바꾼다 심링크로 호환 유지하며 하나씩 전환
5 과거의 생각은 그 자체로 가치다 기존 노트 본문 절대 수정 안 함

AI 원칙 (A1-A6)

# 원칙
A1 CLAUDE.md를 먼저 읽는다
A2 템플릿을 참조해서 생성한다
A3 폴더 번호로 범위를 먼저 좁힌다
A4 자동 분류는 제안만, 실행은 승인 후
A5 모르면 00 Inbox/에 넣는다
A6 기존 패턴을 존중한다

파일 생성 Pre-flight 체크리스트

  1. 대상 폴더가 맞는가? (내것→30, 외부→20, 임시→00, 산출물→40)
  2. 템플릿을 참조했는가? (91 Templates/agents/)
  3. 필수 프론트매터 6필드가 있는가? (type, aliases, author, date created, date modified, tags)
  4. YAML 인덴테이션이 2스페이스인가?
  5. YAML 내 위키링크에 따옴표를 붙였는가? ("링크")
  6. 날짜가 ISO 8601인가? (YYYY-MM-DD)

3-4. 유저 멘탈 모델

볼트 오너

"내 지식의 흐름이 보인다. 호기심이 생기면 00번대에 던지고(Collect), 외부 자료는 20번대, 내 생각은 30번대로 갈라지며(Connect), 40번대에서 결과물이 나와(Create) 세상에 내보낸다(Cast). category로 주제를, tags로 키워드를, index로 모음집을 걸어두면 AI가 뭐든 찾아준다."

AI 에이전트

"CLAUDE.md를 읽으면 규칙을 안다. 폴더 번호로 범위를 좁히고, category→tags→index 순서로 필터링한다. 노트를 만들 때는 Pre-flight 10항목을 확인한다. 확실하지 않으면 00 Inbox/에 넣고 물어본다."

자동화 프로젝트

"나는 정해진 폴더에 노트를 넣는다. 프론트매터는 91 Templates/agents/에서 템플릿을 읽어 따른다. 템플릿이 바뀌면 다음 실행부터 자동 반영된다."
