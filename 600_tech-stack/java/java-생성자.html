Java 생성자

📌 Java에서 Application4 application4 = new Application4(); 문법 자세히 설명하기

위 코드는 Java에서 객체(Object)를 생성하고 변수에 할당하는 과정을 나타냅니다.

📌 코드 구조

Application4 application4 = new Application4();

이 문장은 다음과 같은 3가지 주요 부분으로 구성됩니다:

코드 부분 설명
Application4 클래스 이름 (자료형 역할)
application4 변수 이름 (객체 참조 변수)
new Application4(); 객체 생성 (생성자 호출)

✅ 1. 클래스와 객체 개념

Java는 객체 지향 프로그래밍(OOP, Object-Oriented Programming) 언어이며, 클래스(Class)를 기반으로 객체(Object, 인스턴스)를 생성하여 사용합니다.

🔹 클래스(Class)

  • 객체를 만들기 위한 설계도(템플릿)
  • 속성(변수, 필드)과 동작(메서드)을 정의

🔹 객체(Object)

  • 클래스를 기반으로 생성된 실제 데이터
  • 메모리에 로드되어 사용됨

✅ 2. 코드 실행 흐름 분석

Application4 application4 = new Application4();

이 문장이 실행되는 과정을 하나씩 분석해 보겠습니다.

📌 (1) Application4 application4;

Application4 application4;
  • Application4는 사용자 정의 클래스 (또는 라이브러리 클래스)입니다.
  • application4는 이 클래스의 객체를 참조할 변수(Reference Variable) 입니다.
  • 아직 객체가 할당되지 않았기 때문에 null 값을 가질 수 있음.

📌 비유:

Application4 application4;는 빈 그릇을 준비하는 것과 비슷합니다.
아직 Application4 객체를 담지 않았기 때문에 그릇(application4 변수)은 비어 있음(null 상태).

📌 (2) new Application4();

new Application4();
  • new 키워드는 메모리(Heap 영역)에 새로운 객체를 생성합니다.
  • Application4();는 클래스의 생성자(Constructor)를 호출하여 객체를 초기화합니다.

📌 비유:

new Application4();는 새로운 물건(객체)을 공장에서 만들어내는 과정과 같음.
Application4() 생성자가 객체를 초기화한 후 메모리에 로드됨.

📌 (3) application4 = new Application4();

application4 = new Application4();
  • new Application4();를 통해 생성된 객체가 메모리(Heap 영역)에 로드됩니다.
  • 생성된 객체의 주소(메모리 참조 값)가 application4 변수에 저장됩니다.

📌 비유:

application4 = new Application4();는 빈 그릇에 새로 만든 물건을 담는 것과 같음.
즉, application4 변수는 Application4 객체를 참조하게 됨.

✅ 3. 생성자의 역할

위 코드에서 Application4()는 생성자(Constructor) 입니다.

public class Application4 {
    // 생성자
    public Application4() {
        System.out.println("Application4 객체가 생성되었습니다!");
    }
}

📌 생성자의 역할

  1. 객체를 초기화 (필요한 변수를 설정하거나 메모리 할당)
  2. 클래스와 동일한 이름을 가짐
  3. 리턴 타입이 없음 (void도 사용하지 않음)
  4. new 키워드와 함께 호출됨

🚀 결과:

  • new Application4();를 실행하면 생성자가 호출되고 객체가 초기화됨.
  • 위 생성자에서 "Application4 객체가 생성되었습니다!"가 출력될 것임.

✅ 4. 메모리 구조 (Heap & Stack)

📌 실행 시 메모리 내부 구조

Application4 application4 = new Application4();

📌 실행 과정에서 메모리 내부 구조

  1. Heap 영역
    • new Application4(); 실행 시 Heap 메모리에 객체가 생성됨.
  2. Stack 영역
    • application4 변수(참조 변수)는 Stack에 저장됨.
    • 이 변수는 Heap 영역에 생성된 Application4 객체를 참조하는 주소값을 가짐.

✅ 5. 예제 코드 및 실행 결과

public class Application4 {
    // 생성자
    public Application4() {
        System.out.println("Application4 객체가 생성되었습니다!");
    }

    // 메서드
    public void displayMessage() {
        System.out.println("Hello from Application4!");
    }

    public static void main(String[] args) {
        // 객체 생성
        Application4 application4 = new Application4();
        
        // 메서드 호출
        application4.displayMessage();
    }
}

📌 실행 결과

Application4 객체가 생성되었습니다!
Hello from Application4!

📌 설명

  1. new Application4(); 실행 → 생성자가 호출됨 → "Application4 객체가 생성되었습니다!" 출력
  2. application4.displayMessage(); 실행 → "Hello from Application4!" 출력

📌 결론

코드 요소 설명
Application4 클래스 이름 (자료형 역할)
application4 참조 변수 (객체를 가리키는 변수)
new 메모리에 새 객체를 생성
Application4(); 생성자 호출 (객체 초기화)
