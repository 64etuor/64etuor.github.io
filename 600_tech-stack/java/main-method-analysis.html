Understanding public static void main(String[] args) in Java

In Java, the main method is the entry point of any standalone application. Let's break down each keyword and understand its significance.

Full Syntax

public static void main(String[] args)

1. Breakdown of Each Keyword

Keyword Meaning
public The method is accessible from anywhere (outside the class).
static The method belongs to the class itself, not an instance of the class.
void The method does not return any value.
main The name of the method (this is required by Java).
String[] args An array of strings that can hold command-line arguments.

2. Why is main Method Necessary?

Java needs a defined starting point to execute a program. The JVM (Java Virtual Machine) searches for:

public static void main(String[] args)

If it does not exist, the program will not run.

3. Explanation of Each Keyword in Detail

public (Access Modifier)

  • public means the method must be accessible globally.
  • Since Java applications are executed by the JVM, it must be able to access the method from anywhere.
  • If main were not public, the JVM would not be able to call it.

🔴 Incorrect (private or protected prevents execution)

private static void main(String[] args) {  // JVM cannot access this
    System.out.println("Won't run");
}

static (No Object Needed)

  • static means the method belongs to the class itself, not an instance of the class.
  • The JVM calls main without creating an object of the class.

🔴 Incorrect (main must be static)

public void main(String[] args) {  // JVM cannot call non-static main
    System.out.println("Error");
}

✅ Correct

public static void main(String[] args) {
    System.out.println("Runs correctly");
}

🔍 Why static?

  • If main were not static, Java would require an instance:
public class Test {
    public void main(String[] args) {  // Non-static main
        System.out.println("Error");
    }
}
  • The JVM doesn't create objects automatically, so execution fails.

void (No Return Type)

  • void means the main method does not return any value.
  • If main had a return type (like int), Java would not know what to do with the returned value.

🔴 Incorrect (main must be void)

public static int main(String[] args) {  // JVM expects void
    return 0;
}

✅ Correct

public static void main(String[] args) {
    System.out.println("Runs fine");
}

main (Method Name)

  • main is just a method name, but Java expects it as the entry point.
  • Changing it to anything else will prevent execution.

🔴 Incorrect (main must be named 'main')

public static void start(String[] args) {  // JVM won't recognize this
    System.out.println("Error");
}

✅ Correct

public static void main(String[] args) {
    System.out.println("Runs fine");
}

String[] args (Command-Line Arguments)

  • args is an array of Strings that stores command-line arguments.
  • Example: If you run the program with: 
    java MyProgram Hello 123
    • args[0] = "Hello"
    • args[1] = "123"

Example Usage of args

public class CommandLineExample {
    public static void main(String[] args) {
        if (args.length > 0) {
            System.out.println("First argument: " + args[0]);
        } else {
            System.out.println("No arguments provided.");
        }
    }
}

Running:

java CommandLineExample Hello

Output:

First argument: Hello

4. Can We Modify main Method?

Changing Parameter Name

✅ This works because only the parameter name changes, not the type.

public static void main(String[] input) {  // 'args' can be renamed
    System.out.println("Runs fine");
}

Using varargs Instead of an Array

✅ Java allows varargs (...) instead of an array.

public static void main(String... args) {  // Valid alternative
    System.out.println("Runs fine");
}

Overloading main

✅ Java allows overloading, but the JVM only calls the exact signature.

public class Test {
    public static void main(String[] args) {
        System.out.println("Main method");
        main(5);  // Calls overloaded version
    }

    public static void main(int num) {  // Overloaded main method
        System.out.println("Overloaded main: " + num);
    }
}

Output:

Main method
Overloaded main: 5

🔴 But this does NOT work (JVM only calls the standard signature):

public static void main(int[] args) {  // JVM ignores this
    System.out.println("Won't run automatically");
}

5. Summary

Keyword Purpose
public JVM needs public access to call main.
static No object creation required.
void No return value needed.
main Fixed method name recognized by JVM.
String[] args Stores command-line arguments.

🔹 If any part of main is changed, the program may fail to run.
🔹 Overloading main works, but JVM only calls the standard one.
