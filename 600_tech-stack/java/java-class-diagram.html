Java Class Diagram

Summary: Relationship Symbols

Relationship Symbol Description
Inheritance (Generalization) ▲ Inherits from a parent class (extends)
Interface Realization △ (dashed line) Implements an interface
Association ──> Simple connection between classes
Aggregation ◇──> “Weak” containment relationship
Composition ◆──> “Strong” containment relationship
Dependency ---→ Temporary usage in methods (e.g., parameters)

How to Read Java Class Diagrams

A Java class diagram is one of the UML (Unified Modeling Language) diagrams that visually represents the structure and relationships among classes. To interpret it correctly, you need a solid understanding of basic notation and the various ways to represent relationships.

1. Basic Notation

In a class diagram, each class is represented by a rectangular box, typically divided into three compartments:

+---------------------+
|  Class Name         |
+---------------------+
|  Fields (Attributes)|
+---------------------+
|  Methods (Functions)|
+---------------------+

(1) Class Name

  • The name of the class appears at the top.
  • Generally, the first letter is capitalized.

(2) Fields (Attributes)

  • The variables (attributes, member variables) that the class holds appear in the second compartment.
  • Access modifiers are shown as follows:
    • + (public): Accessible from everywhere
    • # (protected): Accessible within the same package or subclasses
    • - (private): Accessible only within the class
    • ~ (package-private, default): Accessible within the same package
  • Example 
    - name: String
- age: int
    • name is a private String field
    • age is a private int field

(3) Methods (Functions)

  • The methods a class provides are shown in the third compartment.
  • Access modifiers follow the same notation as fields.
  • Method notation 
    + getName(): String
+ setName(name: String): void
    • getName() → a public method returning a String
    • setName(String name) → a public method returning void

2. Relationships (Association, Inheritance, Dependency, etc.)

In class diagrams, relationships between classes are shown using connecting lines and arrows.

(1) Inheritance (Generalization)

  • Parent class → Child class
  • Represented by a triangle arrow (△).
  • Example 
    Animal
   ▲
   │
Dog
    • The Dog class extends the Animal class

(2) Interface Realization

  • Interface → Class
  • Represented by a dashed line with a triangle arrow (△).
  • Example 
    Flyable
   △
   ┆
Bird
    • The Bird class implements the Flyable interface

(3) Association

  • A straight line (─) indicates that two classes are in a relationship.
  • If the relationship has direction, it is shown with an arrow.
  • Multiplicity can also be indicated:
    • 1: Only one
    • 0..*: Zero or more (e.g., arrays, lists)
    • 1..*: One or more
  • Example 
    Person ───> Car
    • Person owns a Car (a typical association)

(4) Aggregation

  • One class contains another (“has-a” relationship).
  • Represented by an empty diamond (◇).
  • Example 
    Team ◇──> Player
    • A Team can have multiple Player objects
    • Each Player can exist independently of the Team

(5) Composition

  • A strong containment relationship (“strong has-a”).
  • Represented by a filled diamond (◆).
  • The contained object ceases to exist if the parent object is removed.
  • Example 
    House ◆──> Room
    • If the House is deleted, the Room is also deleted

(6) Dependency

  • One class uses another class.
  • Represented by a dashed line (---) and arrow (→).
  • Example 
    Order -----> Payment
    • The Order class depends on the Payment class (e.g., as a method parameter)

3. Example Class Diagram

Example: Car System

+-------------------+
| Car               |
+-------------------+
| - brand: String   |
| - speed: int      |
+-------------------+
| + accelerate(): void  |
| + brake(): void       |
+-------------------+

          ▲
          │
+-------------------+
| ElectricCar       |
+-------------------+
| - batteryLevel: int   |
+-------------------+
| + chargeBattery(): void |
+-------------------+

Car ◆──> Engine

Explanation

  1. The Car class has two fields: brand (a String) and speed (an int).
  2. The Car class provides the methods accelerate() and brake().
  3. ElectricCar inherits (extends) the Car class.
  4. ElectricCar adds a batteryLevel field and a chargeBattery() method.
  5. The Car has a composition relationship (◆) with Engine.
    • If Car is removed, the Engine is also removed.
