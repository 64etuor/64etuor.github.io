dotenv-java Library Overview

The dotenv-java library allows Java applications to load environment variables from a .env file. This is particularly useful for managing configuration settings, API keys, and sensitive data without hardcoding them into the source code.

1. Core Components

The dotenv-java library consists of several key components:

1.1 Dotenv Loader (Dotenv)

  • The main class responsible for loading variables from a .env file.
  • Reads key-value pairs and makes them accessible in the application.

1.2 Environment Variables

  • Dotenv loads environment variables without overwriting existing system variables.
  • Variables can be accessed via System.getenv() or Dotenv.get().

1.3 Configuration Files

  • The library looks for a .env file in the project's root directory.
  • Supports multiple files (e.g., .env, .env.dev, .env.test).

2. dotenv-java Module Overview

The dotenv-java library supports various features and configurations:

2.1 Load Environment Variables

  • Reads .env files and makes the variables available in the application.

2.2 Override System Variables

  • By default, dotenv does not overwrite existing environment variables.
  • However, you can configure it to override values if necessary.

2.3 Support for Multiple Files

  • You can load variables from multiple .env files (e.g., .env.test, .env.production).

2.4 Custom Paths

  • The .env file can be placed in a custom directory and loaded explicitly.

3. How dotenv-java Works

The workflow for using dotenv-java in an application follows these steps:

  1. Application Startup: The Java application initializes the dotenv loader.
  2. Dotenv: Reads environment variables from a .env file.
  3. Parsing: Converts .env key-value pairs into usable environment variables.
  4. Storage: Makes variables available in System.getenv() and Dotenv.get().

4. Why Use dotenv-java?

✅ Keeps sensitive data secure – API keys, database credentials, and configurations are not hardcoded in the source code.
✅ Simplifies configuration management – Supports multiple environments (.env.dev, .env.test).
✅ Works across different platforms – Java applications can use environment variables consistently.
✅ Prevents accidental overrides – Existing system variables are not replaced unless explicitly allowed.

5. Sample Code

5.1 Basic Usage

Step 1: Create a .env file

DATABASE_URL=jdbc:mysql://localhost:3306/mydb
API_KEY=abcdef12345

Step 2: Load Environment Variables in Java

import io.github.cdimascio.dotenv.Dotenv;

public class DotenvExample {
    public static void main(String[] args) {
        // Load environment variables from .env file
        Dotenv dotenv = Dotenv.load();

        // Retrieve values
        String databaseUrl = dotenv.get("DATABASE_URL");
        String apiKey = dotenv.get("API_KEY");

        System.out.println("Database URL: " + databaseUrl);
        System.out.println("API Key: " + apiKey);
    }
}

Expected Output:

Database URL: jdbc:mysql://localhost:3306/mydb
API Key: abcdef12345

5.2 Custom .env File Location

If your .env file is in a different directory, specify the path:

Dotenv dotenv = Dotenv.configure()
                     .directory("/config") // Custom directory
                     .load();

5.3 Prevent Overwriting System Variables

By default, dotenv-java does not overwrite existing environment variables.
To force override them:

Dotenv dotenv = Dotenv.configure()
                     .ignoreIfMalformed() // Ignores errors
                     .ignoreIfMissing() // Prevents exceptions if .env is missing
                     .load();

5.4 Accessing Environment Variables

You can access loaded variables using:

String value = dotenv.get("VAR_NAME"); // Preferred way
String systemValue = System.getenv("VAR_NAME"); // Also works

6. dotenv-java in Different Environments

Environment File Name Usage
Development .env.dev Local development settings
Testing .env.test Test-specific configurations
Production .env Secure production settings

To load different files:

Dotenv dotenv = Dotenv.configure()
                     .filename(".env.test") // Load specific file
                     .load();

7. dotenv-java vs System.getenv()

Feature dotenv-java System.getenv()
Supports .env files ✅ Yes ❌ No
Multiple file support ✅ Yes ❌ No
Override system variables ✅ Optional ❌ No
Works in cloud environments ✅ Yes ✅ Yes

8. Integration with Spring Boot

If you are using Spring Boot, dotenv-java can be integrated easily:

8.1 Load Environment Variables in Spring Boot

import io.github.cdimascio.dotenv.Dotenv;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;

@Configuration
public class AppConfig {
    @Bean
    public Dotenv dotenv() {
        return Dotenv.load();
    }
}

8.2 Use in Spring Components

import io.github.cdimascio.dotenv.Dotenv;
import org.springframework.stereotype.Service;

@Service
public class ConfigService {
    private final Dotenv dotenv;

    public ConfigService(Dotenv dotenv) {
        this.dotenv = dotenv;
    }

    public String getDatabaseUrl() {
        return dotenv.get("DATABASE_URL");
    }
}

Conclusion

The dotenv-java library provides a simple and secure way to manage environment variables in Java applications. It helps in keeping configurations flexible, securing sensitive data, and supporting multiple environments.

✅ Easily loads .env files into Java applications
✅ Prevents hardcoding sensitive credentials
✅ Supports multiple environments (dev, test, prod)
✅ Works with Spring Boot and other Java frameworks
