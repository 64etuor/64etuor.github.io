Jackson

Jackson Library Structure Overview

Jackson is a powerful and widely used Java library for handling JSON data. It provides fast, flexible, and feature-rich JSON processing, including serialization and deserialization. The library is structured into multiple core and additional modules, each serving a specific purpose.

1. Core Modules

Jackson's primary core modules include:

1.1 Jackson Core (jackson-core)

  • Provides a low-level streaming API for processing JSON data.
  • Key classes: JsonParser (reading JSON), JsonGenerator (writing JSON).

1.2 Jackson Databind (jackson-databind)

  • Supports data binding between Java objects and JSON.
  • Uses ObjectMapper to serialize (Java Object → JSON) and deserialize (JSON → Java Object).

1.3 Jackson Annotations (jackson-annotations)

  • Provides annotations for controlling serialization and deserialization behavior.
  • Common annotations:
    • @JsonProperty: Customizes field names in JSON.
    • @JsonIgnore: Excludes a field from serialization.
    • @JsonFormat: Defines date/time formatting.

2. Additional Modules

Jackson provides additional modules for extended functionality.

2.1 Data Format Modules

Enable Jackson to work with different data formats beyond JSON:

  • jackson-dataformat-xml (for XML support)
  • jackson-dataformat-csv (for CSV support)
  • jackson-dataformat-yaml (for YAML support)
  • jackson-dataformat-avro (for Avro support)

2.2 Data Type Modules

Provide support for various Java data types:

  • jackson-datatype-jsr310: Supports Java 8 LocalDateTime, Instant, etc.
  • jackson-datatype-joda: Supports Joda-Time.
  • jackson-datatype-guava: Supports Google Guava data structures.

2.3 Performance Modules

Optimize performance using bytecode generation:

  • jackson-module-afterburner: Increases JSON processing speed.

2.4 Integration Modules

Enhance compatibility with other languages and frameworks:

  • jackson-module-kotlin: Provides better support for Kotlin data classes.

3. JSON Processing Approaches

Jackson provides multiple approaches for handling JSON.

3.1 Streaming API

  • Uses JsonParser for reading and JsonGenerator for writing JSON.
  • The most efficient but also the most complex approach.

3.2 Tree Model (Node-based)

  • Uses ObjectMapper.readTree() and writeTree().
  • Converts JSON to JsonNode objects for flexible data handling.

3.3 Data Binding (Java Object Mapping)

  • Uses ObjectMapper.readValue() and writeValue().
  • The most commonly used approach for serializing/deserializing Java objects.

4. Why Use Jackson?

✅ Fast and efficient – Uses streaming-based processing for high performance.
✅ Flexible JSON handling – Supports various data formats (JSON, XML, CSV, YAML, etc.).
✅ Seamless Spring Boot integration – The default JSON processing library in Spring Boot.
✅ Extensible – Easily adaptable with additional modules.

5. Sample Code

Serialize Java Object to JSON

import com.fasterxml.jackson.databind.ObjectMapper;

class User {
    public String name;
    public int age;

    public User(String name, int age) {
        this.name = name;
        this.age = age;
    }
}

public class JacksonExample {
    public static void main(String[] args) throws Exception {
        ObjectMapper mapper = new ObjectMapper();
        User user = new User("Alice", 25);

        // Java Object → JSON
        String jsonString = mapper.writeValueAsString(user);
        System.out.println(jsonString); // {"name":"Alice","age":25}
    }
}

Deserialize JSON to Java Object

public class JacksonExample {
    public static void main(String[] args) throws Exception {
        ObjectMapper mapper = new ObjectMapper();
        String jsonString = "{\"name\":\"Alice\",\"age\":25}";

        // JSON → Java Object
        User user = mapper.readValue(jsonString, User.class);
        System.out.println(user.name + " is " + user.age + " years old.");
    }
}
