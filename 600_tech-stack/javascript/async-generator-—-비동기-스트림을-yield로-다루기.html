Async Generator — 비동기 스트림을 yield로 다루기

Async Generator — 비동기 스트림을 yield로 다루기

페이지네이션 API에서 1000페이지를 순회해야 한다. Promise.all로 동시 요청하면 서버가 429를 뱉고, 재귀 호출로 체이닝하면 코드가 콜백 지옥의 현대판이 된다. 한 페이지씩, 도착하는 대로, 처리하고 다음을 요청하는 흐름이 필요하다.

for await (const page of fetchAllPages('/api/users')) {
	process(page);
}

이 한 줄이 async generator의 존재 이유다.

async function*의 구조

generator 함수에 async를 붙이면 async generator가 된다. 일반 generator가 동기 값을 yield하는 것처럼, async generator는 비동기 작업 사이사이에 값을 yield한다.

async function* countWithDelay(n) {
	for (let i = 0; i < n; i++) {
		await sleep(100);
		yield i;
	}
}

핵심 차이점을 코드로 보면 명확하다:

// 일반 generator → next()가 { value, done } 반환
function* sync() { yield 1; }
sync().next(); // { value: 1, done: false }

// async generator → next()가 Promise<{ value, done }> 반환
async function* async() { yield 1; }
await async().next(); // { value: 1, done: false }

next()의 반환값이 Promise로 감싸진다는 것 — 이것이 유일한 차이이자 모든 차이다. 호출자는 각 yield 지점에서 비동기 대기할 수 있고, 생산자는 await로 외부 리소스를 기다린 뒤 값을 내보낼 수 있다.

for await...of로 소비하기

async generator의 소비자 측 문법이 for await...of다.

async function main() {
	for await (const val of countWithDelay(5)) {
		console.log(val); // 0, 1, 2, 3, 4 (100ms 간격)
	}
}

내부적으로 이 루프는 다음과 동일하다:

const iter = countWithDelay(5);
let result = await iter.next();
while (!result.done) {
	console.log(result.value);
	result = await iter.next();
}

for await...of가 await iter.next()와 done 체크를 추상화해준다. 직접 next()를 호출할 일은 거의 없지만, 구조를 알아두면 디버깅이 쉬워진다.

실전: 페이지네이션 API 클라이언트

가장 흔한 사용처다. 커서 기반 페이지네이션 API를 async generator로 감싸면, 호출자는 무한 스크롤이든 전체 수집이든 동일한 인터페이스로 처리할 수 있다.

async function* fetchPages(baseUrl) {
	let cursor = null;

	do {
		const url = cursor
			? `${baseUrl}?cursor=${cursor}`
			: baseUrl;
		const res = await fetch(url);
		const data = await res.json();

		yield data.items;

		cursor = data.nextCursor;
	} while (cursor);
}

호출 측:

// 전체 수집
const allUsers = [];
for await (const page of fetchPages('/api/users')) {
	allUsers.push(...page);
}

// 조건부 중단 — 100명만 필요할 때
const first100 = [];
for await (const page of fetchPages('/api/users')) {
	first100.push(...page);
	if (first100.length >= 100) break;
}

break로 루프를 빠져나오면 generator의 return() 메서드가 자동 호출된다. 남은 페이지에 대한 불필요한 요청이 발생하지 않는다.

에러 전파와 정리

async generator 내부에서 발생한 에러는 for await...of 루프 바깥으로 전파된다.

async function* riskyStream() {
	yield 1;
	yield 2;
	throw new Error('connection lost');
	yield 3; // 도달하지 않음
}

try {
	for await (const val of riskyStream()) {
		console.log(val); // 1, 2
	}
} catch (e) {
	console.error(e.message); // "connection lost"
}

try...finally를 generator 내부에 두면 정리 로직을 보장할 수 있다:

async function* dbCursor(query) {
	const conn = await db.connect();
	try {
		const cursor = conn.query(query);
		while (await cursor.hasNext()) {
			yield await cursor.next();
		}
	} finally {
		// break, throw, return 어떤 경우든 실행
		await conn.close();
	}
}

소비자가 break로 빠져나가든, 에러가 발생하든, generator 내부의 finally 블록이 실행된다. 리소스 해제를 generator 안에 캡슐화할 수 있다는 뜻이다.

합성: generator를 연결하기

async generator는 다른 async generator를 yield*로 위임하거나, 파이프라인처럼 연결할 수 있다.

async function* fetchAllEndpoints() {
	yield* fetchPages('/api/users');
	yield* fetchPages('/api/orders');
	yield* fetchPages('/api/products');
}

// 변환 파이프라인
async function* map(source, fn) {
	for await (const item of source) {
		yield fn(item);
	}
}

async function* filter(source, pred) {
	for await (const item of source) {
		if (pred(item)) yield item;
	}
}

// 사용
const activeUsers = filter(
	map(fetchPages('/api/users'), page => page.flat()),
	user => user.active
);

for await (const user of activeUsers) {
	console.log(user.name);
}

각 단계가 독립적인 generator이므로, 필요한 변환만 조합하면 된다. RxJS의 pipe와 비슷한 패턴이지만 의존성 없이 언어 수준에서 동작한다.

ReadableStream, RxJS와의 경계

기준 Async Generator ReadableStream RxJS
모델 Pull (소비자가 요청) Pull + 내장 backpressure Push (생산자가 밀어냄)
의존성 없음 (ES2018) 브라우저/Node 내장 외부 라이브러리
취소 break / return() cancel() unsubscribe()
다중 소비자 불가 (단일 반복) tee() share(), multicast()
변환 직접 작성 또는 헬퍼 pipeThrough() 200+ 연산자

async generator를 쓸 때:

  • 단일 소비자가 순차적으로 데이터를 처리
  • 페이지네이션, DB 커서, 로그 파일 순회처럼 pull 모델이 자연스러운 경우
  • 외부 라이브러리 없이 가볍게 구현하고 싶을 때

다른 선택지가 나을 때:

  • 바이너리 스트림 → ReadableStream (청크 단위 backpressure 내장)
  • 이벤트 기반 실시간 데이터, 복잡한 합성(debounce, merge, switchMap) → RxJS
  • 여러 소비자가 같은 스트림을 구독 → RxJS share() 또는 ReadableStream.tee()

기억할 것

async generator는 비동기 데이터의 lazy evaluation이다. 모든 데이터를 메모리에 올리지 않고, 필요할 때 하나씩 비동기로 가져온다. yield가 생산자의 일시정지를, await가 비동기 대기를, for await...of가 소비자의 루프를 담당한다. 이 세 가지가 맞물리면, 1000페이지 API도 배열 순회만큼 단순해진다.
