🧠 핵심 개념

1. 제품 포트폴리오 분석

제품 포트폴리오 분석은 기업이 보유하고 있는 여러 제품군의 성과를 체계적으로 분석하여, 각 제품의 수익성, 비용, 할인율 등을 평가하는 과정을 의미합니다. 이를 통해 효율적인 의사결정과 전략 수립을 지원합니다.

🔍 이번 강의에서의 제품 포트폴리오 분석

이 강의에서 제품 포트폴리오 분석은 다음과 같은 요소를 기반으로 이루어집니다:

  1. 수익성 분석

    • 총 수익 (total_revenue): 제품 판매로 발생한 총 수익.
    • 순이익 (total_net_profit): 총 수익에서 비용과 할인을 뺀 값.
    • 순이익 비율 (net_profit_ratio): 순이익이 전체 수익에서 차지하는 비율로, 제품의 수익성을 나타냄.

  2. 비용 분석

    • 총 비용 (total_cost): 제품을 생산하는 데 들어간 총 비용.
    • 비용 비율 (cost_ratio): 비용이 전체 수익에서 차지하는 비율.

  3. 할인율 분석

    • 총 할인액 (total_discount): 할인으로 인해 감소된 금액.
    • 할인 비율 (discount_ratio): 할인이 전체 수익에서 차지하는 비율로, 마케팅이나 가격 전략 평가에 사용됨.

  4. 시각화를 통한 인사이트 도출

    • 산점도 그래프를 통해 다음 요소들을 한눈에 파악할 수 있습니다:
      • X축: total_revenue (총 수익)
      • Y축: net_profit_ratio (순이익 비율)
      • 크기: total_net_profit (순이익)
      • 색상: discount_ratio (할인율)

📝 제품 포트폴리오 분석의 목적

  1. 고수익 제품 식별

    • 수익성과 순이익 비율이 높은 제품을 파악하여 집중 투자.

  2. 비용 효율성 평가

    • 비용 비율이 높은 제품을 찾아 생산 비용 절감 전략을 수립.

  3. 할인 정책 최적화

    • 할인율이 지나치게 높아 수익성이 저하된 제품을 조정.

  4. 제품 전략 수립

    • 포트폴리오 전체를 평가하여 제품 개발, 마케팅, 가격 책정 등의 전략을 개선.

2. Attribution (기여도 분석)

정의:
고객이 구매나 전환을 결정하는 과정에서 각 마케팅 접점이 얼마나 기여했는지 평가하는 기법입니다.

주요 Attribution 모델

모델 설명 장점 단점
Last Click Attribution 구매 직전의 마지막 접점에 모든 기여를 할당 이해 및 실행이 쉽고 데이터 추적이 용이 초기 및 중간 접점의 기여를 무시할 수 있음
First Click Attribution 첫 번째 접점에 모든 기여를 할당 초기 고객 유입과 브랜드 인지도 평가에 유리 후속 접점의 중요성을 간과할 수 있음
Multi-Touch Attribution 전체 고객 여정에 걸쳐 여러 접점에 기여를 분배 전체 구매 과정을 반영하고 다양한 접점의 기여를 분석 가능 구현이 복잡하고 가중치 설정이 어려울 수 있음

Pandas를 활용한 예제

  • 그룹별 최대값 인덱스 찾기: 
    df.groupby('group')['value'].idxmax()
df.loc[df.groupby('group')['value'].idxmax()]

3. Conversion Window (전환 기간)

정의:
고객이 제품이나 서비스를 처음 접한 후 전환(구매, 가입 등)을 완료하기까지 걸리는 시간을 측정하는 기법입니다.

예시:

  1. 온라인 쇼핑: 제품을 본 순간부터 구매까지의 시간
  2. 앱 사용: 앱 다운로드 후 첫 사용까지의 시간
  3. 서비스 가입: 서비스 가입 후 실제 사용 시작까지의 시간

완전성과 적시성

Conversion Window 설명 특징
짧은 Conversion Window 빠른 데이터 분석 가능하지만 전체 행동 패턴을 반영하지 못할 수 있음 적시성⬆, 완전성⬇
긴 Conversion Window 데이터 분석에 시간이 걸리지만 더 많은 행동 패턴을 반영 적시성⬇, 완전성⬆

Pandas를 활용한 예제

  • 전환 기간 계산: 
    df['pay_at'] = pd.to_datetime(df['pay_at'])
df['apply_at'] = pd.to_datetime(df['apply_at'])
df['conversion_time'] = (df['pay_at'] - df['apply_at']).dt.total_seconds() / (3600 * 24)
  • 히스토그램 시각화: 
    import plotly.express as px

fig = px.histogram(df, x='conversion_time')
fig.show()

4. 통합 데이터 분석과 마케팅 전략 예제

📌 1. 데이터 불러오기 및 전처리

  1. CSV 파일 불러오기

    • 광고, 주문, 구매, 제품, 방문 데이터를 각각 불러옵니다.
    ad = pd.read_csv('eda_ad.csv')
order = pd.read_csv('eda_order.csv')
purchase = pd.read_csv('eda_purchase.csv')
product = pd.read_csv('eda_product.csv')
visit = pd.read_csv('eda_visit.csv')

  2. 날짜 데이터 타입 변환

    order['order_date'] = pd.to_datetime(order.order_date)
ad['date'] = pd.to_datetime(ad.date)
purchase['timestamp'] = pd.to_datetime(purchase.timestamp)
visit['timestamp'] = pd.to_datetime(visit.timestamp)

📌 2. 데이터 결합 및 전처리

  1. 주문 데이터와 제품 데이터 결합

    order_mg = pd.merge(order, product, how='left', left_on='product_id', right_on='id')

  2. 새로운 컬럼 생성

    • 매출, 할인 금액, 비용, 이익 계산
    order_mg['total_revenue'] = order_mg.price * order_mg.quantity
order_mg['total_disc_amt'] = order_mg.total_revenue * order_mg.discount_ratio
order_mg['total_cost'] = order_mg.cost * order_mg.quantity
order_mg['total_profit'] = order_mg.total_revenue - order_mg.total_cost - order_mg.total_disc_amt

📌 3. Attribution 분석

  1. Last Click Attribution

    • 고객의 마지막 접점을 기준으로 기여도를 할당합니다.
    idx = log.groupby(by='order_id')['td_day'].idxmin()
df_l = log.loc[idx]

  2. Attribution Window 적용 (70일)

    • 방문 후 70일 이내의 전환만 유효하게 처리합니다.
    attr_w = log_mg.td_day <= 70
log = log_mg[same_user & time_filter & attr_w]

📌 4. 광고 효율성 분석

  1. ROAS (Return on Advertising Spend)

    ad_agg = ad.groupby(by=['campaign', 'source'])[['spending', 'clicks']].sum().reset_index()
df['roas'] = df.realized_revenue / df.spending

  2. CPC (Cost Per Click) 계산

    ad['cpc'] = ad.spending / ad.clicks

📌 5. 제품 분석

  1. 제품별 전환 소요 시간 계산

    order_mg['td_day'] = (order_mg.pay_date - order_mg.order_date).dt.total_seconds() / (3600 * 24)

  2. 전환율 계산 (3일, 9일 기준)

    order_mg['conv_3'] = np.where(order_mg.td_day <= 3, 1, None)
order_mg['conv_9'] = np.where(order_mg.td_day <= 9, 1, None)

📌 6. 매출 트렌드 및 상관관계 분석

  1. 카테고리별 매출 트렌드

    od_agg_d = order_mg.groupby(by=['order_date', 'category', 'product_id'])['total_revenue'].sum().reset_index()

  2. 제품 간 매출 상관관계 분석

    df_m = od_agg_d.pivot_table(index='order_date', columns='product_id', values='total_revenue')
corr = df_m.corr()

📌 7. 고객 분석

  1. 고객별 지표 계산

    order_mg.groupby(by='user_id').agg(
    total_spending=('total_revenue', 'sum'),
    order_cnt=('order_id', 'count'),
    dist_prod_cnt=('product_id', 'nunique'),
    dist_cate_cnt=('category', 'nunique')
).reset_index().describe()
