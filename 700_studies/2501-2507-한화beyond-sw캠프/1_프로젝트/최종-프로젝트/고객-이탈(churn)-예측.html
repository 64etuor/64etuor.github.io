고객 이탈(Churn) 예측

고객 이탈(Churn) 예측 ― 태그 기반 자동 리텐션 마케팅 구축 설계

0. 개요

고객 이탈을 줄이고 재구매를 높이려면 데이터 수집 → 이탈 예측 → CRM 태깅 → 맞춤 마케팅 → 성과 측정이 끊김 없이 이어지는 ‘폐루프(Closed Loop)’ 컨셉

1. 데이터 파이프라인

데이터 범주 주요 컬럼 예시 주요 지표(Feature)
방문·예약 최종 방문일, 방문 주기, 예약 취소·노쇼 횟수, 예약→방문 전환율 Recency / Frequency / Momentum
매출·결제 누적 구매액, 3·6·12 개월 추세, 서비스·상품 비중 지출 감소율, 객단가 하락
멤버십 사용 선불·정기권 잔액, 소진 속도, 포인트 적립률 잔액 소진 중단
메시지 반응 SMS·카카오톡 오픈·클릭·응답 Engagement 하락
리뷰·만족도 리뷰 점수, NPS, CS 티켓 수 불만·무응답 패턴

ETL 예시: 원천 DB + 로그 → cron → Python

2. 이탈(Churn) 예측 모델

단계 실무 팁
정의 예: “최근 180일간 방문·구매 0회 && 이전 180일 ≥ 1회”
피처 엔지니어링 주 단위 RFM, 지출·리뷰·잔액 추세, 메시지 반응률
알고리즘 XGBoost / LightGBM → SHAP 해석불균형 심할 때 CatBoost·Vertex AI AutoML
평가 지표 ROC-AUC ≥ 0.80, PR-AUC·Recall(이탈 클래스) 함께 모니터링
배포 FastAPI 컨테이너(도커) → Airflow/Cloud Scheduler로 배치 예측

3. CRM 태깅 & 세그먼트

위험 레벨 태그 예시 적용 예
High (p > 0.8) churn_high 리터치 쿠폰, 전담 상담
Mid (0.5–0.8) churn_mid 신상품 체험, 포인트 2배
Low (p < 0.5) 미태깅 일반 로열티 플로우

예측 결과(customer_id, churn_prob, risk_level)를 REST API로 PATCH → 직원 화면·자동 발송 규칙 즉시 반영.

4. 맞춤 마케팅 자동화

위험군 메시지·오퍼 예시 자동화
High “고객 감사 이벤트 – 20 % 할인 쿠폰” 카카오 알림톡 + 만료 7일 전 재알림
Mid 신상품 체험 초대, 포인트 2배 SMS 그룹 발송, 재방문 시 리뷰 요청
Low 생일·VIP 감사 쿠폰 기존 로열티 시나리오 지속

A/B/C 실험

할인율·기간·카피를 교차 실험 → Python 서버에 실적 적재 → Lift 분석으로 모델·캠페인을 동시에 개선.

5. 6주 실행 로드맵

주차 주요 목표
1주 ETL 모델 설계, 이탈 정의 확정
2주 피처 집계·샘플링, 베이스라인 모델
3주 하이퍼튜닝, SHAP 리포트 공유
4주 예측 API + 스케줄러, 태깅 연동
5주 High/Mid 캠페인 콘텐츠·쿠폰 확정, A/B 설계
6주 캠페인 더미데이터 주입, Lift 대시보드 구축

6. 기술 스택 (Java + Vue 기반)

계층 도구
데이터 Airbyte · Python
모델 Python + XGBoost / CatBoost, Vertex AI AutoML
서빙 FastAPI → Cloud Run
오케스트레이션 Airflow / Cloud Scheduler
프런트 대시보드 Vue + Chart.js
메시징 SMS/카카오 API
캠페인 빌더 Java Rule Engine + BPMN(Camunda 등)

7. 실제 적용 사례

산업·기업 적용 범위 성과
Beauty e-commerce allbeauty 90일 이탈 위험 예측 + 쿠폰 재구매 +12 %, e-mail 비용 −28 %
Taco Bell / KFC AI 개인화 오퍼 “눈에 띄는 churn 감소”
구독 건강식품 Cured Nutrition 해지 직전 업셀 이탈 −16 %, 반복 매출 +24 %
Telco (Vertex AI 사례) 일일 수백만 예측 ROC-AUC 0.87, 3주 MVP
카드사 연구 CatBoost + SHAP 해지 문의 고객 25 % 유지

8. 도입 장애 & 해결책

장애 원인 사례 해결책
ID 불일치·사일로 예약·POS·메시지 DB 구조 상이 단일 customer_id, DB 변환 계층
클래스 불균형 이탈 비중 < 10 % Focal Loss / SMOTE, CatBoost
태깅 API 지연 대량 PATCH → rate limit Pub/Sub Batch(1 k)
메시지 실패율 카카오 템플릿 오류 사전 검수 + SMS 대체
모델 드리프트 계절/프로모션 변화 월간 재학습, Drift 모니터링

9. 과제

  1. 데이터 품질이 알고리즘보다 중요 : 데이터를 어떻게 구함???ㅋㅋ
  2. 예측으로 끝나지 말고, 태그·캠페인까지 폐루프를 완성해야 함.
  3. A/B 실험과 Lift 대시보드로 매출·재방문 증대를 지속 측정하면서 모델과 오퍼를 주기적으로 업데이트.
