11-2강. JAVA 문법(상속, 다형성, 추상 클래스, 인터페이스 part.2) |

이 강의는 자바의 객체지향 프로그래밍에서 중요한 개념인 상속, 다형성, 추상 클래스, 인터페이스를 심도 있게 다룹니다. 상속을 통해 클래스 간의 관계를 확립하고, 다형성을 사용하여 다양한 객체를 통일된 방식으로 사용할 수 있는 방법을 설명합니다. 강의 내용은 타입 변환, 메서드 재정의, 그리고 추상 클래스 및 인터페이스의 활용에 중점을 두고 있습니다. 이러한 개념을 통해 프로그래밍의 유연성과 재사용성을 높일 수 있습니다. 자바의 객체지향 원리를 이해하고 발전시킬 수 있는 풍부한 기회가 됩니다.

1. 📘 자바의 자동 타입 변환과 다양성 구현

  • 자바에서 자동 타입 변환은 프로그램 실행 도중 부모 타입의 변수에 자식 객체를 넣을 때 발생한다.
  • 상속 계층에서 상위 클래스 타입일 경우, 직접적인 관계가 없어도 자동 타입 변환이 가능하다.
  • 자식 클래스의 메서드가 재정의된 경우, 부모 클래스 참조 변수를 통해 자식 메서드를 호출할 수 있다.
  • 필드 타입을 부모 타입으로 선언하고 다양한 자식 객체를 저장하여 사용하는 방식으로 다양성을 구현할 수 있다.
  • 예제를 통해, 부모 클래스의 타이어와 이를 상속받은 한국타이어, 금호타이어를 활용하여 객체 상속과 메서드 오버라이딩을 구현하는 방법을 설명한다.

1.1. 클래스의 타입 변환과 다양성

  • 클래스에서도 타입 변환이 존재하며, 이는 기본 타입과 유사하다.
  • 다양성은 사용 방법은 동일하지만, 다양한 객체를 활용해 여러 실행 결과를 도출하는 성질이다.
  • 메소드 재정의와 타입 변환을 통해 다양성을 구현할 수 있다.
  • 자동 타입 변환은 프로그램 실행 중 자동으로 타입이 변경되는 상황을 의미한다.
  • 부모 타입의 변수에 자식 타입의 객체를 삽입하는 것이 자동 타입 변환의 한 예시이다.
  • 애니멀 상위 클래스와 이를 상속받은 캐시 클래스가 있으며, 부모 클래스의 참조 변수에 자식 클래스의 객체를 넣을 수 있는 구조이다.

1.2. 상속과 자동 타입 변환

  • 상속 계층에서 상위 타입인 경우에는 자동 타입 변환이 가능하여, 부모 클래스의 참조 변수에 서브 클래스 객체를 담을 수 있다.
  • 예를 들어, 클래스 a 아래에 b와 c가 있고, b를 상속한 d와 c를 상속한 클래스가 존재하므로, d 객체는 a 참조 변수에 담길 수 있다.
  • 그러나 c 클래스에서 생성된 객체는 a 참조 변수에 담을 수 없으며, b 참조 변수에는 담길 수 없는 이유는 직접적인 관계가 없기 때문이다.
  • 자동 타입 변환 후에는 부모 클래스의 필드 및 메소드에만 접근할 수 있으며, 재정의된 메소드는 자식 클래스의 메소드가 호출된다.
  • 부모 클래스의 메소드 규격은 그대로 유지되기 때문에, 호출이 가능하다는 점이 중요하다.

1.3. 자식 클래스의 메소드 재정의와 호출

  • 자식 클래스는 부모 클래스로부터 상속을 받아 메소드를 재정의할 수 있다.
  • 자식 객체를 생성하고 부모의 참조변수에 담으면, 부모 클래스의 메소드를 호출할 수 있다.
  • 부모의 참조변수를 통해 메소드 호출 시, 부모 클래스의 메소드가 호출되며, 자식 클래스의 재정의된 메소드도 호출 가능하다.
  • 하지만 부모 클래스의 참조변수를 사용할 경우, 자식 클래스에서 정의된 메소드는 호출할 수 없으며, 이로 인해 에러가 발생한다.

1.4. 자식 객체를 활용한 필드 사용의 다양성

  • 필드 타입을 부모 클래스로 선언하면, 다양한 자식 객체를 저장하고 사용할 수 있다.
  • 예를 들어, 타이어라는 부모 클래스를 통해 자동차 클래스의 필드로 한국타이어 또는 금호타이어 객체를 선택할 수 있다.
  • 4개의 타이어 객체를 자동차에 삽입할 수 있어 필드를 통해 다양한 조합이 가능하다.
  • 타이어 클래스와 자식 클래스인 한국타이어, 금호타이어를 생성하고, 메소드를 통해 맥스 회전수와 누적 회전수를 관리한다.
  • 각 자식 클래스에서 오버라이드된 메소드를 통해 특정 동작이나 메시지를 제공할 수 있다.

1.5. 메소드 구현 및 반환값 설정

  • 클래스 안에 run 메소드를 작성하여 자동차가 달린다는 출력을 한다.
  • 프런트 레프트 타이어가 롤 상태일 경우, 스톱 메소드를 호출하고 1 값을 반환한다.
  • 백 레프트 타이어와 백 라이트 타이어에 대해서도 각각 롤 상태를 체크하고, 상태에 따라 각각 2, 3, 4 값을 반환한다.
  • 만약 모든 조건에 해당되지 않을 경우, 0 값을 반환하도록 설정한다.
  • 스톱 메소드는 자동차가 멈춘다는 출력을 포함하고 있으며, 별도로 구현해야 한다.
  • 마지막으로, example이라는 실행 클래스를 생성하고 메인 메소드를 선언한다.

2. 🚗 자동차 객체의 타이어 교체 및 타입 변환

  • Car 참조 변수를 선언하고 new Car()를 통해 자동차 객체를 생성한 후, 반복문을 사용하여 타이어 교체 작업을 진행한다.
  • 스위치 문을 사용하여 위치에 따라 다양한 타이어 교체 작업을 수행하며, 각 타이어의 수명을 설정한다.
  • 자동차가 달리면서 타이어가 펑크나면 이를 교체하고, 각 타이어의 수명이 여전히 나가기 전에 교체가 이루어짐.
  • 다양성의 개념을 도입하여 부모 클래스의 참조 변수로 여러 자식 클래스의 객체를 사용할 수 있음을 보여준다.
  • 강제 타입 변환을 통해 부모 타입 변수에 자식 클래스를 대입하고, 이를 확인하기 위해 instanceof 연산자를 사용할 수 있다.

2.1. 자동차 객체 생성 및 스위치 문 활용

  • Car 참조 변수를 생성하고 new Car()로 자동차 객체를 만든다.
  • i 값이 1부터 5까지 실행되도록 포문을 사용할 예정이다.
  • INT problemLocation 변수를 선언하고, 이 값을 car.getLocation()을 통해 가져온다.
  • 스위치 문을 사용하여, 프로블럼 로케이션에 따라 다양한 케이스를 나누어 적용할 예정이다.
  • 1번부터 4번까지의 값을 통해 프론트와 백에 대한 다양한 값을 생성한다.

2.2. ️ 타이어 교체 코드 구현 과정

  • 앞 왼쪽 타이어는 한국 타이어로 교체되며, 수명은 15로 설정된다.
  • 앞 오른쪽 타이어는 런타이 라이트 타이어로 교체되고, 수명은 13으로 설정된다.
  • 왼쪽 뒷 타이어도 한국 타이어로 교체되며, 수명은 14로 지정된다.
  • 뒤 오른쪽 타이어는 금호 타이어로 교체되며, 해당 타이어의 수명은 적당히 설정된다.
  • 브레이크가 누락된 간략한 설명이 있음을 언급하며, 이후 실행 결과를 확인할 예정이다.

2.3. 자동차 타이어 수명과 교체 과정

  • 자동차는 처음에 총 타이어 수명이 주어지며, 그 중에서 펑크가 나기 쉬운 타이어가 있다.
  • 오른쪽 타이어가 펑크가 나서 자동차는 멈추고, 금호타이어로 교체된다.
  • 금호타이어를 교체한 후, 타이어의 수명은 13으로 줄어들지만, 사용 후 하나씩 소모된다.
  • 다음으로 뒤쪽 타이어가 펑크가 나게 되고, 교체 과정이 진행된다.
  • 교체 후 현재 타이어의 수명은 14개로 늘어나지만, 또 다른 타이어에서 펑크가 발생하기 전에 프로그램이 종료된다.

2.4. 객체 지향 프로그래밍의 다양성 이해

  • 부모 클래스의 참조변수를 통해 다양한 자식 클래스의 객체를 사용할 수 있어, 이로 인해 다양성이 생긴다.
  • 자식 클래스를 부모 클래스의 타입으로 교체할 수 있으며, 이로 인해 매개변수의 다양성도 형성될 수 있다.
  • 강제 타입 변환이 필요한 경우, 자식 클래스의 객체가 부모 타입에 있을 때 다시 원래 타입으로 변환하여 사용할 수 있다.
  • 객체의 타입을 확인하기 위해 인스턴스 오브 연산자를 사용하여 타입이 일치하는지 검증할 수 있으며, 이조건이 맞으면 강제 타입 변환이 가능하다.
  • 클래스 타입 변환 시 자동 타입 변환과 강제 캐스팅을 구분하며, 자식 객체는 부모 변수를 통해 자동으로 대입될 수 있다.

2.5. 추상 클래스와 상속의 개념

  • 추상화는 여러 클래스의 공통된 특성을 뽑아내어 선언하는 과정이며, 추상 클래스는 이러한 역할을 맡는다.
  • 여러 클래스에서 공통된 특징을 모아 추상 클래스를 만들 수 있으며, 예를 들어 "애니멀"이라는 클래스는 다양한 동물 클래스의 일반적인 특성을 포함할 수 있다.
  • 추상 클래스는 자식 클래스에서 상속받을 수 있는 규격을 정의하여, 자식 클래스들이 공통적으로 필요한 사항들을 포함하게 한다.
  • 예를 들어, 기본적인 추상 클래스로서 "폰"이 있을 경우, "텔레폰"과 "스마트폰" 클래스는 이 폰의 특징을 모두 갖출 수 있다.
  • 자동차의 타이어 규격처럼, 추상 클래스는 설계 기준을 제시하고, 이 기준에 맞춰 다양한 구현체들을 만들 수 있도록 한다.

3. 📱 추상 클래스와 메소드의 이해

  • 추상 클래스는 직접 객체로 생성할 수 없으며, 자식 클래스에서만 생성이 가능하다.
  • 추상 메소드는 메소드 선언만 하고 구체적인 실행 내용은 자식 클래스에서 반드시 구현해야 하며, 이를 통해 강제할 수 있다.
  • 예를 들어, 애니멀 클래스를 앱스트랙트 클래스로 만들고, 사운드라는 추상 메소드를 선언하여 자식 클래스에서 오버라이드하도록 할 수 있다.
  • 해시와 로그라는 두 개의 자식 클래스를 애니멀로부터 상속받아 각각 개와 고양이에 대한 소리를 정의할 수 있다.
  • 메인 메소드를 통해 다양한 동물의 소리를 호출하며, 자연스러운 타입 변환을 통해 동물 객체에 맞는 소리를 출력할 수 있다.

4. 📚 인터페이스의 정의와 활용

  • 추상 클래스는 클래스들의 공통적인 필드와 메소드를 정의하며, 자식 클래스에서 반드시 재정의해야 하는 추상 메소드를 포함한다.
  • 인터페이스는 객체의 사용 방법을 정의하는 타입이며, 이를 통해 다양한 객체를 동일한 방법으로 사용할 수 있다.
  • 인터페이스에 선언된 메소드는 객체의 메소드와 연결되어 호출되며, 객체가 변경되더라도 인터페이스에 정의된 메소드를 통해 다용성을 유지할 수 있다.
  • 인터페이스는 객체로 생성할 수 없고, 추상 클래스와 동일하게 생성자를 갖지 않으며, 상수만 선언할 수 있다고 한다.
  • 인터페이스에서 상수를 정의할 때는 public static final을 생략할 수 있으며, 상수 이름은 대문자로 작성하는 것이 원칙이다.

4.1. ️ 추상 클래스와 인터페이스의 개념 및 의의

  • 추상 메소드는 자식 클래스에서 반드시 재정의해야 하는 메소드로, 추상 클래스 내에만 선언할 수 있다.
  • 추상 클래스는 객체로 만들 수 없지만, 참조 변수를 통해 다양성을 확보할 수 있다.
  • 인터페이스는 객체에 대한 사용 방법을 정의한 타입으로, 다양한 객체를 동일한 방법으로 사용할 수 있게 해 준다.
  • 인터페이스의 메소드를 호출하면 객체의 해당 메소드가 실행되며, 개발 코드를 수정하지 않고도 객체 교환이 가능하다.
  • 인터페이스에 정의된 메소드를 구현한 객체들은 메소드 호출에 지장이 없으므로, 다양성을 가능하게 한다.

4.2. ️ 인터페이스의 특징과 사용법

  • 인터페이스는 객체로 생성할 수 없으며, 추상 클래스와 동일하게 생성자를 갖지 않는다. 그러나 추상 클래스는 생성자를 가질 수 있다.
  • 인터페이스 내에서는 상수 필드만 선언 가능하고, 데이터 저장을 위한 인스턴스나 정적 필드는 불가능하다.
  • 상수는 기본적으로 public static final로 정의되며, 이 표현을 생략할 수도 있지만 인터페이스 내에서 스태틱 파이널로 간주된다.
  • 상수 이름은 대문자로 작성하는 것이 관례이며 , 추상 메소드를 선언할 수 있으나 내용은 작성하지 않는다.
  • 인터페이스에서 구현된 메소드를 정의할 경우 에러가 발생하며, 오직 추상 메소드만을 사용할 수 있다.

4.3. ️ 인터페이스 구현을 통한 OOP 개념 정리

  • 인터페이스를 구현하기 위한 클래스로 오디오 클래스를 생성하여, 리모트 컨트롤 인터페이스를 구현하도록 한다.
  • 오디오 클래스에서는 프라이빗 INT 볼륨 필드를 추가하고, 추상 메소드를 오버라이드하여 기능을 구현해야 한다.
  • 볼륨 설정 시, 리모트 컨트롤의 최대 및 최소 볼륨 값을 확인하여 조정하는 로직이 포함된다.
  • 두 번째 클래스인 텔레비전 역시 리모트 컨트롤 인터페이스를 구현하며, 많은 코드가 오디오 클래스와 유사하기 때문에 코드의 재사용성을 보여준다.
  • 인터페이스 변수를 선언하고, 구체적인 객체(오디오, 텔레비전)로 대입하여 다양한 타입을 처리할 수 있는 다형성의 개념을 실현할 수 있다.

4.4. 다중 인터페이스 구현 및 클래스 예제

  • 다중 인터페이스를 구현할 수 있어, 여러 가지 인터페이스를 정의할 수 있다.
  • 예제로 '이그잼 공'이라는 인터페이스 내에 리모트 컨트롤을 구현하여 설명할 예정이다.
  • 리모트 컨트롤은 인터페이스로 정의되고, 이를 복사하여 다른 클래스를 구현한다.
  • '서치 어플' 클래스는 두 개의 인터페이스를 구현하며, 추상 메소드를 포함하고 있다.
  • 인터페이스는 객체 사용 방법을 추상적으로 정의하고, 상수 필드와 추상 메소드를 가질 수 있다.

5. 🚀 인터페이스와 다형성 구현

  • 인터페이스를 사용할 때 필드, 매개변수, 로컬 변수로 활용할 수 있으며, 메소드 재정의와 타입 변환이 가능하다는 점이 강조된다.
  • 인터페이스를 통해 다양성을 구현할 수 있으며, 구현 객체를 교체해 가며 사용할 수 있다.
  • 구형 객체를 인터페이스 타입으로 자동 변환하면, 해당 인터페이스에 선언된 메소드만 사용 가능하고, 구형 클래스의 필드나 메소드를 사용할 경우 강제 타입 변환이 필요하다.
  • 인터페이스는 다중 상속이 가능하며, 이를 통해 여러 인터페이스를 상속받는 새로운 인터페이스를 정의할 수 있다.
  • 다형성과 추상화를 통해 상위 인터페이스를 참조변수로 활용할 수 있으며, 코드의 영향 없이 구현체를 변경할 수 있는 장점이 있다.

6. 📚 복습의 중요성

  • 이번 강의 내용이 많이 어렵고 광범위하니 시간을 가지고 천천히 살펴볼 것을 권장한다.
  • 책에 있는 연습문제를 통해 자기 자신의 실력을 점검해 보기를 추천한다.
  • 따라서, 더욱 깊이 있는 학습을 위해 복습하는 시간이 필요하다고 강조된다.
  • 강의를 마치며 감사의 인사를 전한다.

Key term

자바

자바는 프로그래밍 언어 중 하나로, 객체지향 프로그래밍을 지원합니다. 쉽게 말해, 프로그램을 만드는 데 필요한 다양한 기능과 구조를 갖춘 도구입니다. 개발자들이 자바를 사용해 프로그램을 작성하면, 더 유연하고 효율적인 코드를 작성할 수 있습니다.

상속

상속은 한 클래스가 다른 클래스의 특성을 물려받는 것입니다. 예를 들어, 부모가 자식에게 유전자를 주듯, 클래스도 다른 클래스를 통해 속성이나 기능을 받을 수 있습니다. 이를 통해 코드의 재사용성이 높아지고, 프로그램을 효율적으로 만들 수 있습니다.

다형성

다형성은 하나의 메소드가 다양한 방식으로 동작할 수 있는 성질입니다. 쉽게 비유하자면, 여러 종류의 동물들이 '소리 내기'라는 같은 행동을 하지만, 각각 다르게 소리를 낸다고 할 수 있습니다. 프로그래밍에서는 이 덕분에 같은 코드로 서로 다른 객체를 다룰 수 있습니다.

타입 변환

타입 변환은 자료형을 변환하는 과정을 말합니다. 자바에서는 부모 클래스 타입의 변수를 자식 클래스 객체로 쉽게 변환할 수 있는 자동 타입 변환이 있습니다. 반대로, 자식 클래스의 객체로부터 부모 클래스의 메소드나 필드만을 사용할 수 있을 때는 강제 타입 변환이 필요합니다.

메서드 재정의

메서드 재정의는 상속받은 메서드를 자식 클래스에서 다시 정의하는 것입니다. 이를 통해 자식 클래스가 부모 클래스의 기능을 변경하거나 보완할 수 있습니다. 비유하자면, 부모가 자전거를 타는 방법을 알려줬지만, 자식이 오히려 전기 자전거를 만들어 더 빠르고 효율적으로 탈 수 있도록 하는 것입니다.

추상 클래스

추상 클래스는 구체적인 객체를 만들 수 없는 클래스입니다. 이 클래스는 자식 클래스에게 공통적인 조항이나 규칙을 제공하여, 자식 클래스들이 이 규칙을 따르도록 강요합니다. 마치 학교의 기본 규칙을 정해놓고 학생들이 이를 따라야 하는 것과 비슷합니다.

인터페이스

인터페이스는 객체가 가져야 할 기능을 정의한 것입니다. 즉, 특정한 메소드를 반드시 구현하겠다는 계약을 의미합니다. 비유하자면, 게임에 참여하는 모든 캐릭터가 '말하기'라는 기능을 가지지만 각 캐릭터는 자신만의 방식으로 이야기하는 것처럼 말입니다.

객체지향 프로그래밍

객체지향 프로그래밍은 프로그램을 객체라는 독립적인 단위로 나누어 개발하는 방법입니다. 즉, 실제 세계에 존재하는 사물이나 개념처럼 프로그램을 구성하여 더욱 직관적으로 개발할 수 있게 돕는 방법입니다. 이는 코드의 관리가 용이해지는 장점이 있습니다.

자동 타입 변환

자동 타입 변환은 자바에서 내부적으로 부모 타입의 변수에 자식 객체를 할당할 때 발생합니다. 마치 어린이가 성인이 되었을 때 기본적인 권리를 갖게 되는 것처럼, 자식은 부모의 속성을 자동으로 갖게 되는 것입니다.

강제 타입 변환

강제 타입 변환은 자식 객체를 부모 변수에 담고 올바른 메소드를 사용하기 위해 다시 자식 타입으로 변환해야 하는 경우입니다. 이건 마치 작은 열쇠로 큰 자물쇠를 열기 위해 다시 작고 적합한 열쇠를 만드는 과정을 비유할 수 있습니다.

다양성

다양성은 객체가 가진 여러 모습이나 행동을 갖춘 성질입니다. 예를 들어, 동물의 소리를 낼 때, 개는 '멍멍', 고양이는 '야옹'처럼, 서로 다른 결과를 만들어내는 것입니다. 그래서 같은 행동을 하면서도 각 객체에 따라 다른 결과를 유도할 수 있습니다.

코드 재사용

코드 재사용은 한 번 작성한 코드를 여러 곳에서 다시 사용할 수 있는 것을 말합니다. 이는 개발 시간과 노력을 절약해주며, 안정성을 높여주는 장점이 있습니다. 예를 들면, 샌드위치 레시피를 바탕으로 다양한 종류의 샌드위치를 쉽게 만들 수 있는 것과 같습니다.

상속 계층

상속 계층은 클래스 간의 상속 관계를 나타내는 구조입니다. 이 구조는 마치 가족의 계보처럼 부모에서 자식으로 이어지는 형태를 보여줍니다. 이를 통해 클래스들 간의 관계를 명확하게 할 수 있습니다.

추상 메소드

추상 메소드는 실행 내용 없이 메소드의 이름과 매개변수만 정의한 것입니다. 자식 클래스가 이 메소드를 반드시 구현하도록 요구하는 방법으로, 마치 수업에서 주의사항만 돌려주는 것처럼 구체적인 내용을 자식에게 맡기는 것입니다.

클래스

클래스는 객체를 생성하기 위한 설계도와 같습니다. 이 설계도에 따라 실제 물건을 만들 수 있는데, 예를 들어 자동차 클래스를 설계하면 이를 기반으로 다양한 자동차 객체를 만들 수 있습니다.

인터페이스 구현

인터페이스 구현은 클래스가 인터페이스에 정의된 메소드들을 실제로 작동하도록 만드는 행위입니다. 하여, 마치 약속한 계약을 따르는 것처럼 클래스가 명세된 규칙을 준수하도록하는 것입니다.

객체 생성

객체 생성은 클래스를 기반으로 실제로 사용할 수 있는 인스턴스를 만드는 과정입니다. 이 단계는 설계도를 바탕으로 실제 건물이 세워지는 순간처럼 이해할 수 있습니다.

메소드 호출

메소드 호출은 특정 객체의 기능을 사용하는 것입니다. 프로그램 진행 시 특정 행동이나 데이터를 요청하는 것이므로, 마치 전화로 친구에게 구체적인 일을 부탁하는 것과 유사합니다.

다중 상속

다중 상속은 한 클래스가 여러 부모 클래스를 상속받는 것을 말합니다. 하지만 자바에서는 클래스 상속은 하나만 가능하지만 인터페이스는 여러 개를 상속받을 수 있습니다. 비유하자면, 여러 취미를 가지며 여러 스승에게 배운다는 것과 같습니다.

필드

필드는 클래스 내부에 있는 변수로, 클래스가 가진 속성을 정의합니다. 사람의 경우, 이름, 나이, 성별 등이 필드가 될 수 있습니다. 이는 객체의 상태를 표현해줍니다.

매개변수

매개변수는 메소드에 정보를 전달하는 변수입니다. 예를 들어, 요리를 할 때 재료를 그 요리 메소드에 전달하는 것과 같습니다. 매개변수를 사용하여 메소드를 더 유연하게 사용할 수 있습니다.

로컬 변수

로컬 변수는 메소드 내에서 선언된 변수입니다. 마치 한 방 안에서만 사용되는 도구처럼, 그 방 안에서만 유효하고 사용됩니다.

인스턴스 오브

인스턴스 오브는 객체의 실제 타입을 확인하는 연산자입니다. 간단히 말해, '이 객체가 이 타입인가?'를 확인하는 질문을 던지는 것과 같습니다.

구현체

구현체는 인터페이스나 추상 클래스를 실제로 사용하기 위해 만든 클래스를 의미합니다. 이는 마치 이론적인 수업을 바탕으로 실제로 전시물을 만드는 전시체와 같습니다.
