4강. JAVA 문법(연산자)

이번 영상에서는 자바의 연산자에 대해 자세히 배우게 됩니다. 기본 타입과 비교 연산자, 문자열 비교, 산술 연산자, 대입 연산자 등 다양한 연산자들의 활용법과 기본적인 사용 방법을 설명합니다. 연산자의 우선순위와 이를 효율적으로 사용하는 방법을 사례와 함께 쉽게 풀어내어, 프로그래밍에 필요한 필수 지식을 제공합니다. 이 과정에서 연산자가 어떻게 작동하는지 이해하고, 실제 코드 작성에 적용할 수 있는 능력을 키우게 됩니다. 효과적인 프로그래밍을 위한 필수 요소 중 하나인 연산자에 대한 깊은 이해를 돕는 영상입니다.

1. ⚙️ 연산자의 이해

  • 연산자는 기본 타입의 값 비교 시 등호 두 개를 사용하며, 이를 통해 bool 타입 값인 true 또는 false를 반환한다.
  • 문자열 비교에서는 equals 메소드를 사용하고, 대소문자를 엄격히 구분하여 비교 결과를 반환한다.
  • 연산자는 데이터 처리 및 결과 산출에서 중요한 역할을 하며, 이를 통해 프로그램의 오퍼레이션이 형성된다.

2. 🧮 자바 연산자 및 우선순위 개념

  • 자바의 연산자는 연산에 사용되는 기호이며, 더하기, 빼기, 곱하기, 나누기, 나머지 등을 포함한다.
  • 피연산자는 연산자와 함께 연산되는 값이며, 연산식은 연산자와 피연산자를 사용하여 기술된 전체이다.
  • 연산자의 종류에는 산술, 비교, 논리, 대입 연산자가 있으며, 각 연산자는 서로 다른 우선순위를 가진다.
  • 우선순위에 따라 연산자는 보통 산술 → 비교 → 논리 → 대입 순으로 실행되며, 같은 우선순위의 연산자는 왼쪽에서 오른쪽으로 수행된다.
  • 소괄호를 사용하면 연산식의 우선 처리를 조절할 수 있으며, 이로써 사용자가 원하는 순서로 연산할 수 있다.

3. ✨ JAVA 연산자와 부호 연산자

  • 소괄호를 사용하여 연산 순서를 조절할 수 있으며, 덧셈을 먼저 수행하도록 지정할 수 있다.
  • 부호연산자에서 마이너스는 부호를 바꾸고, 더하기는 값에 변화를 주지 않는다.
  • 바이트 타입의 결과는 INT 타입으로 형변환되며, 이 경우 바이트에 직접 담을 수 없다.
  • 증감 연산자는 변수의 값을 1 증가시키거나 감소시키며, 위치에 따라 결과값이 달라질 수 있다.
  • 증감 연산자를 단독으로 쓸 때는 변수 위치와 관계없이 동작하지만, 다른 연산자와 함께 쓸 경우 위치가 중요하다.

4. 🖥️ JAVA 연산자 개요

  • x와 y의 값은 각각 2와 -2가 출력되며, z 값은 y에 대한 증가 없이 1로 설정된다.
  • 증가 연산 후 y의 값은 -1이 되고, x는 3이 되는 결과를 보인다.
  • 논리 부정 연산자는 boolean 타입의 값을 반전시켜 true와 false 간의 변화를 보여준다.
  • 산술 연산자에는 덧셈, 뺄셈, 곱셈, 나눗셈, 나머지가 포함되며 이를 이용한 기본적인 계산 예제를 다룬다.
  • 비교 연산자는 변수 값의 같음과 다름, 크기 비교를 포함하고, 이를 기반으로 불린 타입의 결과를 도출한다.

4.1. 변수의 증가와 연산 결과

  • sout v를 사용하여 Z, x, y 값을 출력하며 실행 결과를 확인한다.
  • 실행 결과, x 값은 1에서 2로 증가하고, y 값은 각각 -1과 -2가 출력된다.
  • x 값에 1을 더한 후 결과를 Z에 대입하는데, y 값은 증가하지 않은 상태에서 대입된다.
  • 이로 인해 Z에는 1 값이 들어가고, 이후 y 값은 -1로 증가한다.
  • 최종적으로 x 값은 이미 증가하여 3으로 출력된다.

4.2. 증감 연산자 개요

  • 증감 연산자에 대한 설명이 진행되고 있다.

4.3. 논리 부정 연산자 설명

  • 논리 부정 연산자는 값의 true와 false를 반전시킨다.
  • 예를 들어, 초기값으로 true를 넣고, 이를 반전시켜 false가 만들어진다.
  • 다시 반전시킬 경우, 값을 true로 되돌릴 수 있다.
  • 이 연산자는 피연산자로 두 개의 값이 필요하다.

4.4. ️ JAVA 산술 연산자 및 문자열 결합 연산자

  • 산술 연산자는 덧셈, 뺄셈, 곱셈, 나눗셈, 나머지 등의 기본 연산을 포함하며, 변수를 사용하여 결과를 출력할 수 있다.
  • INT 타입의 변수 v1과 v2를 초기값으로 설정하고, 이들 간의 기본 산술 연산 결과를 INT 타입의 result 변수에 담아 출력하는 방식으로 설명된다.
  • double 타입으로 나눈 결과는 소수점 단위로 반환되며, 이를 통해 보다 정확한 계산 결과를 확인할 수 있다.
  • 문자열 결합 연산자는 문자열 간의 덧셈으로, 왼쪽부터 진행되며, 다양한 문자열이 합쳐진 결과를 만들어낸다.
  • 예를 들어, "jdk 6.0"에 "특징"이라는 문자열을 추가하면 "jdk 6.0 특징"이 되고, 각 결과를 통해 문자열 결합을 실습할 수 있다.

4.5. 비교 연산자와 논리 연산자

  • 비교 연산자는 '같다', '다르다', '크다', '작다', '크거나 같다', '작거나 같다'의 형식을 지닌다.
  • 예시를 통해, 두 정수 변수를 비교하여 결과를 불린 타입으로 저장하는 방식이 설명된다.
  • 유니코드 값을 비교하고, 대문자와 소문자의 결과를 예측해야 하며, 이를 통해 트루와 펄스 결과를 도출한다.
  • 부동소수점 수 비교에서, float과 double 형의 오차가 발생할 수 있는데, 이로 인해 서로 다른 결과를 가져오는 경우를 언급한다.
  • 논리 연산자인 AND는 두 조건이 모두 만족할 때만 true가 되며, 이를 통해 대문자와 소문자 또는 숫자의 범위를 확인하는 과정이 설명된다.

4.6. 대입 연산자의 개념과 활용

  • 대입 연산자는 오른쪽 피연산자의 값을 왼쪽 변수에 대입하는 역할을 한다.
  • 더하기 대입 연산자는 기존 변수 값에 피연산자 값을 더하여 재할당하는 방식으로 작동한다.
  • 빼기 대입 연산자는 기존의 값에서 피연산자 값을 빼고 다시 변수에 넣는다.
  • 곱하기 대입 연산자는 변수에 피연산자만큼을 곱해서 다시 변수에 재할당하는 기능을 수행한다.
  • 예제에서는 INT 타입의 변수에 대입 연산자와 복합 대입 연산자를 활용하여 여러 값을 대입하고 출력하는 과정을 설명하고 있다.

5. 📊 JAVA 연산자 활용 방법

  • 나눗셈과 나머지 계산을 통해 result라는 변수에 값을 할당하는 방법을 설명하고, 최종적으로 5로 나눈 값과 3으로 나눈 나머지를 result에 저장하는 것이었다.
  • 복합 대입 연산자와 삼항 연산자에 대한 설명이 포함되며, 삼항 연산자는 세 개의 피연산자를 필요로 하고 조건식에 따라 결과를 선택하는 구조로 되어 있다.
  • 특정 캐릭터의 점수에 따라 등급을 정하는 예시로, 점수 90 이상은 'A', 80 이상은 'B', 그 외 경우는 'C'로 설정되는 로직이 구현된다.
  • 실행 결과로 점수를 기준으로 한 등급이 출력되며, 값 연산의 결과는 단 하나의 값으로 나타난다고 주장하고 있다.
  • 연산자 사용의 필요성을 강조하며, 처음에 잘 정리하는 것이 중요하다는 메시지로 마무리된다.

Key term

자바

자바는 프로그래밍 언어 중 하나로, 다양한 장소에서 사용되는 강력한 언어입니다. 주로 모바일 애플리케이션과 서버 사이드 애플리케이션을 개발하는데 많이 쓰이죠.

연산자

연산자는 프로그램에서 수학적 또는 논리적 연산을 수행하는 기호입니다. 이렇게 생각해보면, 연산자는 레고 블록과 같아서 조합하여 다양한 결과를 만들어낼 수 있습니다.

비교 연산자

비교 연산자는 두 값을 비교하여 맞는지 틀리는지를 판단합니다. 예를 들어, 두 개의 사과를 비교해서 크거나 같은지 확인하는 것과 비슷하죠.

산술 연산자

산술 연산자는 숫자들 간의 기본적인 수학 연산을 수행합니다. 즉, 더하기, 빼기, 곱하기, 나누기와 같은 것들이죠.

대입 연산자

대입 연산자는 오른쪽의 값을 왼쪽 변수에 저장합니다. 예를 들어, 'x = 5'는 5라는 값을 x라는 곳에 담는 것이죠.

문자열 비교

문자열 비교는 두 문자열이 같은지 비교하는 과정입니다. 예를 들어, '자바'와 '자바'를 비교할 때, 같은 문자열인지 확인하는 것과 같아요.

증감 연산자

증감 연산자는 특정 변수의 값을 1씩 증가시키거나 감소시키는 역할을 합니다. 예를 들면, 'x++'는 x의 값을 1 증가시키는 것이죠.

조건 연산자

조건 연산자는 조건에 따라 두 값을 선택하는 기능을 합니다. 쉽게 말하면, '만약 A면 X, 그렇지 않으면 Y'와 같은 구조입니다.

논리 연산자

논리 연산자는 여러 조건을 결합하여 참(true) 또는 거짓(false)을 결정합니다. 예를 들어, 'A가 참이고 B가 참이면 참'이라는 식으로 말이죠.

피연산자

피연산자는 연산자와 함께 연산이 이루어지는 대상입니다. 쉽게 말해, 연산을 하는 대상인 숫자나 변수를 뜻합니다.

오퍼레이터

오퍼레이터는 연산자를 영어로 표현한 것입니다. 즉, 프로그래밍에서 연산을 수행하는 기호를 의미해요.

연산식

연산식은 연산자와 피연산자를 조합하여 결과를 나타내는 전체 표현입니다. 레고 블록을 조립하여 하나의 구조물을 만드는 것과 비슷하죠.

우선순위

우선순위는 여러 연산자가 있을 때 어떤 연산을 먼저 수행할지를 결정합니다. 마치 요리할 때 재료를 어떤 순서로 넣는지 정하는 것과 같아요.

형변환

형변환은 데이터 타입을 다른 타입으로 바꾸는 과정을 말합니다. 예를 들어, 숫자를 문자로 바꾸는 것처럼 다양한 방식으로 이루어질 수 있습니다.

실행

실행은 작성한 코드가 실제로 작동하여 결과를 만드는 과정을 의미해요. 요리를 한 후, 음식을 맛보기 위해 먹는 것처럼 생각할 수 있죠.

트루

**트루(true)**는 참이라는 의미로, 가정이나 조건이 사실일 때 나타납니다. '사과가 빨갛다'가 사실이라면 이 조건은 트루입니다.

펄스

**펄스(false)**는 거짓이라는 의미입니다. 예를 들면, '사과가 파란색이다'가 사실이 아닌 경우입니다.

리터럴

리터럴은 프로그램 코드에 직접 쓰이는 데이터 값을 의미합니다. 예를 들어, 숫자 5나 문자열 '안녕하세요'가 리터럴입니다.

변수

변수는 데이터를 저장할 공간으로, 이름 붙인 메모리 상자와 비슷합니다. 다양한 값을 저장하고 관리할 수 있어요.

예제

예제는 배운 내용을 실제로 보여주기 위해 사용하는 사례입니다. 마치 설명을 이해하기 위한 그림이나 시각적 학습처럼요.

코드

코드는 컴퓨터가 이해할 수 있는 명령어의 집합으로, 프로그램을 만드는 기본적인 요소입니다. 요리 레시피와 비슷하다고 할 수 있죠.

프로그래밍

프로그래밍은 문제를 해결하기 위해 컴퓨터에 명령어를 작성하는 과정입니다. 마치 요리를 위해 재료를 준비하고 조리법을 따르는 것과 유사합니다.

조건문

조건문은 주어진 조건에 따라 다른 동작을 하도록 프로그램을 제어하는 방법입니다. 예를 들어, '비가 오면 우산을 챙기세요'와 같은 방식으로 조건을 설정합니다.

결과값

결과값은 수행된 연산이나 함수의 최종 결과를 의미합니다. 시험 결과처럼, 어떤 행동의 결과가 나오는 것이죠.

조건식

조건식은 조건문에서 평가되는 표현으로, 참인지 거짓인지 결정합니다. 예를 들어, 'x가 10보다 큰가요?'와 같은 질문입니다.
