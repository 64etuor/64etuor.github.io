3강. JAVA문법(시스템 입출력)

이 강의는 자바의 시스템 입출력에 대한 내용을 다룹니다. 자바의 표준 입력과 출력을 통해 어떻게 데이터를 전달하고 출력하는지를 이해하는 데 도움이 됩니다. 변수에 저장된 값을 모니터에 출력하는 방법과 키보드로부터 입력을 받는 과정을 상세히 설명합니다. 주요 메소드인 println, print, 그리고 printf의 사용법을 배우고, 이를 통한 데이터 형식 지정 및 출력 포맷팅의 중요성을 강조합니다. 이 강의를 통해 시청자는 자바 프로그래밍에서 필수적인 입출력 기능을 숙지할 수 있습니다.

1. 🖥️ 자바 시스템 입출력 이해하기

  • 자바에서 시스템 입출력은 변수와 함께 모니터, 키보드를 기본으로 한다.
  • 시스템 아웃은 모니터에 변수 값을 출력하고, 시스템 인은 키보드로부터 데이터를 읽어오는 역할을 한다.
  • 이 과정에서 변수의 값 입력과 출력 방법에 대해 학습하는 것이 중요하다.

2. 🖥️ JAVA의 출력 메소드 이해

  • println 메소드는 괄호 안에 리터럴을 넣으면 그대로 출력되며, 변수를 사용할 경우 저장된 값이 출력된다.
  • printf는 개발자가 원하는 형식화된 문자열을 출력할 때 유용하게 사용되는 메소드로, 형식 문자열과 변환될 값들을 순서대로 적어주어야 한다.
  • 형식 문자열에서 퍼센트와 변환될 문자 외에는 생략이 가능하며, 문자열은 %s로, 정수는 %d로 표시된다.
  • 여러 개의 값이 포함될 경우 출력 순서에 따라 표시하는 것이 좋으며, 예를 들어 정수와 문자열을 각각 다른 플레이스 홀더에 집어넣어 출력할 수 있다.
  • 실습을 통해 출력 형식을 조정할 수 있으며, 각 값의 정렬 방식을 설정할 수 있고, 소수점 자리수도 제어할 수 있다.

2.1. JAVA에서의 출력 메소드

  • println 메소드는 괄호 안에 리터럴을 넣으면 그대로 출력된다.
  • 변수를 사용할 경우 저장되어 있는 값이 출력되는 기능이 있다.
  • 다양한 출력 메소드 중에서 println은 sout을 통해 자동으로 생성된 메소드이다.
  • ln이라는 메소드는 괄호 안의 내용을 출력하지만, 행을 바꾸지 않는 특징이 있다.

2.2. ️ 프린트 f와 형식 문자열 출력

  • 프린트 f는 개발자가 원하는 형식화된 문자열을 출력할 때 유용하다.
  • 형식 문자열에는 변환될 문자들이 포함되며, 이후에 플레이스홀더로 대체할 값들을 순서대로 적어주어야 한다.
  • 형식 문자열에서 퍼센트와 변환 문자(예: s, d)만 필수적으로 적어주면 나머지는 생략할 수 있다.
  • 예를 들어, s는 문자열을, d는 정수를 의미하여, 이에 따라 자리에는 정수와 문자열을 적어주면 된다.
  • 형식 문자열에 포함된 값이 2개 이상일 경우는 값의 순번을 표시하는 것이 좋다.

2.3. ️ JAVA 입출력 실습 준비 과정

  • 패키지는 section 01 아래의 exam 01로 만들며, 그 안에 printf example이라는 이름을 주기로 한다.
  • 메인 메서드에서 int value를 이용해 1, 2, 3의 값을 선언하고, soutf 형식으로 출력을 시도한다.
  • 형식 문자열을 통해 가격과 퍼센트를 출력하고, 원을 포함한 포맷을 설정한다.
  • 실행 결과, 값들은 오른쪽 정렬되며 빈 곳은 공백으로 채워진 것을 확인할 수 있다. 또한, 마이너스 기호는 왼쪽 정렬을 유도하고, 0은 공백을 채운 사례를 보여준다.

2.4. 변수 선언 및 형식 설정

  • 더블형의 area 변수를 선언하고, 이를 3.14159 곱하기 10 곱하기 10으로 초기화하는 과정을 설명한다.
  • 서식을 적용하기 위해 반지름을 %d로 나타내고, 넓이는 퍼센트 10.2 f로 설정한다.
  • 두 개의 플레이스 홀더가 존재하고, 각각 정수와 실수 값을 하나씩 넣어줘야 한다.
  • area와 10이라는 값을 사용하여 플레이스 홀더에 값을 삽입할 예정이다.
  • 값이 나오는 방식은 이미 여러 번 시행된 내용을 기반으로 간략히 이해할 수 있다.

2.5. ️ JAVA 스트링 타입과 출력 포맷

  • 스트링 타입 변수를 생성하고 값으로 "홍길동"을 할당한다.

  • 이 예제를 통해 JAVA에서의 출력 포맷을 이해한다.

3. 🖥️ 시스템 입력과 keyCode 처리

  • 시스템 입력을 다루기 위해서는 리드 메소드를 사용하여 값을 입력 받아야 한다.
  • 입력 받은 값은 코드 값 하나로, 주로 int(integers) 형식으로 처리된다.
  • keyCode는 특정 키를 입력했을 때 생성되며, 예를 들어 엔터키 입력 시 특정 keyCode 값이 발생한다.
  • system.read 메소드는 두 개 이상의 키 조합이나 문자열을 읽을 수 없다는 한계가 있다.
  • 반복 입력을 처리할 수 있으며, 특정 키 입력에 따라 프로그램 종료를 구현할 수 있다. 예를 들어, 'Q' 키를 입력하면 프로그램이 종료되도록 할 수 있다.

4. 📥 스캐너 클래스와 문자열 입력 처리

  • 문자열을 입력받기 위해서는 스캐너 클래스를 사용해야 하며, 이는 새로운 객체를 생성하는 new 키워드를 이용해 변수에 담는다.
  • nextLine 메소드를 통해 엔터키 이전까지 입력된 문자열 값을 읽어올 수 있으며, 이 메소드는 한 줄씩 문자열 입력을 처리한다.
  • 입력 데이터는 equals 메소드를 통해 특정 문자열과 비교할 수 있으며, 입력된 값이 q일 경우 프로그램이 종료되도록 설정할 수 있다.
  • 프로그램은 반복문을 통해 사용자 입력을 받고, 마지막으로 System.out을 통해 입력된 내용을 출력하며 종료 메시지를 표시한다.
  • 이러한 과정은 스캐너 객체를 통해 쉽게 구현 가능하며, 예제를 통해 실습이 이루어진다.

5. 📝 입출력 정리

  • 입출력에 대한 내용을 정리하고 마무리짓는다.
  • 강의에 참여해준 모든 사람들에게 감사의 인사를 전한다.

Key term

자바

자바는 컴퓨터 프로그래밍 언어의 하나로, 소프트웨어를 만들기 위해 사용됩니다. 마치 사람의 말을 배우듯, 컴퓨터에게 명령을 주기 위해 우리가 사용하는 언어입니다. 이 언어는 '객체 지향 프로그래밍(OOP)'이라는 특징을 가지고 있어요.

시스템 입출력

시스템 입출력이란 컴퓨터와 사용자가 정보를 주고받는 방법을 말해요. 예를 들어, 키보드로 입력한 내용을 프로그램이 처리하고, 그 결과를 모니터에 보여주는 방식이 이에 해당하죠.

변수

변수는 데이터를 저장하는 공간입니다. 마치 책상 위의 서랍처럼, 필요한 정보를 담아두는 곳이죠. 예를 들어, 학생의 나이를 저장한다면 변수에 '나이'라는 이름을 붙이고 해당 나이를 기록할 수 있어요.

표준 출력

표준 출력은 프로그램이 처리한 결과를 사용자에게 보여주는 방법입니다. 일반적으로는 모니터를 통해 결과를 보여주는데, 사람의 눈에 보이는 출력이라고 할 수 있어요.

표준 입력

표준 입력은 프로그램이 사용자로부터 데이터를 받는 방법을 말합니다. 주로 키보드를 통해 정보를 입력받는데, 사용자가 무엇을 원하는지 알아내는 과정이죠.

System.out

System.out은 자바에서 표준 출력을 담당하는 객체입니다. 즉, 개발자가 프로그램에서 결과를 출력하고 싶을 때 주로 사용하는 도구예요.

System.in

System.in은 자바에서 표준 입력을 담당하는 객체입니다. 사용자가 키보드로 입력한 데이터를 읽어오는 역할을 해요.

println

println은 출력 후 줄을 바꾸는 메소드입니다. 이 메소드를 사용하면 모니터에 출력한 후 자동으로 다음 줄로 내려가죠. 마치 메모장에 글을 쓸 때 줄바꿈 하는 것과 같아요.

print

print는 출력 후 줄을 바꾸지 않는 메소드입니다. 즉, 같은 줄에서 계속 출력할 수 있게 해주죠. 예를 들어, '안녕하세요'를 연속으로 출력할 수 있어요.

printf

printf는 형식화된 출력을 위해 사용하는 메소드입니다. 출력할 형식을 정할 수 있어, 특정한 모양으로 데이터를 보여줄 수 있습니다. 요리할 때 저울로 정확히 측정하는 것과 비슷해요.

형식 문자열

형식 문자열은 printf 메소드에서 출력을 어떻게 할지를 정의하는 문자열입니다. 예를 들어, '이름: %s, 점수: %d' 와 같이 정해놓으면, 나중에 실제 값이 대신 들어가게 돼요.

포멧팅

포멧팅은 데이터를 원하는 형식으로 정리하는 것을 의미합니다. 예를 들어, 날짜나 숫자를 특정한 규칙으로 표시하는 것이죠. 마치 옷을 입히는 것과 같아요.

문자열

문자열은 문자들이 모여서 만드는 데이터 형식입니다. 예를 들어, '안녕하세요' 또한 문자열이에요. 단어, 문장 등을 저장할 수 있죠.

정수

정수는 숫자 중에서 소수점이 없는 것을 말해요. 예를 들어, 1, 2, 3 같은 숫자들이죠. 계산할 때 주로 사용됩니다.

SCANNER

SCANNER는 자바에서 입력을 쉽게 받을 수 있도록 도와주는 클래스입니다. 누군가 질문을 하면 대답하는 친구와 같아요.

nextLine

nextLine은 SCANNER 클래스에서 한 줄의 문자열을 입력받는 메소드입니다. 사용자가 엔터키를 누를 때까지 기다렸다가 그 내용을 읽어옵니다.

예외 처리

예외 처리란 프로그램 실행 중 오류가 발생했을 때 이를 해결하는 방법을 찾는 것입니다. 마치 건강 검진에서 문제가 생겼을 때 의사에게 의뢰하는 것과 같죠.

키보드 입력

키보드 입력은 사용자가 키보드를 통해 프로그램에 정보를 주는 방법입니다. 사용자가 직접 데이터를 입력하는 과정이에요.

컴퓨터 프로그래밍

컴퓨터 프로그래밍은 컴퓨터에게 수행할 작업을 명령어로 주는 과정입니다. 요리할 때 레시피를 따라 하는 것과 비슷하죠.

입출력 메소드

입출력 메소드는 정보를 입력받고 출력하는 기능을 수행하는 함수들입니다. 이러한 메소드들을 활용해 우리는 컴퓨터와 소통할 수 있어요.

데이터 전송

데이터 전송은 정보를 한 곳에서 다른 곳으로 옮기는 과정을 말합니다. 마치 편지를 보내는 것과 같은데, 정보를 이동시키는 거죠.

코드 예제

코드 예제는 특정한 기능이나 개념을 설명하기 위해 제공하는 코드입니다. 실습을 통해 배운 내용을 이해하는 데 도움이 되죠.

대학교 수업

대학교 수업은 고등 교육 기관에서 제공하는 수업으로, 전문적인 지식을 배울 수 있는 기회를 제공합니다. 자바 프로그래밍 같은 과목이 포함됩니다.

실습

실습은 실제로 코드를 작성하고 실행해보는 과정을 의미해요. 이론과 함께 실습을 통해 지식이 더 확고해지지요.

객체 생성

객체 생성은 특정 클래스에서 새로운 객체를 만드는 과정입니다. 마치 나무를 심어 새로운 나무를 만드는 것과 비슷해요.

무한 반복

무한 반복은 특정한 조건 없이 계속해서 반복하는 코드를 의미합니다. 마치 계속해서 같은 노래를 반복 재생하는 것과 같아요.

keyCode

keyCode는 키보드의 각 키가 입력될 때 생성되는 고유한 숫자입니다. 각 키마다 정해진 번호가 있어, 그 번호로 키를 인식하죠.
