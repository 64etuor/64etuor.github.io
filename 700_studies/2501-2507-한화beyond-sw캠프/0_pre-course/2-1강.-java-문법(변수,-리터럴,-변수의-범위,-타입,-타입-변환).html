2-1강. JAVA 문법(변수, 리터럴, 변수의 범위, 타입, 타입 변환)

이번 영상에서는 자바의 변수와 타입에 대한 기본 개념을 배웁니다. 변수를 어떻게 선언하고, 리터럴을 어떤 방식으로 입력하는지, 그리고 초기화와 타입변환에 대한 내용을 쉽게 설명합니다. 또한, 변수의 범위와 타입의 중요성을 강조하며, 코드 작성 시 주의할 점을 수록합니다. 이를 통해 자바프로그램을 작성하는 데 필요한 기본적인 지식과 규칙을 습득할 수 있습니다. 변수와 타입의 이해는 프로그래밍의 기초이며, 코드의 명확성을 높이는 데 필수적입니다.

1. 📦 자바의 변수와 메소드 개요

  • 자바의 각 실행문은 반드시 세미콜론(;)으로 끝나야 하며, 이는 문법적 필수 요소이다.
  • 클래스와 소스 파일의 이름은 대소문자를 구분하며 일치해야 하며, 'public class'는 공개 클래스를 나타낸다.
  • "main" 메소드는 자바프로그램의 시작점으로, 프로그램이 실행될 때 가장 먼저 호출되는 부분이다.
  • 주석은 코드 실행에 영향을 미치지 않으며, 코드에 대한 설명을 추가하여 유지보수성과 재사용성을 높이는 데 기여한다.

1.1. Java의 변수와 타입 개요

  • 이번 강의에서는 변수와 타입에 대해서 설명한다.
  • 강의는 "Hello.java"라는 소스 파일을 바탕으로 진행된다.
  • "package sec03.exam01;"는 "Hello.java" 파일이 특정 패키지에 포함되어 있음을 나타낸다.
  • 각 실행문의 끝에 반드시 세미콜론(;)을 추가해야 한다.

1.2. ️ 자바 클래스와 메인 메서드 개요

  • "public class Hello"는 클래스의 선언으로, "public"은 공개적이며 "class"는 클래스임을 나타낸다.
  • 클래스의 이름은 소스 파일명과 동일해야 하며, 대소문자를 정확하게 구분해야 한다.
  • 메인 메서드는 프로그램의 실행 진입점으로 "public static void main(String[] args)"로 정의되며, "args"는 매개 변수이름이다.
  • 메인 메서드는 자바프로그램의 시작점으로, 가장 먼저 실행되는 부분이다.
  • 메서드 블록의 시작과 끝은 중괄호로 표시되며, 자바프로그램 실행 시 메인 메서드 안의 블록이 실행된다.

1.3. 자바의 주석 사용

  • 주석은 프로그램이 실행되지 않는 영역으로, 코드를 설명하는 용도로 주로 사용된다.
  • 주석을 통해 코드의 유지보수성을 높이고, 수정할 부분을 명확히 파악할 수 있게 도와준다.
  • 주석을 다는 습관은 코드의 재사용성을 증가시킬 수 있다.
  • 문자열 리터럴안에는 주석을 포함할 수 없으며, 이는 큰따옴표로 묶인 범위에서 적용된다.
  • 자바에는 세 가지 유형의 주석표현 방식이 존재하는데, 두 가지 방법으로는 라인 주석과 범위 주석이 있다.

1.4. 자바 메서드와 주석 작성의 중요성

  • 메서드를 작성할 때 다큐먼트 주석을 추가하여 API 문서를 생성할 수 있다.
  • 주석이 포함된 소스 코드는 클래스파일로 변환될 때 주석이 제거되므로, 코드의 용량에는 영향을 주지 않는다.
  • 주석은 코드에 대한 부가적인 설명을 추가하는 데 유용하며, 주석작성을 연습하여 효과적인 사용법을 익혀야 한다.
  • 세미콜론은 자바에서 실행문의 끝을 나타내는 중요한 문법으로, 누락 시 컴파일 에러가 발생할 수 있다.
  • 변수의 선언 및 값 저장 시, 등호는 값을 변수에 할당하는 역할을 한다.

1.5. 변수의 정의 및 역할

  • 변수는 컴퓨터 메모리에 값을 저장하는 데 사용된다.
  • 컴퓨터 메모리는 수많은 주소로 구성되어 있으며, 변수는 이 주소들 중 하나에 값을 저장하는 방식을 정의하는 것이다.
  • 변수를 재정리할 필요성이 있다.

2. 💾 변수의 개념과 역할

  • 컴퓨터의 메모리는 전기 신호로 구성된 데이터의 저장 공간이며, 메모리의 각 위치에는 고유한 주소가 부여되어 있다.
  • 변수는 특정 값에 이름을 붙여 해당 값을 식별하고 접근할 수 있게 해주는 개념이다.
  • 변수를 선언할 때는 데이터의 타입을 지정하고, 변수의 이름을 함께 작성해야 하며, 이는 코드의 가독성을 높인다.
  • 변수의 이름은 특정 규칙을 따라야 하며, 첫 글자는 문자, 달러 기호, 언더스코어중 하나여야 하고, 대소문자를 구분해야 한다.
  • 변수명은 그 변수가 무엇을 저장하는지 의미를 잘 드러낼 수 있도록 지어야 하며, 자바의 예약어는 변수이름으로 사용할 수 없다.

2.1. ️ 메모리와 변수의 개념

  • 컴퓨터에는 메모리라는 저장 공간이 존재하며, 이는 주소를 통해 접근할 수 있는 구조이다.
  • 메모리에는 전기 신호가 0과 1로 구성된 데이터 형태로 저장되어 있으며, 특정 위치에서의 데이터를 사용하게 된다.
  • 변수는 특정 값에 이름을 붙여 그 값을 식별하고 접근할 수 있도록 하는 개념이다.
  • 변수는 메모리에서 값들을 포함하고 있으며, 이 값들은 1과 0으로 이루어져 있다.
  • 메모리의 주소는 고유하여 각 위치를 식별할 수 있으며, 이를 통해 해당 메모리주소에서 값을 가져올 수 있다.

2.2. 변수의 선언과 타입

  • 변수를 선언하기 위해서는 변수의 타입을 명시하는 것이 필요하다.
  • 변수의 타입은 저장할 데이터의 종류를 나타내며, 인트(int)와 더블(double)이 기본 타입의 예이다.
  • 변수를 선언할 때는 타입을 먼저 적고, 그 뒤에 변수의 이름을 붙여서 완성한다.
  • 같은 타입의 변수를 여러 개 선언할 경우, 콤마를 사용하여 한 번에 선언할 수 있다.
  • 예를 들어, 'a'라는 변수는 정수값을 저장하며 '밸류'라는 변수는 실수값을 저장하는 데 사용된다.

2.3. ️ 변수 이름 규칙

  • 변수이름을 지을 때는 규칙을 준수해야 하며, 첫 번째 글자는 문자, 달러 기호, 또는 언더스코어여야 한다.
  • 변수이름은 숫자로 시작할 수 없으며, 사용하지 않아야 할 기호가 섞여서는 안 된다.
  • 영어 대소문자를 구분하므로, "first name"과 "first Name"은 서로 다른 변수로 인식된다.
  • 여러 단어를 합칠 때, 첫 번째 단어는 소문자로 시작하고 이후 단어의 첫 글자는 대문자로 표기하는 것이 관례이며, 이를 케이스라고 한다.
  • 변수의 문자 수에는 길이 제한이 없다.

2.4. 자바의 예약어와 변수 명명 규칙

  • 자바의 예약어는 변수의 이름으로 사용할 수 없으며, 이는 자바문법에서 이미 규정된 단어들이다.
  • 변수의 이름은 저장하고 있는 값이나 역할을 쉽게 파악할 수 있도록 의미 있게 지어야 하며, 예를 들어 abc 같은 이름은 혼란을 초래할 수 있다.
  • 변수의 이름 길이에는 제약이 없고, 실행 속도와도 크게 무관하므로, 충분히 길고 정확하게 작성하는 것이 좋다.
  • 변수이름은 영어로 명명하는 것이 바람직하며, 한글은 협업 시 긍정적으로 받아들여지지 않을 가능성이 높다.
  • 예약어는 다양하고, 이들을 변수이름으로 사용하면 컴파일 에러가 발생하므로, 이를 기억할 필요는 없으며 학습을 통해 자연스럽게 익혀가는 것이 중요하다.

2.5. ️ 변수의 선언 및 값 할당

  • 변수는 특정 값을 접근하기 위한 이름으로, 이름을 통해 해당 값을 사용할 수 있다.
  • 변수선언에는 int와 같은 데이터 타입이 지정되고, 예를 들어 "스코어"라는 변수에 정수 90이 할당된다.
  • 값 할당은 대입 연산자인 등호(=)를 사용하여 이루어지며, 이는 왼쪽에 변수를 두고 오른쪽에 값을 넣는 방식이다.
  • 대입 연산자는 해당 값이 변수에 저장된다는 의미로 사용되며, '같다'라는 의미로 해석하지 않는다.
  • 변수를 선언하고 값을 할당하는 과정은 프로그램에서 매우 중요한 기본 개념이다.

3. 📊 변수 초기화와 범위 이해하기

  • 대입 연산자를 사용하여 변수에 값을 할당하는 과정을 초기화라고 하며, 이때 할당된 값을 초기 값이라고 한다.
  • 초기화를 하지 않은 변수는 메모리에서 값을 읽을 수 없고, 필요한 값을 할당하지 않으면 에러가 발생한다.
  • 변수를 사용할 때는 변수의 이름을 활용해 값을 관리하며, 이를 통해 계산이나 출력 등을 수행할 수 있다.
  • 로컬 변수는 메소드블록 내에서만 사용 가능하며, 블록 밖에서는 해당 변수를 접근할 수 없다.
  • 변수의 타입에 따라 저장할 수 있는 값의 종류와 범위가 결정되므로, 변수의 선언 위치를 고려해야 한다.

3.1. 변수 초기화와 할당

  • 대입 연산자를 사용하여 변수에 값을 할당하는 과정이 설명된다. 예를 들어, 스코어라는 변수에 90을 대입하는 것이 해당 사례이다.
  • 변수선언 후 처음 할당된 값을 초기값이라고 하며, 이를 통해 변수가 생성되고 값을 초기화한다고 설명한다.
  • 초기화를 하지 않은 변수는 메모리에서 값을 읽을 수 없기 때문에, 반드시 초기화를 요구한다.
  • 값을 선언과 동시에 할당하는 방법도 있으며, 30이 초기값으로 주어질 때의 결과도 다루어진다.
  • 변수선언과 값 할당을 코드 예제를 통해 구체적으로 보여주며, 변수x에 1, 변수y에 2가 저장되는 과정을 설명한다.

3.2. JAVA의 변수와 초기화 과정

  • INT 타입의 변수를 선언하고, x와 y의 덧셈 결과를 변수결과에 저장하는 방법을 설명한다.
  • 변수를 선언한 뒤 값을 넣지 않으면 초기화되지 않은 변수로 간주되어 에러가 발생한다.
  • 초기화를 하지 않은 변수는 메모리에서 값을 읽을 수 없다는 점을 강조한다.
  • 변수이름을 활용하여 저장된 값을 사용하고, 출력문에서 변수를 활용할 수 있음을 보여준다.
  • 대입 연산자를 사용하여 한 변수의 값을 다른 변수에 복사할 수 있다는 것을 설명하며, 이 과정에서 변수의 값이 동일하게 유지됨을 명시한다.

3.3. 변수의 범위

  • 변수의 범위는 메소드블록 내에서 선언된 변수를 로컬 변수라고 정의한다.
  • '메인' 메소드블록 내에서 선언된 변수들은 그 블록 안에서만 사용 가능하다.
  • 메인 메소드블록 외부에서는 메인 메소드안에서 생성한 로컬 변수를 사용할 수 없다.

3.4. ️ 변수의 범위와 사용

  • 회사의 로컬 변수는 해당 중괄호 블록 내에서만 사용이 가능하다.
  • 메인 메소드내에서 선언된 변수는 같은 블록 내에서 접근할 수 있지만, 중괄호 밖에서는 사용할 수 없다.
  • 중괄호 안에서 유효한 변수는 블록을 벗어나면 더 이상 사용할 수 없다.
  • 변수를 선언할 때 어떤 범위안에서 사용될 것인지 고려해야 하며, 이는 코드 작성의 핵심이다.

3.5. 변수의 범위와 타입 선언

  • 메소드블록 전체에서 사용하는 경우, 메소드 블록 전체에 변수를 선언하는 것이 바람직하다.
  • 특정 블록 내부에서만 사용하는 경우에는 해당 블록의 첫 부분에 변수를 선언하는 것이 좋다.
  • 변수의 타입을 살펴보면, 변수 타입에 따라 저장할 수 있는 값의 종류와 허용 범위가 달라진다.

4. 📊 자바의 기본 데이터 타입과 리터럴

  • 프리미티브 타입은 크게 정수값, 실수값, 논리값으로 나누어지며, 각각의 타입의 허용 범위가 정해져 있다.
  • 자바의 기본 정수 타입은 바이트(8비트), 숏, 인트, 롱으로 나뉘며, 바이트는 -128부터 127까지의 값을 표현할 수 있다.
  • 리터럴은 프로그래머가 코드에 직접 입력하는 값으로, 이진수, 팔진수, 10진수, 16진수 등 다양한 형식으로 표현될 수 있다.
  • 캐릭터 타입은 하나의 문자를 저장하며, 작은 따옴표로 감싼 문자 리터럴은 유니코드 정수값으로 변환되어 저장된다.
  • 실수 타입은 플로트와 더블로 나뉘며, 소수점의 표현 방식과 메모리사용량이 다르다. 더블이 기본 타입으로 사용되며, 플로트형 리터럴은 f를 붙여서 구분해야 한다.

4.1. 기본 데이터 타입 및 리터럴 이해

  • primitive 타입은 정수값, 실수값, 그리고 논리값으로 크게 세 가지로 분류된다.
  • 기본 타입에는 바이트, 숏, 인트, 롱, 플로트, 더블, 불린이 포함된다.
  • 바이트는 8비트를 의미하며, -128부터 127까지의 숫자를 표현할 수 있다.
  • 데이터 타입은 사용하는 메모리의 크기에 따라 선택해야 하며, 일반적으로 인트형으로 계산하는 일이 많다.
  • 리터럴은 프로그래머가 소스 코드에 직접 입력한 값을 의미하며, 이진수, 팔진수, 10진수, 16진수 등 여러 가지 형식으로 표현할 수 있다.

4.2. 캐릭터 타입과 유니코드 저장 방법

  • 캐릭터 타입은 하나의 문자를 저장할 수 있는 데이터 타입이다.
  • 문자 리터럴은 작은 따옴표로 감싸져 유니코드로 변환되어 저장된다.
  • 캐릭터 타입은 기본적으로 정수 타입으로 여겨지며, 유니코드는 정수 값으로 저장된다.
  • 특정 문자의 유니코드 값은 정수 값으로 맵핑되며, 예를 들어 'A'는 65, 'B'는 66으로 저장된다.
  • 글자 하나를 표현할 때는 작은 따옴표를 사용하여 글자를 쌍으로 감싸고, 직접적으로 정수 형태의 리터럴값도 사용할 수 있다.

4.3. 문자열 리터럴과 스트링 타입

  • 문자열 리터럴은 큰따옴표로 감싸야 하며, 이렇게 표현된 값은 캐릭터 타입에 저장할 수 없다.
  • 자바에서는 스트링 타입이 있어, 이를 사용하여 문자열 리터럴을 표현해야 하며, 스트링은 첫 글자가 대문자이다.
  • 스트링 클래스 타입의 변수는 큰따옴표로 만든 리터럴값을 대입해 저장할 수 있다.
  • 이스케이프 문자는 역 슬래시를 사용하여 특정 문자를 표현하거나 문자열 출력을 제어하는 데 쓰인다.
  • 문자열 안에 큰따옴표를 삽입하고 싶을 때는, 역 슬래시를 붙여 문자를 표현할 수 있다.

4.4. 실수 타입과 리터럴 표현

  • 실수 타입은 소수점을 포함하며, 정수 타입에서 다름을 나타낸다.
  • 실수 표현에는 플로트와 더블이 있으며, 두 타입은 메모리사용 크기와 허용 범위가 다르다.
  • 정밀도와 관련하여 메모리사용 크기가 클수록 더 높은 정확도로 소수점을 표현할 수 있다.
  • 자바의 기본 실수 리터럴은 더블 타입으로, 실수 리터럴에서 자동적으로 더블로 인식된다.
  • 플로트형 리터럴은 값을 입력할 때 숫자 뒤에 f를 붙여서 구분해야 한다.

4.5. 논리타입과 타입 변환의 이해

  • 논리타입은 참(true)과 거짓(false)을 나타내는 데이터 타입이다.
  • 변수스탑에는 true 값을, 변수스테이트에는 false 값을 넣는 예를 통해 사용 방법을 설명한다.
  • 대소문자를 구분하는 특성으로 인해 'true'와 'false'는 소문자로만 작성해야 한다.
  • 정수, 실수, 논리 타입등을 포함하는 이들 데이터 타입은 프리미티브 타입이라고 불리며, 필요에 따라 타입 변환이 가능하다.
  • 타입변환에는 여러 형식이 있으며, 예를 들어 바이트 타입을 int로, 더블을 int로 변환할 수 있다는 사실을 설명한다.

5. 📚 자바의 자동 및 강제 타입 변환 개념

  • 자동 타입 변환은 작은 타입에서 큰 타입으로 값을 저장할 때 발생하며, 이 경우에는 타입 변환이 자동으로 이루어진다.
  • 타입의 크기는 더블, 플로트, 인트, 바이트 순서로 구성되며, 예를 들어 더블 타입 변수에 바이트, 숏, 인트, 플로트, 더블의 값을 모두 대입할 수 있다.
  • 캐릭터 값은 인트 타입으로 자동 변환되며, 예를 들어 캐릭터 'A'는 유니코드 값 65로 변환되어 저장된다.
  • 강제 타입 변환은 큰 타입에서 작은 타입으로 값을 변환할 때 사용되며, 이때는 소괄호를 사용하여 명시적으로 타입을 지정해줘야 한다.
  • 기본적으로 자바에서 정수 연산 시 인트 타입이 사용되며, 정수 연산의 결과는 항상 정수형으로 유지되기 때문에, 정수를 나누거나 계산할 때 소수점 이하 정보는 손실된다.

5.1. 자동 타입 변환의 개념

  • 인트 타입의 변수 'b'에 'a'의 값이 저장되면서 타입 변환이 발생할 수 있다.
  • 값의 허용 범위가 작은 타입이 큰 타입에 저장될 경우 자동으로 타입이 변환된다.
  • 작은 허용 범위의 타입을 큰 범위의 타입에 넣는 개념은 자동 타입 변환으로 이해할 수 있다.
  • 타입의 순서는 더블, 플로트, 기본 타입으로 이루어지며, 더블 타입 변수에는 다양한 타입의 값을 넣을 수 있다.
  • 이 과정에서 자동으로 타입변환이 이루어지며 값이 들어간다.

5.2. 캐릭터와 인트 타입의 자동 형변환

  • 캐릭터는 인트 타입으로 자동 변환되어 유니코드 값이 인트 타입으로 저장된다.
  • 예를 들어, 65라는 정수값이 캐릭터 타입으로 존재할 수 있다.
  • 이 값은 인트 타입의 변수에 담길 때 자동 형변환이 이루어져 INT 타입의 값 65로 변환된다.
