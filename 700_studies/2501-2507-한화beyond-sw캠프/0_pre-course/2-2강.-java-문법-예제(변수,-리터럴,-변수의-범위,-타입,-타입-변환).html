2-2강. JAVA 문법 예제(변수, 리터럴, 변수의 범위, 타입, 타입 변환)

이 강의 영상은 JAVA 문법의 기초를 이해하는데 도움을 줍니다. 특히, 변수와 리터럴, 각 변수의 범위, 데이터의 타입과 타입 변환의 필요성을 설명합니다. 실습을 통해 직접 코드 작성을 하며 변수를 선언하고 연산 결과를 출력하는 기회를 제공합니다. JAVA의 기초적인 문법을 차근차근 이해하고 활용할 수 있는 중요한 기초를 다지는 시간이 될 것입니다. 이 강의를 통해 프로그래밍 입문에 필요한 실무적인 기술을 익히게 됩니다.

1. 🛠️ 변수 선언 및 초기화 실습

  • 새로운 프로젝트를 생성하고, "베리어블"이라는 이름으로 작업을 시작한다.
  • 실습을 위해 INT 타입의 변수를 선언하고, 정수 리터럴 22로 초기화하는 과정을 진행한다.
  • "베리업"이라는 패키지를 만들고, 이를 "이니셜라이제이션 이그젬플"이라는 이름으로 설정한다.

2. 🖥️ 변수와 산술 연산의 예제

  • 변수의 밸류에 10을 더하는 산술 연산을 수행하고, 결과 값을 변수 result에 초기화하는 코드를 작성하였다.
  • 변수 result의 값을 읽고 콘솔에 출력하기 위해 System.out.println을 사용했다.
  • 연산 결과는 30으로, 이는 변수 value에 20 값을 할당하고 10을 더한 결과로 확인되었다.
  • 두 번째 예제로, 정수 타입의 변수 hour와 minute를 각각 3과 5로 초기화하고, 문자열과 더한 결과를 출력하기 위한 코드를 작성하였다.
  • 변수 totalMinutes는 hour에 60을 곱하고, minute 값을 더하여 최종 결과를 출력하였다.

3. 📝 변수와 스코프의 이해

  • 변수 선언: INT 타입의 x와 y 변수를 각각 3과 5로 초기화한 후, 출력하는 예제를 구현하였다.
  • 값 교환: 임시 변수인 temp를 사용하여 x와 y의 값을 교환하는 방법을 설명하였다.
  • 스코프 개념: 조건문 if 내부에서 선언된 변수 v2는 해당 블록을 벗어나면 유효하지 않다고 하여 에러를 발생시키는 예시를 보여주었다.
  • 변수 유효성: 선언된 변수는 생성된 블록 내부에서만 사용할 수 있으며, 블록 외부에서는 해당 변수를 찾을 수 없음을 강조하였다.
  • 주석 처리: 코드 이해를 돕기 위해 필요한 주석 처리를 언급하며 강의를 마무리하였다.

4. 🖥️ 자바의 리터럴 및 타입

  • 이진수는 0b 또는 0B로 시작하며, 뒤에는 0과 1로 이루어진 숫자를 사용해야 한다.
  • 8진수는 0으로 시작하고, 뒤에는 0부터 7 사이의 숫자를 사용해야 하며, 예를 들어 03은 유효하다.
  • 10진수는 일반 숫자로 나타내며, 특별한 접두사 없이 사용된다.
  • 16진수는 0x 또는 0X로 시작하고, 뒤에는 0부터 9까지의 숫자와 a부터 f까지의 알파벳을 사용할 수 있다.
  • 각 정수 타입의 값들은 출력하여 확인할 수 있으며, 모든 리터럴 형식을 알아야 올바른 값을 부여할 수 있다.
  • 바이트 타입은 -128부터 127까지의 값을 표현하며, 범위를 벗어나는 값은 컴파일 에러가 발생한다.
  • 롱 타입은 기본적으로 정수 리터럴로 인식되며, 큰 값을 사용할 경우 l 또는 L을 붙여 롱 리터럴로 정의해야 한다.
  • 캐릭터 타입은 작은 따옴표로 한 글자를 나타내며, 정수값으로 초기화할 경우 10진수나 16진수가 가능하다.
  • 이스케이프 시퀀스를 사용하는 예로는 역슬래시(\)를 이용하여 특수 문자를 출력할 수 있다.
  • 부동 소수점 타입에서 플로트형은 f를 붙여서 정의해야 하며, 더블형은 기본적으로 리터럴을 받을 수 있다.
  • 불린 타입은 true와 false의 두 가지 값을 가지며, 조건문에서 분기 처리를 위해 널리 사용된다.

4.1. 인티저 리터럴의 사용법

  • 이그잼 01이라는 예제를 위해 메인 메소드에서 INT 타입의 변수를 사용하도록 한다.
  • 이진수는 0과 b를 사용하여 표현하며, 그 뒤에 1과 0이 따라야 한다.
  • 인트 변수에서는 8진수를 사용하고 0으로 시작해야 하며, 0부터 7 사이의 숫자를 써야 한다.
  • 변수 값으로 08을 입력할 것이나, 8은 올바르지 않기에 다양한 숫자를 시도해야 한다.
  • 예로 3을 입력해보는 실습을 진행할 예정이다.

4.2. ️ 자바의 숫자 리터럴 및 표현 방식

  • 8진수는 0으로 시작하며, 뒤에는 0부터 7 사이의 숫자가 올 수 있다.
  • 10진수는 일반 숫자로 작성하며, 그냥 쓰면 된다.
  • 16진수는 0x 혹은 0X로 시작하며, 0부터 9까지의 숫자와 a부터 f까지의 알파벳을 사용할 수 있다.
  • 이진수는 0b 혹은 0 대문자 B로 시작하고, 그 뒤에는 0과 1만 올 수 있다.
  • 다양한 수 체계의 정수값을 출력하여 이들의 표현 방식을 확인할 수 있다.

4.3. ️ 자바 클래스 및 바이트 변수 생성

  • 자바 클래스를 생성하기 위해 우클릭 후 "new"를 선택하고 클래스명을 "바이트 이그잼플"로 지정한다.
  • 메인 메서드를 선언한 후, 변수를 선언하고 바이트 타입으로 -128 값을 입력한다.
  • 추가로 -30, 0, 30, 127, 128 값을 각각 다른 바이트 변수에 입력하는 과정이 이루어진다.
  • 바이트 타입은 -128부터 127까지의 값을 표현할 수 있으며, 이를 초과하는 128 값은 컴파일 에러를 발생시킨다.
  • 입력한 바이트 변수의 값이 잘 출력되는지 확인하는 절차가 진행된다.

4.4. 롱 타입의 이해와 활용

  • 롱 타입 변수를 생성하기 위해 "롱 타입 이그잼 03"과 "롱 이그잼플"을 만든다.
  • 롱 타입 변수 "C"에 인트 리터럴 값을 대입하는 예시가 소개된다.
  • 롱 변수 "롱 베리어블 2"에는 long 리터럴 값 20L이 대입되며, 이를 통해 롱 타입 변수와 리터럴의 관계를 짚는다.
  • 10진수 값을 롱 타입 변수에 잘못 대입하면 인티저로 인식되어 범위를 초과할 경우 컴파일 에러가 발생한다고 설명한다.
  • 연산 결과를 출력해 롱 타입의 동작을 확인하며 내용을 마무리한다.

4.5. 자바 캐릭터 타입 및 이스케이프 예제

  • 캐릭터 타입 변수는 작은 따옴표로 한 글자만 선언해야 하며, 이는 문자열 초기화를 나타낸다.
  • 10진수와 16진수 리터럴을 캐릭터 변수에 사용하여 변수를 초기화할 수 있으며, 각각 65와 0x41의 예가 있다.
  • 한글 캐릭터 역시 10진수 및 16진수 값을 사용하여 변수에 초기화 가능하고, 예를 들어 0xAC00로 저장할 수 있다.
  • 이스케이프 문자를 활용하여 출력 형식을 조정할 수 있으며, 예를 들어, 역슬래시(\)와 t로 탭 간격을 설정하거나, \n으로 줄바꿈을 수행한다.
  • float와 double 타입 간 리터럴 저장 시에는 정확한 타입 맞춤이 필요하며, 예를 들어, float형 변수에 값을 저장할 때는 f를 붙여야 한다.

5. 🔄 자바의 자동 및 강제 타입 변환 예제

  • 메인 메서드에서 자동 타입 변환을 통해 바이트 값을 인트 타입으로 변경하며, 10이라는 값을 성공적으로 출력했다.
  • 캐릭터 타입의 변환을 통해 인트 타입에 캐릭터 값이 적용되며, 유니코드 값을 출력하는 예시가 있었다.
  • 롱 타입에 인트 값을 넣는 것이 가능하며, 반대로는 강제 변환이 필요하다는 점을 강조하였다.
  • 플로트 값을 롱 타입의 변수에 대입하고, 제품값을 성공적으로 출력하는 과정을 통해 타입 변환의 유연성을 설명했다.
  • 바이트 연산에 대한 예제에서, 바이트간의 덧셈은 자동으로 인트로 반환되며, 이로 인해 에러가 발생할 가능성이 있는 점을 지적하였다.
  • 문자열 결합 시 소괄호의 우선순위에 따라 연산이 진행되며, 결과가 다르게 출력되는 예시를 보였다.
  • 프리미티브 타입의 변환을 위한 메소드 활용에 대해 다루며, parseInt, parseDouble, valueOf 등의 메소드 사용을 강조하였다.

5.1. 자동 타입 변환 예제

  • 메인 메서드에서 자동 타입 변환을 시연한다.
  • 바이트(byte) 타입의 변수인 바이트 밸류에 10을 초기화한다.
  • INT(int) 타입의 변수인 인트 밸류를 생성하고 바이트 밸류의 값을 가져와 자동 타입 변환한다.
  • 출력문을 통해 인트 밸류의 값을 표시하고 결과를 확인한다.
  • 바이트 타입이 인트 타입으로 변환되며, 10이 잘 출력되는 것을 통해 변환이 원활하게 이루어짐을 알 수 있다.

5.2. ️ 캐릭터와 정수 타입의 사용 예시

  • 캐릭터 타입에 특정 문자를 삽입하는 과정이 설명된다.
  • 민트 타입의 밸류를 활용하려는 의도가 있다.
  • 인트 타입의 값을 캐릭터 타입으로 변환하여 사용하는 방법이 논의된다.
  • 유니코드와 관련된 내용을 출력하는 방법에 대한 실습이 진행된다.
  • 롱 타입의 변수에 인트 타입 자료를 삽입할 수 있는 가능성이 언급되며, 인트 타입에 바이트와 캐릭터 타입을 넣는 것이 불가능함이 설명된다.

5.3. 플로트와 롱 타입의 관계 및 자동 타입 변환

  • 플로트 타입의 값을 생성하기 위해 롱 타입의 값을 대입해 볼 예정이다.
  • 플로트형 자료형에 롱 타입을 넣는 것이 가능하다는 것을 보여주고 있다.
  • 롱 타입의 값은 플로트형 자료형에 대입될 수 있으며, 이에 따른 예시로 100.5f 값을 들어 보여준다.
  • 더블형이 가장 큰 자료형으로, 그 다음으로 플로트형, 롱 타입 순으로 크기가 결정되며, 이러한 크기 덕분에 자동 타입 변환이 가능하다.
  • 출력 결과는 모두 정상적으로 작동하며, 자료형 변환의 원리를 실습으로 확인할 수 있었다.

5.4. ️ 변환 예제: 강제 타입 변환

  • 캐릭터형으로 강제 타입 변환을 통해 인트 타입의 값(44032)을 캐릭터로 변경하여 출력한다.
  • 롱 타입의 값(500)을 인트 타입으로 변환하기 위해 강제 타입 변환을 사용하고, 출력 결과를 확인한다.
  • 더블 값(3.14)을 인트 타입으로 변환할 때, 수수점 자리를 버리게 되어 출력 결과에 집합된 값을 확인한다.
  • 캐릭터와 롱 타입 변환 예제를 통해 값이 변환되는 과정과 결과를 명확히 보여준다.
  • 강제 타입 변환(casting)의 다양한 예를 살펴보며 변환의 중요성을 강조한다.

5.5. ️ 바이트 및 롱 타입 연산 예제

  • 바이트 타입 변수를 선언하고, 리터럴 값 10과 20을 더한 결과를 저장하여 인트 타입으로 출력한다.
  • 바이트 간 연산의 결과는 인트 타입으로 반환되며, 이를 바이트 타입으로 변환하려고 하면 에러가 발생한다.
  • 롱 타입 변수를 사용하여 10, 100, 1000을 각각 더한 결과를 롱 타입으로 저장하고 출력한다.
  • 바이트와 캐릭터 연산에서 인트 타입으로 반환되며, 이를 바이트나 캐릭터로 받으려 할 경우 에러가 발생한다.
  • 나누기 연산에서 자바는 결과를 정수로 반환하므로, 실수 값을 얻으려면 적절한 형변환이 필요하다.

5.6. 다양한 데이터 타입과 변환 방법

  • 인트 타입의 변수를 생성하고, 문자열 "10"을 parseInt 메소드를 통해 정수로 변환하여 에러 없이 처리된다.
  • 더블 타입 변수를 선언하고, 문자열 "3.14"를 parseDouble 메소드를 사용해 실수로 변환한다.
  • 불린 타입 변수에 문자열 "true"를 넣어 변환하며, 이를 위해 parseBoolean 메소드를 활용한다.
  • 각 데이터 타입 변환 후, 스트링으로 변환하기 위해 String.valueOf 메소드를 사용하고, 이러한 변환을 통해 변수 값들을 출력한다.
  • 다양한 프리미티브 타입 변수의 스트링 변환이 성공적으로 이루어져, 타입 변환의 예시를 보여준다.

Key term

JAVA

JAVA는 프로그래밍 언어의 하나로, 소프트웨어를 만들기 위해 사용합니다. JAVA는 객체 지향 언어로, 실제 세계의 사물을 컴퓨터 프로그램으로 표현하는 방법을 제공합니다.

문법

문법은 프로그래밍 언어를 사용할 때 지켜야 하는 규칙입니다. 문법이 올바르지 않으면 컴파일 에러가 발생하여 프로그램이 작동하지 않습니다.

변수

변수는 데이터를 저장할 수 있는 상자입니다. 예를 들어, 숫자나 문장을 저장하고 필요할 때 꺼내 쓸 수 있습니다.

리터럴

리터럴은 숫자, 문자열 등 변수에 담겨진 실제 값을 의미합니다. 예를 들어, 숫자 22, 문자열 'Hello'가 리터럴입니다.

변수 범위

변수 범위는 변수의 유효한 지역을 정의합니다. 특정한 코드 블록 안에서만 사용할 수 있는 변수도 있고, 전체 프로그램에서 사용 가능한 변수도 있습니다.

타입

타입은 변수가 가질 수 있는 데이터의 형태를 설명합니다. 예를 들어, 정수형, 실수형, 문자형 등이 있습니다.

타입 변환

타입 변환은 변수의 데이터 타입을 다른 타입으로 바꾸는 과정입니다. 예를 들어, 정수를 실수로 변환할 수 있습니다.

실습

실습은 배운 내용을 실제로 해보는 과정입니다. 코드를 직접 써보며, 이론적으로 배운 점을 잊지 않고 익힐 수 있습니다.

정수형

정수형은 소수점이 없는 정수를 저장하는 타입입니다. 예를 들어, 1, 10, -5 등이 정수형 데이터입니다.

출력

출력은 콘솔 창이나 화면에 결과를 보여주는 과정입니다. 이를 통해 프로그램이 제대로 작동하는지 확인할 수 있습니다.

산술 연산

산술 연산은 숫자에 대해 보통 하는 수학적 계산입니다. 덧셈, 뺄셈, 곱셈, 나눗셈 등이 있습니다.

조건문

조건문은 특정 조건을 검사하여 다르게 행동하도록 하는 코드입니다. 예를 들어, '만약 A라면 B를 하라'는 형태입니다.

자료형

자료형은 변수에 저장되는 값의 형태와 용량을 정의하는 키워드입니다. 예를 들어, 정수형, 실수형, 문자형이 있죠.

값 초기화

값 초기화는 변수를 선언할 때 초기 값을 설정하는 것을 말합니다. 예를 들어, 변수를 만들면서 10이나 'Hello' 같은 값을 주는 것입니다.

문자열

문자열은 문자로 이루어진 데이터입니다. 예를 들어, 'Hello', 'JAVA'가 문자열입니다.

배열

배열은 여러 개의 데이터를 하나의 이름으로 묶어 저장하는 자료구조입니다. 같은 타입의 여러 값을 효율적으로 관리할 수 있습니다.

클래스

클래스는 객체 지향 프로그래밍에서 설계도입니다. 객체를 만들기 위한 틀을 제공하며, 속성과 메소드를 가집니다.

객체 지향

객체 지향은 프로그램을 객체 단위로 구성하는 신념입니다. 객체는 속성과 행동을 가지며 현실 세계의 사물을 모방합니다.

컴파일 에러

컴파일 에러는 코드를 작성할 때 문법적 오류로 인해 프로그램이 실행되지 않는 상태입니다. 이러한 오류를 수정해야 합니다.

형변환

형변환은 특정 데이터 타입을 다른 데이터 타입으로 변경하는 과정입니다. 예를 들어, 정수를 실수로 바꾸는 것입니다.

문자열 결합

문자열 결합은 여러 문자열을 하나로 합치는 과정입니다. '+' 연산자를 사용합니다. 예를 들어, 'Hello' + ' World'는 'Hello World'가 됩니다.

제어문

제어문은 코드의 실행 흐름을 제어하는 문법으로, 조건문과 반복문이 이에 해당합니다.

기본 자료형

기본 자료형은 JAVA에서 제공하는 간단한 데이터 타입입니다. 예를 들어, 정수형(int), 실수형(float), 문자형(char) 등이 있습니다.

리터럴 타입

리터럴 타입은 변수에 직접 참조되는 값의 타입을 가리킵니다. 예를 들어, 숫자 리터럴, 문자열 리터럴 등이 있습니다.

프로그래밍

프로그래밍은 컴퓨터에게 특정 작업을 하도록 지시하는 과정입니다. 소프트웨어를 만들거나 자동화할 때 사용됩니다.
