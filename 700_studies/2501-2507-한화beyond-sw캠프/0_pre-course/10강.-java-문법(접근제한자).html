10강. JAVA 문법(접근제한자)

이번 강의는 JAVA의 접근 제한자에 대한 상세한 설명과 예제를 제공합니다. 패키지 구조와 클래스의 식별법, 그리고 접근 제한자를 통해 클래스와 멤버의 접근 범위를 설정하는 방법을 배우게 됩니다. 특히, public, protected, private, default 접근 제한자의 차이를 이해하고 활용하는 것이 중요합니다. 각 접근 제한자에 맞는 예제를 통해 실제 적용하는 방법을 익히게 됩니다. 이 강의를 통해 JAVA 프로그래밍에서 클래스 관리와 정보 은닉의 중요성을 깨닫게 될 것입니다.

1. 📦 패키지와 접근 제한자의 역할과 활용

  • 패키지는 물리적으로는 파일 시스템의 폴더이며, 이를 통해 클래스를 체계적으로 관리하고 구분할 수 있다.
  • 패키지는 클래스의 일부로 작용하며, 클래스의 이름이 동일할 경우 패키지 정보를 통해 유일한 식별자 역할을 한다.
  • 도메인의 유일성을 이용해 패키지 이름은 도메인의 역순으로 작성하며, 이는 식별성을 높이기 위한 방법이다.
  • 패키지 선언은 클래스 작성 시 맨 윗줄에 위치하며, 패키지 이름은 관례적으로 모두 소문자로 작성되어야 한다.
  • 서로 다른 패키지의 클래스를 사용하기 위해서는 import문을 이용해야 하며, 와일드카드를 통해 하위 패키지의 모든 것을 가져올 수 있다.

1.1. 패키지와 접근 제한자 이해하기

  • 패키지는 파일 시스템의 폴더 구조를 나타내며, 클래스 관리에 도움을 준다.
  • 동일한 클래스 이름을 가진 경우, 패키지 위치를 통해 서로 구분할 수 있다.
  • 클래스의 전체 이름은 패키지와 클래스를 합쳐서 식별하며, 이는 중요하다.
  • 접근 제한자는 클래스의 멤버에 대한 접근 제약을 두는 역할을 한다.
  • 패키지 이름은 보통 도메인 이름의 역순으로 작성하여 유일성을 제공한다.

1.2. 자바 패키지 명명 규칙

  • 자바 패키지를 명명할 때 숫자로 시작할 수 없으며, 언더스코어와 달러 기호를 제외한 특수기호를 사용할 수 없다.
  • 자바로 시작하는 패키지는 자바 표준 API에서만 사용되므로, 이러한 패키지 사용을 지향해야 한다.
  • 패키지 명은 모두 소문자로 작성하는 것이 관례이다.
  • 예제를 통해 이러한 규칙을 살펴보도록 한다.
  • 프로젝트 설정을 위해 인텔리제이를 실행하고, 새로운 패키지를 생성하여 명명을 진행한다.

1.3. 패키지와 클래스 구조 이해하기

  • 패키지 구조 내에 여러 클래스를 작성하고, 예를 들어 '한국' 패키지에 '타이어'와 '스노우 타이어' 클래스를 선언한다.
  • '현대' 패키지 안에 '엔진' 클래스를 추가하고, 다른 패키지의 클래스를 사용할 때 import 선언이 필요함을 강조한다.
  • '금호' 패키지 안에 '타이어' 및 '빅 타이어' 클래스를 추가하여 다양한 패키지 구조를 활용하는 방법을 보여준다.
  • 동일한 클래스 이름의 경우, 패키지 정보를 명시하는 것이 중요하며, 이를 통해 클래스 간의 구분이 가능함을 설명한다.
  • 패키지와 클래스 구조의 식별 방법을 간단한 예제로 정리하여 이해를 돕는다.

1.4. 자바 import 문과 패키지 활용

  • import문은 다른 패키지에 소속된 클래스나 인터페이스를 사용할 때 반드시 필요하다.
  • 이를 통해 사용하고자 하는 클래스 또는 인터페이스를 가져와서 사용할 수 있다.
  • 와일드카드를 사용하면 하위 패키지의 모든 내용을 가져올 수 있는 기능을 제공한다.
  • 하위 패키지는 별도로 import를 해야 하며, 동일한 이름의 클래스가 있을 경우 클래스 전체 이름을 기술해야 한다.
  • 패키지의 개략적인 구조와 활용 방법에 대해 검토하였다.

2. 🔒 접근 제한자 종류 및 사용법

  • 접근 제한자는 4가지의 종류가 있으며, 이는 클래스와 인터페이스의 멤버 접근을 제한하는 역할을 한다.
  • public 접근 제한자는 외부 클래스가 자유롭게 접근할 수 있도록 허용한다.
  • 프로텍티드는 동일 패키지 내의 클래스 또는 해당 클래스를 상속받은 자식 클래스에서 접근 가능하다.
  • private 접근 제한자는 클래스 내부에서만 접근이 가능하며 외부에서는 사용할 수 없도록 제한한다.
  • 디폴트 접근 제한자는 같은 패키지의 클래스에서만 접근할 수 있으며, 프로텍티드와의 차이는 자식 클래스에서도 접근 가능하다는 점이다.

3. 📦 접근 제한자와 클래스 간의 관계

  • 패키지 1 내에서 'a' 클래스를 만들고, 같은 패키지 내에 있는 'b' 클래스에서 'a' 클래스를 필드로 선언하였으나, 다른 패키지에서 접근할 수 없는 것으로 나타났다.
  • 'a' 클래스는 디폴트 접근 제한자를 가지고 있어 다른 패키지에서 접근할 수 없으며, 'b' 클래스는 public 접근 제한자를 사용하고 있어 접근이 가능하다.
  • 생성자 또한 접근 제한자를 적용할 수 있으며, 예시에서는 public, 디폴트, private 접근 제한자를 가진 생성자들이 생성되었고, 같은 패키지 내에서는 public과 디폴트 접근 제한자를 사용한 생성자만 접근이 가능하다.
  • 필드와 메소드에 대해 접근 제한자도 똑같은 원칙이 적용되며, 각 접근 제한자별로 접근 가능 여부가 다르게 나타난다.
  • private 접근 제한자를 통해 객체의 무결성을 보호할 수 있으며, 이를 위해 Getter와 Setter 메소드를 사용하는 것이 권장된다.

3.1. 패키지 및 클래스 생성 과정

  • 'a' 클래스를 패키지 1 내에서 만들기로 설정되어 있다.
  • 'a' 클래스와 별개로 내용을 작성하지 않기로 하였다.
  • 'b' 클래스를 동일한 패키지에서 만들어 'a' 클래스를 사용할 예정이다.
  • 'b' 클래스 안에서 'a' 클래스에 대한 필드값을 선언할 계획이다.

3.2. 접근 제한자의 동작 원리

  • 다른 패키지에서 'a'라는 클래스의 접근 제한자를 public에서 제거해도, 같은 패키지 내에서는 접근이 가능하다.
  • c라는 클래스는 다른 패키지에 위치하기 때문에 'a' 클래스에 접근할 수 없으며, 이는 import를 수행해도 마찬가지이다.
  • 디폴트 접근 제한자는 같은 패키지에 있을 경우에만 사용 가능하므로, 'a' 클래스는 외부에서 접근 불가하다.
  • 반면 'b' 클래스는 public 접근 제한자로 인해 다른 패키지에서도 접근할 수 있다.
  • 접근 제한자는 클래스뿐만 아니라 생성자에도 적용되어 객체 생성 시 접근 제한을 설정할 수 있다.

3.3. ️ 생성자와 접근 제한자 이해하기

  • 예제 03번에서는 패키지 안에 클래스 a를 만들고 필드를 선언하는 것으로 시작한다.
  • 생성자는 public a와 디폴트 생성자, private 생성자 세 가지를 만들며, 각각 다른 접근 제한자를 가질 수 있다.
  • 클래스 내부에서는 모든 생성자가 접근 가능하므로, 객체를 생성할 때 제한 없이 이용할 수 있다.
  • 같은 패키지에 있는 클래스에서 public과 디폴트 생성자는 접근 가능하지만, private 생성자는 접근할 수 없다.
  • 다른 패키지에서 클래스 A를 필드로 사용할 때, public 생성자는 문제없이 접근 가능하며, 디폴트 생성자는 접근 불가능하고 private 생성자 역시 접근할 수 없다.

3.4. 필드와 메소드의 접근 제한자

  • 접근 제한자는 필드와 메소드를 제어하며, public, 디폴트, private으로 구분된다.
  • 같은 패키지 내에서는 public과 디폴트 필드에 접근할 수 있지만, private 필드는 접근할 수 없다.
  • 생성자 내에서 정의된 필드와 메소드들은 접근 제한자에 상관없이 사용할 수 있으며, 이를 통해 내부적으로 자유롭게 조작할 수 있다.
  • 다른 패키지에서 접근 시, public 필드는 접근 가능하지만, 디폴트와 private 필드는 접근할 수 없다.
  • 클래스와 메소드의 접근 제한자를 이해하면, 코드 작성 시 적절한 범위 내에서 필드와 메소드에 접근할 수 있게 된다.

3.5. 객체 필드 접근 제한

  • 객체의 무결성은 외부에서 필드에 마음대로 접근할 경우 깨질 수 있다.
  • 필드에 대해 접근 제한자를 private으로 선언하여 보호할 수 있다.
  • 특정 메소드를 통해서만 필드를 수정할 수 있도록 설정할 수 있다.
  • 자바에서는 Getter와 Setter를 통해 필드 접근을 표준적으로 권장하고 있다.
  • 이전 예제에서 은연중에 만든 Setter를 구체적으로 다룰 것이다.

4. 🚗 카 클래스와 Getter, Setter 메소드 이해하기

  • 카 클래스는 private 필드로 스피드와 스탑을 가지며, 값을 외부로부터 받아오는 Setter 메소드를 제공한다.
  • Getter 메소드는 필드 값을 반환하는 목적을 가지고 있으며, 반환 타입이 존재한다.
  • Setter 메소드는 전달받은 값을 필드에 저장하며, 특정 값의 범위 체크를 통해 유효성을 검증할 수 있다.
  • 예제에서 마이너스 속도를 입력했을 경우 속도는 0으로 설정되며 이후 스탑의 값에 따라 속도가 조정된다.
  • Getter 메소드의 경우, 불린 타입의 필드는 'is'로 시작하는 이름 짓기 규칙을 따른다.

5. 📦 접근 제한자와 메소드 명명 규칙

  • 패키지 선언은 패키지라는 예약어로 시작하며, 해당 패키지의 정의를 포함한다.
  • import를 통해 다른 패키지의 클래스나 인터페이스를 가져오는 방법을 설명하였다.
  • 접근 제한자는 public부터 private까지 범위가 줄어들며, public은 전역에서 접근 가능하다.
  • 프로텍티드는 같은 패키지나 자식 클래스에서 접근할 수 있으며, 디폴트는 같은 패키지 내에서만 접근 가능하다.
  • private은 클래스 내부에서만 사용할 수 있으며, 필드 값에 대한 접근은 Getter, Setter 메소드를 통해 간접적으로 이루어진다.
  • 메소드 명명 규칙으로는 불리언 타입의 필드에는 '이즈'를, 다른 타입에는 '계시'를 접두사로 사용하고, 각각 카멜케이스로 작성해야 한다.

Key term

JAVA

**자바(JAVA)**는 객체 지향 프로그래밍 언어로, 다양한 플랫폼에서 실행 가능하도록 설계되었습니다. 쉽게 말해, 자바로 작성한 프로그램은 어떤 컴퓨터에서도 같은 방식으로 동작할 수 있습니다. 또한, 자바는 많은 프레임워크와 라이브러리와 함께 사용되어, 소프트웨어 개발에 널리 활용되고 있습니다.

접근 제한자

접근 제한자는 클래스와 그 안의 멤버(필드와 메소드)에 대한 접근 범위를 결정하는 키워드입니다. 쉽게 말해, 어떤 정보가 누구에게 보일 수 있는지 정해 주는 역할을 합니다. 자바에서 대표적인 접근 제한자는 public, protected, private, default입니다.

패키지

패키지는 관련된 클래스들을 그룹으로 묶는 방법입니다. 예를 들어, 같은 종류의 클래스들을 폴더에 모아두는 것과 같습니다. 이렇게 그룹화함으로써 코드의 관리가 쉽고, 이름이 같은 클래스도 패키지를 통해 구분할 수 있습니다.

클래스

클래스는 객체 지향 프로그래밍의 기본 단위로, 객체를 정의하는 청사진과 같습니다. 실생활의 사물처럼 여러 특성과 기능을 가진 객체를 만들기 위해 클래스에서 정의한 내용대로 객체를 생성합니다.

식별자

식별자는 프로그램의 특정 요소를 구별하는 이름입니다. 마치 사람을 부를 때 사용하는 이름과 같습니다. 자바에서는 클래스, 변수, 메소드 등을 구분하기 위해 각기 다른 식별자를 사용합니다.

public

public은 가장 넓은 접근 범위를 가진 접근 제한자입니다. public으로 선언된 클래스나 멤버는 모든 클래스에서 접근이 가능합니다. 즉, 누구나 사용해도 괜찮다는 것을 의미합니다.

protected

protected는 같은 패키지 내의 클래스나, 이 클래스를 상속받은 자식 클래스에서 접근할 수 있는 접근 제한자입니다. 외부로서는 볼 수 없지만, 상속받은 클래스에서는 사용할 수 있다는 점이 특징입니다.

private

private는 가장 제한적인 접근 제한자입니다. private으로 선언된 멤버는 그 클래스 내부에서만 접근할 수 있습니다. 즉, 외부에서는 전혀 사용할 수 없게 막아 놓는 것입니다.

default

default는 접근 제한자를 명시하지 않을 경우 적용되는 기본값입니다. 같은 패키지에 있는 클래스들끼리만 접근할 수 있고, 외부 패키지에서는 접근할 수 없습니다.

import

import는 다른 패키지에 있는 클래스를 현재의 클래스에서 사용하기 위해 가져오는 방법입니다. 이는 다른 패키지에 포함된 클래스들을 사용하고 싶을 때 필요합니다. 쉽게 말해, 다른 책에서 정보를 가져오는 것과 유사합니다.

예제

예제는 이론을 이해하기 쉽게 구체적인 사례를 들어 보여주는 것입니다. 자바에서 접근 제한자나 클래스를 설명할 때, 실제로 코드로 작성된 경우가 예제입니다.

객체 지향 프로그래밍

객체 지향 프로그래밍은 프로그램을 '객체'라는 단위로 나누어 설계하는 방법입니다. 각 객체는 고유의 속성과 기능을 가지고 서로 상호작용을 합니다. 이렇게 함으로써 프로그램을 더 이해하기 쉽고 관리하기 간편하게 만들 수 있습니다.

내부 클래스

내부 클래스는 다른 클래스 안에 정의된 클래스를 말합니다. 이는 관련된 작업을 더 간편하게 처리할 수 있도록 도와줍니다. 내부 클래스는 외부 클래스의 멤버에 쉽게 접근할 수 있는 장점이 있습니다.

개체 무결성

개체 무결성은 객체의 상태를 안전하게 유지하는 것을 의미합니다. 즉, 외부에서 객체의 중요한 속성을 마음대로 바꾸지 못하게 하여, 객체의 상태가 항상 유효하도록 보장하는 것입니다.

getter

getter는 객체의 필드 값을 외부에서 안전하게 가져오는 메소드입니다. 예를 들어, 비공개(private) 속성을 읽어올 수 있도록 하는 엑세서 메소드입니다.

setter

setter는 객체의 필드 값을 외부에서 설정할 수 있게 해주는 메소드입니다. 마치 외부에서 어떤 값을 설정할 필요가 있을 때 쓰는 도구입니다. 이 메소드를 통해 유효성 검사를 할 수도 있습니다.

정적 메소드

정적 메소드는 객체를 생성하지 않고도 사용할 수 있는 메소드입니다. 클래스에 속하지만, 인스턴스(객체)와는 독립적으로 작동하여 여러 객체가 공유하는 기능을 제공할 수 있습니다.
