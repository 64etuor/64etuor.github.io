12강. JAVA문법(예외처리) |

이 강의에서는 자바의 예외 처리에 대해 자세히 설명합니다. 예외와 에러의 차이를 이해하고, 예외가 발생했을 때 프로그램이 종료되지 않도록 처리하는 방법을 배웁니다. try-catch-finally 구조를 통해 여러 종류의 예외를 효율적으로 다루는 기법을 소개하며, 예외클래스를 활용한 예외관리 방식을 설명합니다. 또한 예외를 메소드 호출 시 떠넘기는 방법까지 다루어, 복잡한 프로그램에서도 안정적으로 동작할 수 있도록 합니다. 이 내용을 통해 개발자는 예외처리의 중요성과 실제 사용법을 익히게 됩니다.

1. 📊 예외처리 개요

  • 자바에서 에러는 하드웨어 고장으로 인한 응용 프로그램 실행 오류를 의미하며, 예외는 프로그램 내의 사용자 또는 개발자의 잘못으로 발생하는 오류이다.
  • 예외를 적절히 처리하지 않으면 프로그램이 예기치 않게 종료될 수 있으므로, 이에 대한 핸들링이 필요하다.
  • 자바는 예외발생 가능성이 있는 코드를 컴파일할 때 예외처리 여부를 확인하며, 처리하지 않을 경우 컴파일 에러가 발생한다.

2. 🛠️ 자바의 예외 처리 개념

  • 자바에서 예외처리를 위해서는 예외의 종류를 알아야 하며, 이를 일반 예외와 실행 예외로 나눌 수 있다.
  • 실행 예외는 컴파일러가 파악하기 어려운 예외로, 프로그램 실행 중 발생하는 예외를 의미하며, 예로는 NullPointerException과 IndexOutOfBoundsException이 있다.
  • 모든 예외클래스는 Exception 클래스를 상속받아 관리되며, JVM이 예외가 발생할 경우 해당 예외클래스의 객체를 생성하게 된다.
  • NumberFormatException은 문자열을 숫자로 변환할 수 없는 경우에 발생하며, 잘못된 사용자 입력이 원인이다.
  • 자바에서 발생한 예외는 사용자의 잘못된 조작이나 개발자의 오류로 인해 발생하며, 이러한 예외는 객체로 생성되어 프로그램에서 활용된다.

3. 🛠️ JAVA 예외처리 방법

  • 프로그램에서 예외가 발생하더라도 종료되지 않고 별도로 처리하는 방법을 설명한다.
  • 런타임 익셉션과 일반적인 Exception의 개념이 소개된다.
  • 예외처리는 try-catch-finally 블록을 통해 이루어진다.
  • try블록에는 예외가 발생할 수 있는 코드가 작성되며, 실행 중 예외발생 시 catch블록이 활성화된다.
  • finally 블록은 예외발생 여부와 관계없이 무조건 실행된다.

4. ⚠️ 예외 처리 및 try-catch 구조

  • 예외처리를 위해, try 블록 안에 예외가 발생할 수 있는 코드를 작성하는 것이 일반적이다.
  • 예외발생 시 낱낱이 에러 메시지를 출력하여 프로그램 종료전의 상황을 명확히 알린다.
  • 여러 종류의 예외가 발생할 수 있으며, 이때는 우선순위를 정해 위에서 아래로 예외를 처리해야 한다.
  • 상위 예외 클래스는 하위 예외클래스보다 아래에 위치해야 하며, 그렇지 않으면 하위 예외가 처리되지 않을 수 있다.
  • 예외를 떠넘기는 방법으로는 throws 키워드를 사용하여 해당 예외를 메소드 호출에 전달하는 방식이 있다.

4.1. ️ 자바 예외 처리 예제 소개

  • 새로운 프로젝트 "셉션"을 만들어 예제를 통해 예외 처리를 살펴볼 예정이다.
  • " catch finally이그잼플리"라는 자바클래스를 생성하고, 메인 메소드를 선언한 후 예외처리를 위한 " try" 블록을 사용할 것이다.
  • 예외가 발생할 수 있는 코드를 포함한 클래스에서 "class not found exception"을 매개변수로 받아 처리할 예정이며, 이 상황에서 "클래스가 존재하지 않습니다"라는 메시지를 출력할 것이다.
  • 프로그램이 종료될 때는 "프로그램이 종료됩니다"라는 메시지가 출력되도록 설정할 것이다.
  • 실행 결과, 특정 클래스가 존재하지 않을 때 나타나는 에러메시지를 확인할 수 있다.

4.2. ️ 예외 처리 방법과 예제

  • try-catch 문 내에서 다양한 예외가 발생할 수 있으며, 발생한 예외에 따라 우선순위에 따라 예외블록이 실행된다.
  • 예를 들어, " 인덱스 아웃 오브 바운드 익셉션"이 발생하면 해당 블록만 실행되고 프로그램이 종료된다.
  • 인티저 값을 파싱하고 결과를 출력하기 위한 메인 메소드에서, 적절한 예외처리 코드가 포함된다.
  • 넘버 포맷 익셉션이 발생할 경우, 해당 catch블록이 실행되며, 이는 문자열을 숫자로 변환할 수 없기 때문이다.
  • 정상적인 값 10과 20을 입력하였을 때, 프로그램이 정상적으로 동작하는 것을 확인할 수 있다.

4.3. ️ 다중 catch문 작성 시 예외 클래스 순서

  • 다중 catch문 블록 작성 시 상위 예외클래스는 하위 예외클래스보다 아래에 위치해야 한다.
  • 상위 예외클래스의 catch문이 위에 작성되면 하위 예외클래스는 실행될 수 없다.
  • 상위 예외클래스가 위에 있을 경우 모든 예외가 상위 catch문에서 처리되므로 하위 catch문에 도달하지 않게 된다.
  • catch블록을 작성할 때는 상위와 하위 예외의 구조를 고려해야 한다.

4.4. ️ JAVA 예외 처리 및 떠넘기기

  • 예외를 처리할 수 있는 방법으로 try 문을 사용하거나, throws 키워드를 통해 메소드에서 예외를 떠넘길 수 있다.
  • 모든 예외를 떠넘기고자 할 경우, 최상위 부모 클래스인 Exception을 사용하면 간단하게 처리 가능하다.
  • throws가 붙어 있는 메소드는 반드시 try 블록 내에서 호출해야 한다.
  • 예외를 떠넘길 수 있는 메소드를 호출하고, 다시 예외를 떠넘길 수 있다는 점도 존재한다.
  • 최종적으로 예외는 JVM에 의해 처리되며, 내용은 콘솔에 출력된다.
  • 메인 메소드에서 throws 키워드를 활용하여 예외를 처리하는 것은 좋은 방법이 아니라고 여겨진다.

4.5. ️ 예외 처리 방법

  • 프로그램의 예외내용은 사용자가 다 볼 수 있기 때문에 최종 처리의 중요성이 있다.
  • 따라서, 메인 메서드까지는 반드시 try catch 문을 사용하여 예외를 처리하는 것이 바람직하다.

5. 🔍 예외 처리 개념 정리

  • 예외처리는 프로그램이 예외가 발생할 때 갑작스럽게 종료되는 것을 막기 위해 존재한다.
  • try, catch, finally 블록을 사용하여 다양한 예외상황에 대해 분기 처리가 가능하다.
  • 다중 catch문을 활용하여 여러 예외를 동시에 처리할 수 있다.
  • 스로우스 키워드를 사용하여 예외처리를 호출하는 부분으로 떠넘길 수 있다.
  • 예외처리의 개념은 비교적 간단하므로 확인 문제를 통해 연습하는 것이 추천된다.

Key term

자바

자바는 프로그래밍 언어 중 하나로, 다양한 운영 체제에서 실행될 수 있도록 설계되었습니다. 웹 개발, 모바일 앱, 게임 등 여러 분야에서 사용되어요. 자바는 여러 사람과 기계가 쉽게 이해할 수 있는 언어입니다.

예외처리

예외처리는 프로그램이 실행 도중 발생할 수 있는 오류나 예외 상황을 처리하는 방법입니다. 즉, 프로그램이 갑자기 중단되지 않고 정상적으로 계속 동작하도록 만들어 주는 것이죠. 예외를 잘 처리하는 것은 좋은 프로그램을 만드는 데 중요해요.

에러

에러는 프로그램이 외부 요인으로 인해 발생하는 심각한 문제입니다. 주로 하드웨어나 운영 체제의 문제로 인해 발생하고, 일반적으로 사용자나 개발자가 해결할 수 없습니다. 예를 들어, 컴퓨터가 고장나면 프로그램도 멈출 수 있어요.

예외

예외는 프로그램 내부에서 발생하는 오류입니다. 일반적으로 개발자의 실수나 잘못된 입력 때문에 발생해요. 프로그램이 예외를 만나면 정리된 방식으로 처리해야 합니다.

try

try는 예외가 발생할 가능성이 있는 코드 블록을 감싸는 부분입니다. 이 안에서 오류가 발생하면, 그에 맞는 처리를 할 수 있는 기능을 제공합니다. 비유하자면, '위험할 수 있는 도로'에 들어가는 것과 같아요.

catch

catch는 try 블록에서 발생한 예외를 잡아내는 역할을 합니다. 예외를 처리하기 위한 코드를 여기에 작성하는 것이죠. 예를 들어, 오류가 났을 때 '이렇게 해결할 거야'라고 지침을 주는 것과 비슷해요.

finally

finally는 try와 catch 블록을 묶고, 무조건 실행되는 코드를 작성할 수 있게 해줍니다. 예를 들어, 어떤 일이 있든지 항상 결과를 알려주는 스텝을 마련한다고 생각하면 쉬워요.

널 포인터 익셉션

널 포인터 익셉션은 객체가 없을 때 발생하는 예외입니다. 즉, 어떤 변수에 접근하려고 했는데, 그 변수가 아무것도 가리키고 있지 않을 때 생겨요. 마치 빈 주머니를 만져보고 '어? 아무것도 없네?' 하는 것과 같죠.

인덱스 아웃 오브 바운드 익셉션

인덱스 아웃 오브 바운드 익셉션은 배열에서 지정한 범위를 벗어나 접근하려고 할 때 발생하는 예외입니다. 예를 들어, 5개의 공이 들어 있는 상자에 6번째 공을 넣으려 할 때 생기는 문제가 비슷해요.

넘버 포맷 익셉션

넘버 포맷 익셉션은 문자열을 숫자로 변환할 때 문제가 생기는 경우입니다. 사용자가 '10A'라고 입력해서 숫자로 바꿀 수 없을 때 이 예외가 발생하죠. 마치 '10을 숫자로 읽어줘' 했더니 'A'가 있어 당황하는 상황과 같아요.

실행 예외

실행 예외는 프로그램이 실행 중에만 발견되는 예외를 의미합니다. 이 오류는 컴파일러가 알 수 없는 경우가 많아서, 코드가 실행될 때 까지 문제가 나타나지 않아요.

일반 예외

일반 예외는 프로그램이 컴파일 될 때 발견될 수 있는 오류입니다. 예를 들어, 문법 오류는 일반 예외로 분류되죠. 코드가 완전한지를 체크하는 단계에서 문제가 드러나는 것입니다.

예외 클래스

예외 클래스는 자바에서 예외를 관리하는 클래스입니다. 모든 예외는 Exception이라는 기본 클래스를 상속받아 만들어져요. 즉, 예외에 대한 정보를 담고 있는 특별한 데이터 구조와 같아요.

JVM

JVM은 자바 프로그램을 실행하는 환경입니다. 자바의 코드가 실제로 실행되는 곳으로, JVM이 예외가 발생하면 이를 처리하는 역할을 합니다. 이것은 자바의 자동으로 패키징된 실행 장치라고 생각하면 됩니다.

예외 객체

예외 객체는 발생한 예외에 대한 정보를 담고 있는 데이터입니다. 예를 들어, 어떤 오류가 발생했는지, 어떤 원인인지 알려주는 정보를 포함하고 있어요. 마치 사건 발생 시 경찰이 상황 보고서를 만드는 것과 같죠.

예외 처리 블록

예외 처리 블록은 try, catch, finally를 통해 예외를 처리하는 코드의 모음입니다. 이 블록을 사용하여 프로그램의 안정성을 높이고 오류 발생 시 대처할 수 있게 해 줍니다.

다중 catch문

다중 catch문은 여러 종류의 예외를 한 번에 처리하기 위해 사용하는 구조입니다. 여러 상황이 있을 때, 각각을 따로 처리하지 않고 한 곳에서 처리할 수 있도록 해 주죠.

throws

throws는 메소드에서 발생할 수 있는 예외를 전달하는 키워드입니다. 이것을 사용하면 이 메소드를 호출하는 곳에서 이 예외를 처리해야 하도록 강제할 수 있어요. 마치 소포를 누군가에게 전달하는 것과 같다고 할 수 있습니다.

예외 떠넘기기

예외 떠넘기기는 발생한 예외를 위쪽으로 전달하는 과정입니다. 즉, 현재 메소드에서 처리하지 않고 상위 메소드로 이관하는 것이죠. 전화가 계속 연결되어 위로 가는 것처럼 생각할 수 있어요.

정상 실행

정상 실행은 프로그램이 문제없이 기대한 대로 작동하는 상태입니다. 모든 코드가 예상과 맞게 돌아가서 사용자가 목적한 결과를 얻는 것이죠.

프로그램 종료

프로그램 종료는 프로그램이 정상적이지 않게 끝나거나 사용자가 종료한 상태를 의미합니다. 예외가 발생했을 때, 적절히 처리하지 않으면 프로그램이 예상하지 못하고 종료될 수 있어요.
