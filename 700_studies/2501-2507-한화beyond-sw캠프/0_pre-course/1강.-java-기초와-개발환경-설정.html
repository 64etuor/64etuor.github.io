1강. JAVA 기초와 개발환경 설정

이 강의는 자바에 대한 기초 및 개발 환경 설정을 다룹니다. 자바언어의 특징과 복잡한 명령을 어떻게 이해하고 처리하는지를 설명하며, 소스 코드 작성 및 컴파일과정에 대해 배웁니다. 강의 중간에는 자바개발에 필요한 JDK 설치 방법과 함께 IDE설정도 안내합니다. 초보 개발자에게 자바언어를 이해하고 환경을 세팅하는 데 실질적인 도움이 되는 내용입니다. 전반적으로 자바프로그래밍을 시작하기 위한 기초 지식 와 환경 설정을 설명하는 강의입니다.

1. 📚 자바 프로그래밍 언어 개요

  • 자바는 프로그래밍 언어중 하나로, 기계어보다 사람이 작성하기 쉬운 언어 체계이다.
  • 이 수업에서는 '혼자 공부하는 자바'라는 교재를 기반으로 자바의 특징과 환경 설정에 대해 학습한다.
  • 컴퓨터는 이진코드로 동작하며, 이를 위해 프로그래밍 언어가 필요하다. 코딩은 소스 코드를 작성하는 과정이다.

2. 🖥️ 자바 소스 코드의 이해

  • 자바에서 작성된 파일을 소스 코드라 하며, 이는 특정 프로그래밍 언어의 문법에 따라 텍스트 파일로 작성된다.
  • 소스 코드는 파일 확장자를 .java로 하여 저장되며, 텍스트 편집기 종류와 상관없이 작성할 수 있다.
  • 작성된 소스 코드는 컴퓨터가 이해할 수 있는 형태로 변환되어야 하며, 이 과정에서 추가적인 작업이 필요하다.
  • 컴퓨터는 최종적으로 소스 코드를 이진 코드인 0과 1로 처리하게 되며, 이를 통해 프로그램이 실행된다.

3. 🖥️ 자바의 컴파일러와 인터프리터 이해하기

  • 자바는 컴파일러 방식으로 작동하며, 작성된 소스 코드를 바이트 코드로 변환하여 JVM에서 실행된다.
  • 컴파일은 사람이 작성한 텍스트 파일을 컴퓨터가 이해할 수 있도록 번역하는 작업으로, 이 과정을 수행하는 것이 컴파일러이다.
  • 인터프리터 방식은 즉석에서 해석을 하여 실행하게 되는데, 이와 대비되는 컴파일러방식은 실행 시점에 번역 과정을 생략하므로 속도적인 이점이 있다.
  • 인터프리터는 코드 실행 결과를 즉각적으로 볼 수 있어 생산성에서 장점이 있음을 나타낸다.
  • 자바언어의 특성상, 한 번의 컴파일만으로 계속 실행이 가능하여 번역 단계가 필요 없음을 알 수 있다.

4. ☕ 자바 언어의 특징과 구조

  • 자바는 운영체제와 독립적으로 동작할 수 있는 특징을 가지며, 이는 Java Virtual Machine( JVM) 덕분이다.
  • 자바는 객체지향 프로그래밍 언어로, 코드는 반드시 클래스안에 작성되며 이를 통해 객체를 생성한다.
  • 클래스는 객체의 특징을 정의하는 필드와 객체의 동작을 설명하는 메서드로 구성된다.
  • JVM은 자바애플리케이션의 동작 플랫폼일 뿐만 아니라, 메모리 관리를 자동으로 해준다, 그래서 프로그래머가 메모리 관리에 대한 걱정을 덜 수 있다.
  • 자바생태계는 풍부한 오픈소스 라이브러리가 존재하여 개발자들이 필요로 하는 기능을 무료로 사용할 수 있는 장점을 갖는다.

5. 🚀 자바 개발 환경 설정 및 JDK 설치

  • 자바애플리케이션 개발을 위해서는 JDK(Java Development Kit)를 설치해야 하며, JDK는 각 기업이나 단체들이 자바 스탠다드 에디션(Java SE)의 표준에 맞춰 개발한다.
  • JDK는 다양한 버전이 존재하며, 버전 선택 시 LTS(Long-Term Support)를 고려하는 것이 중요하다. LTS는 장기간 지원 및 업데이트를 보장하는 버전이다.
  • JDK17 버전을 기준으로 설치 방법이 설명되며, 윈도우 운영체제에서 설치하는 절차가 안내된다.
  • JDK설치 후에는 설치 경로를 환경 변수에 추가해야 하며, ' 자바홈' 변수와 '패스' 변수에 경로를 설정한다.
  • 인텔리제이와 같은 통합 개발 환경( IDE)을 통해 효율적으로 자바개발을 진행할 수 있으며, IDE는 소스 코드편집과 여러 개발 도구를 통합해 제공한다.

5.1. ️ 자바 개발 환경 설정을 위한 JDK

  • 자바애플리케이션 개발을 위해서는 자바 디벨롭먼트 킷(JDK)을 설치해야 한다.
  • JDK는 메이저 버전의 표준인 자바 스탠다드 에디션(SE)에 기반하여 개발되며, 각 회사는 이에 맞춰 자신들의 jdk를 제작한다.
  • JDK는 다양한 형태로 제공되며, 각 개발 주체에 따라 라이센스가 다를 수 있다.
  • 무료로 사용할 수 있는 오픈 JDK를 활용해 자바개발 환경을 설정할 계획이다.
  • JDK의 선택 시 롱텀 서포트(LTS) 버전을 고려하는 것이 중요하며, 이는 오랜 기간 업데이트와 지원을 제공한다.

5.2. JAVA JDK 17 버전 소개

  • JDK 17 버전은 최신 버전 중 하나로, 최신이라고는 할 수 없지만 여전히 상당히 최근에 출시된 버전이다.
  • 이 버전은 사용해볼 가치가 있으며, JAVA의 기초와 개발환경 설정을 위해 선택되었다.

5.3. JDK 설치 과정 안내

  • 64비트 윈도우 기준으로 JDK설치 방법을 설명한다.
  • 자신의 컴퓨터 버전 확인을 위해 시작 버튼을 우클릭하고 시스템을 선택하여 사양을 확인할 수 있다.
  • 다운로드를 위해 jdk.j.net/s/l/archive에서 17 버전을 선택하고, 윈도우를 기반으로 설치를 진행한다고 언급한다.
  • 다운로드 후 압축을 해제해서 C 드라이브에 폴더를 옮기는 과정을 수행한다.
  • 설치 과정이 완료되면 윈도우 기준으로 JDK설치 방법을 마무리한다.

5.4. JAVA 환경 변수 설정 과정

  • 환경 변수에 JDK폴더 경로를 추가해야 하며, 이는 시스템 설정에서 이루어진다.
  • 시스템 설정에서 "고급 시스템 설정"을 선택하고, " 환경 변수" 옵션을 클릭해야 한다.
  • 자바 홈이라는 이름으로 새 시스템 변수를 만들어야 하며, 이 변수의 값은 JDK의 경로가 된다.
  • 패스 변수에 비닐 폴더 경로를 추가하여 자바명령어를 어디서든 사용할 수 있게 설정해야 한다.
  • 명령 프롬프트에서 " 자바-버전" 명령어를 입력하여 자바버전을 확인함으로써 환경 변수설정이 정상적으로 완료되었는지를 검증할 수 있다.

5.5. ️ IDE 설정 및 선택

  • 통합 개발 환경( IDE)은 소프트웨어 개발을 위해 다양한 기능을 제공하는 소프트웨어를 의미하며, 인텔리제이와 이클립스 등이 그 예시이다.
  • 수업에서는 편리함을 고려하여 인텔리제이를 설치하여 진행할 예정이며, 이 소프트웨어는 유료 버전과 무료 버전이 있다.
  • IDE는 코드 편집, 컴파일러, 디버거, 빌드, 자동화 도구, 프로젝트 관리, 버전 관리 등을 통합적으로 처리하기 때문에 편리성을 제공한다.
  • 사용자에게 맞는 도구를 선택하는 것이 중요하며, 정답은 없으므로 자신에게 편한 것을 사용하면 된다.

5.6. 인텔리제 설치 과정

  • 인텔리제는 Windows와 Mac에서 모두 사용 가능하며, 해당 아키텍처에 맞춰 설치를 진행해야 한다.
  • 설치 시 얼티메이트 버전은 유료이며, 무료 버전으로 설치하는 방법을 소개하고 있다.
  • 설치 파일을 실행 후, 필요한 용량은 약 2.7GB로, 설치 경로를 확인하고 기본 설정을 통해 진행한다.
  • 설치 완료 후 "Completing IntelliJ IDEA"라는 화면이 나타나며, 이를 통해 설치가 완료됨을 확인할 수 있다.
  • 마지막으로 설치를 마치기 위해 "Finish" 버튼을 눌러 종료한다.

5.7. 자바 프로그래밍 과정과 개발환경 설정

  • 자바는 파일 확장자가 .java인 파일로 작성되며, 이 파일은 JVM이 이해할 수 있는 바이트 코드로 변환된다.
  • 자바에서 생성된 클래스 파일의 확장자는 .class이며, 자바프로그램의 코드가 기계어로 번역된 결과물이다.
  • 자바 프로그램은 완전한 기계어가 아니며, JVM 소프트웨어 안에서 동작하도록 설계되었다.
  • 개발 환경으로는 인텔리제이를 사용하며, 프로젝트를 생성할 때 파일 경로, 언어, JDK 버전을 선택해야 한다.
  • 패키지는 유사한 기능을 제공하는 클래스의 집합으로, 자바소스코드의 기본 단위로서 기능적 그룹화를 위한 폴더 구조를 제공한다.

5.8. Hello World 프로그램 실행하기

  • 프로그래밍 언어를 배울 때 가장 먼저 하는 일은 'Hello world' 문자열을 출력하는 것이다.
  • 'Hello world'를 출력하기 위해 메인 메소드를 만들고, System.out.println 메소드를 호출한다.
  • 문자열을 소괄호 안에 큰따옴표로 작성하여 메소드에 전달한다.
  • 바이트 코드를 생성하고 실행하기 위해 인텔리제이에서 '런' 메뉴를 선택해야 한다.
  • 프로그램을 실행한 결과로 'Hello world'라는 문자열이 출력되는 것을 확인할 수 있다.

Key term

자바

자바는 컴퓨터 언어의 한 종류로, 사람들이 프로그램을 만들기 쉽게 만든 것입니다. 자동차를 조종하는 데 운전면허가 필요하듯, 컴퓨터를 조작하려면 프로그래밍 언어가 필요합니다. 자바는 다양한 기기에서 잘 작동하도록 설계되어 있어 인기가 많습니다.

프로그래밍 언어

프로그래밍 언어는 컴퓨터와 소통하기 위한 언어입니다. 우리가 사람 사이에 이야기를 할 때 사용하는 언어처럼, 컴퓨터와 소통할 때 사용하는 문법과 규칙이 있는 것입니다. 쉽게 말해, 컴퓨터에게 '무엇을' 하고 '어떻게' 하는지를 알려주는 언어입니다.

JDK

JDK는 Java Development Kit의 약자로, 자바 프로그램을 만들기 위한 도구입니다. 이 도구는 자바 코드를 작성하고, 컴파일하여 실행 가능한 프로그램으로 변환해줍니다. JDK는 요리하는 데 필요한 조리도구와 같은 존재입니다.

IDE

IDE는 Integrated Development Environment의 약자로, 프로그래머가 코드를 작성하고 오류를 찾아내기 쉽게 도와주는 소프트웨어입니다. 예를 들어, 요리할 때 주방이 여러 가지 도구로 가득 차 있듯이, IDE는 코드 작성, 디버깅, 그리고 프로젝트 관리를 도와주는 모든 도구가 모여 있는 공간입니다.

소스 코드

소스 코드는 우리가 프로그래밍 언어로 작성한 텍스트 파일입니다. 마치 소설의 원고처럼, 컴퓨터가 이해할 수 있도록 정리된 글이라고 생각하면 됩니다. 소스 코드는 후에 컴파일러를 통해 기계어로 변환됩니다.

컴파일러

컴파일러는 소스 코드를 컴퓨터가 이해할 수 있는 형태로 변환하는 프로그램입니다. 마치 외국어를 번역해주는 통역사와 같습니다. 즉, 소스 코드를 컴퓨터가 실행 가능한 바이트 코드로 변환해줍니다.

바이트 코드

바이트 코드는 컴파일러가 소스 코드를 변환한 중간 단계의 코드입니다. 이 코드는 컴퓨터가 직접 이해하는 기계어와는 다르지만, 가상의 컴퓨터인 JVM에서 실행될 수 있습니다. 즉, 바이트 코드는 번역된 문서처럼, 원래 언어에서 새로운 언어로 바뀐 중간 형태라고 볼 수 있습니다.

클래스

클래스는 객체 지향 프로그래밍에서 사용되는 기본 구조입니다. 이는 프로그래밍의 틀과 같아서, 특정 속성과 기능을 가지는 객체를 만들기 위한 설계도라고 생각할 수 있습니다. 즉, 건물을 지을 때 사용되는 청사진과 같은 것입니다.

객체 지향 프로그래밍

객체 지향 프로그래밍은 문제를 객체라는 단위로 나누어 해결하는 방식입니다. 이는 마치 거대한 퍼즐을 작은 조각으로 나누어 맞추는 것과 같아서, 보다 쉽게 관리하고 이해할 수 있습니다.

패키지

패키지는 비슷한 기능을 가진 클래스들을 그룹으로 묶어주는 구조입니다. 다양한 물건을 정리하기 위해 상자에 담는 것처럼, 패키지는 관련된 클래스들을 효과적으로 관리할 수 있게 해줍니다.

JVM

JVM은 Java Virtual Machine의 약자로, 바이트 코드를 해석하고 실행하는 소프트웨어입니다. 이는 마치 영화관에서 영화를 상영하는 프로젝터와 같습니다. JVM이 있어야 자바 프로그램을 실제로 실행할 수 있습니다.

운영 체제

운영 체제는 컴퓨터 하드웨어와 소프트웨어를 관리하는 프로그램입니다. 이는 마치 교통 경찰이 도로에서 차를 관리하는 것처럼, 운영 체제가 컴퓨터 안의 다양한 작업들을 조정하여 원활히 작동하도록 도와줍니다.

환경 변수

환경 변수는 시스템에서 실행되는 프로그램에 대한 정보를 담고 있는 변수입니다. 비유하자면, 요리할 때 사용하는 재료 목록과 같습니다. 이 변수들이 제대로 설정되어야 프로그램이 올바르게 실행될 수 있습니다.

코딩

코딩은 프로그래밍 언어를 사용하여 소스 코드를 작성하는 과정입니다. 이는 마치 작가가 이야기를 글로 옮기는 과정과 비슷하여, 아이디어를 실제 코드 형태로 표현하는 작업을 의미합니다.

정적 타이핑

정적 타이핑은 변수를 선언할 때 그 변수의 타입을 명시적으로 정의하는 것 입니다. 이는 마치 음료수를 주방에 올려 놓기 전에 어떤 음료인지 알려주는 라벨을 붙이는 것과 같습니다.

Hello World

Hellow World는 프로그래밍에서 첫 번째로 작성하는 가장 기본적인 예제입니다. 이는 프레젠테이션에서 청중에게 인사를 하는 것처럼, 첫 단추를 잘 끼우는 의미를 지니고 있습니다.

컴파일

컴파일은 소스 코드를 바이트 코드로 변환하는 과정입니다. 즉, 요리 재료를 조리하여 음식을 만드는 과정과 유사합니다.

인텔리제이

인텔리제이는 자바 언어를 위한 통합 개발 환경(IDE) 중 하나입니다. 이는 요리사가 여러 도구를 한 데 모아 효율적으로 요리하는 것과 같습니다. 인텔리제이를 사용하면 프로그래밍 작업이 한층 간편해집니다.

프로젝트 구조

프로젝트 구조는 소스 코드와 같은 파일들이 어떠한 방식으로 구성되어 있는지를 나타냅니다. 이는 건축물의 설계도와 같아서, 어떤 자원이 어디에 위치하는지 보여줍니다.

메모리 관리

메모리 관리는 프로그램이 실행될 때 필요한 메모리를 효율적으로 사용하게 도와주는 작업입니다. 이는 마치 집안의 물건들을 적절하게 정리하여 필요한 것을 쉽게 찾을 수 있도록 하는 것과 같습니다.
