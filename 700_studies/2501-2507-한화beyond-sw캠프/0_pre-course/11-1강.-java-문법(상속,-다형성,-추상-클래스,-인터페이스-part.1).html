11-1강. JAVA 문법(상속, 다형성, 추상 클래스, 인터페이스 part.1)

이번 강의에서는 자바의 객체지향 프로그래밍에서 기본적인 개념인 상속, 다형성, 추상 클래스, 인터페이스를 다룹니다. 상속을 통해 부모 클래스의 필드와 메소드를 자식 클래스에서 활용하는 방식을 배우며, 이를 통해 코드의 재사용성과 유지보수의 효율성을 높일 수 있습니다. 또한, 메소드 오버라이드에 대한 이해를 통해 자식 클래스에서 부모 클래스의 메소드를 재정의하는 방법을 익힐 수 있습니다. 이 강의는 자바의 구조를 파악하고, 객체지향 프로그래밍의 주요 개념에 대한 실용적인 이해를 도와줄 것입니다. 이러한 기술을 통해 효율적인 프로그래밍을 할 수 있습니다.

1. 📚 자바의 상속 개념 이해하기

  • 자바에서 상속은 부모 클래스의 멤버를 자식 클래스에게 물려주는 기능으로, 이를 통해 중복 코드를 줄이고 이미 개발된 클래스를 재사용할 수 있다.
  • 상속의 장점은 유지보수비용을 줄일 수 있고, 한 번의 수정으로 모든 자식 클래스에 적용할 수 있는 효과가 있다.
  • 자바에서는 자식 클래스가 부모 클래스를 상속받을 수 있으며, 클래스 간의 상속은 오직 하나의 클래스에 대해서만 가능하다.

2. 📚 상속을 통한 클래스 생성 및 메서드 활용

  • 상속을 통해 DMB 셀폰 클래스는 셀폰 클래스의 모든 필드와 메서드를 자동으로 갖게 된다.
  • 새로운 필드와 메서드를 추가하여 DMB 셀폰을 확장할 수 있으며, 예를 들어 DMB 채널과 관련된 메서드를 정의할 수 있다.
  • 객체 생성 시 부모 클래스의 생성자가 꼭 호출되며, super 키워드를 사용하여 부모 클래스의 생성자를 명시적으로 호출할 수 있다.
  • 학생 클래스를 만들고 이를 People 클래스에서 상속받아, 학생 번호를 추가하는 방법을 설명하며 super를 통해 부모 클래스의 필드를 초기화하는 과정을 다룬다.
  • 상속에서 프라이빗 필드와 메서드는 접근할 수 없으며, 부모 클래스와 자식 클래스가 다른 패키지에 있을 경우 디폴트 접근의 필드와 메서드 또한 제외된다.

2.1. 상속의 개념과 셀폰 클래스 구현

  • 새로운 프로젝트를 시작하며 상속을 설명하기 위해 '익스텐즈' 키워드를 사용하여 셀폰 클래스를 구현하기로 했다.
  • 셀폰 클래스에는 두 가지 필드로 모델과 컬러의 변수를 추가하였다.
  • 메소드로 전원 켜기와 전원 끄기를 파워온과 파워오프로 구현하였고, 각 메소드는 단순히 동작을 출력한다.
  • 메시지를 수신하고 전송하는 기능을 위해 샌드 보이스와 리시브 보이스 메소드를 추가하여 메시지를 출력하도록 설정하였다.
  • 마지막으로 전화 끊기 동작을 행업 메소드를 통해 구현하였다.

2.2. DMB 셀폰 클래스 구현

  • DMB 셀폰 클래스를 만들기 위해 기존 셀폰 클래스를 extends 키워드를 사용하여 상속한다.
  • DMB 셀폰은 상속된 셀폰 클래스의 모든 멤버를 포함하게 되며, 필요한 추가 기능만 구현할 수 있다.
  • DMB 셀폰 클래스에는 채널을 나타내는 변수와 해당 값을 설정하는 생성자가 포함된다.
  • 메소드로는 DMB 방송을 켜고 끄는 기능과 채널을 변경하는 turnOnDmb, changeChannel, turnOffDmb가 구현된다.
  • 마지막으로, DMB 셀폰 클래스의 구현이 완성되었다.

2.3. DMB 셀폰 클래스 실행 예제

  • DMB 셀폰 클래스를 실행하기 위해 메인 메소드를 선언하고, DMB 셀폰 객체를 생성하였다.
  • 객체 생성시 모델, 색상, 채널을 입력하고, 필드를 통해 상속받은 멤버를 출력하였다.
  • DMB 셀폰의 메소드를 호출하여 전원을 켜고 벨이 울리도록 하며, 전화 송수신 기능을 테스트하였다.
  • DMB 방송 수신을 시작한 후, 채널을 변경하고 DMB 방송 수신을 종료하는 기능을 구현하였다.

2.4. 자식 클래스의 생성과 부모 클래스의 상속 관계

  • 상속받은 멤버는 사용 가능하나, 여러 클래스를 동시에 상속할 수는 없다.
  • 부모 클래스의 프라이빗 필드와 메소드는 상속에서 제외된다.
  • 부모와 자식 클래스가 다른 패키지에 있을 경우, 디폴트 접근 제한의 필드와 메소드또한 상속되지 않는다.
  • 자식 객체를 생성할 때, 부모 객체가 먼저 생성된 후 자식 객체가 생성된다.
  • super 키워드를 통해 부모 생성자를 호출하며, 이로 인해 부모 객체가 먼저 생성된다.

2.5. ‍ 클래스 상속과 생성자 호출 과정

  • People 클래스를 생성하고, 필드로 퍼블릭 스트링 name과 퍼블릭 스트링 ssn을 포함하는 생성자를 구현한다.
  • Student 클래스는 People 클래스를 상속하며, 추가적으로 퍼블릭 인트 studentNumber 필드를 정의한다.
  • Student 생성자는 부모 클래스의 생성자를 호출하기 위해 super를 사용하지만, 기본 생성자가 없기 때문에 명시적으로 부모 생성자를 호출해야 한다.
  • Student 객체를 생성하고, 주민번호와 학번을 포함한 값을 설정하여 출력하는 실행 클래스를 구현한다.
  • 출력 결과로는 이름, 주민번호, 학번이 포함된 정보를 가져와 잘 표시되었다.

3. 🔄 메소드 오버라이딩의 이해

  • 부모 메소드는 자식 클래스에서 새롭게 정의되어 재정의되며, 이를 영어로 오버라이드라고 한다.
  • 메소드를 재정의할 때는 부모 메소드와 동일한 시그니처를 가져야 하며, 이를 통해 호출할 수 있다.
  • 접근 제한자를 더 좁은 범위로 재정의할 수 없으며, 새로운 예외를 추가하여 던질 수는 없다.
  • 메소드가 재정의되면 부모 객체의 메소드는 호출될 수 없으며, 자식 객체의 메소드만 호출된다.
  • 차일드 클래스는 부모 클래스인 parents의 상속을 받으며, 동일한 시그니처로 메소드를 다시 선언하면 자식 클래스의 메소드가 호출된다.

4. ✈️ 상속과 메소드 오버라이드의 예시

  • 자식 클래스에서 부모 클래스 메소드를 호출할 때 'super' 키워드를 사용하여 명시적으로 호출할 수 있다.
  • 자식 클래스는 부모 클래스의 메소드를 사용할 수 있으며, 예제로 설명된다.
  • 부모 클래스를 상속받고, 메소드2번을 오버라이드한 상태로 자식 클래스가 구현된다.
  • 예시로 'Airplane' 클래스를 만들며 다양한 메소드를 정의하고 각각의 기능을 출력한다.
  • 'SuperSonicAirplane' 클래스를 추가하여 기본 항공기 클래스에서 상속받고, 추가적인 상수를 정의하며 비행 모드에 따라 다르게 동작하도록 설정된다.

5. 🔑 파이널 클래스와 메소드의 특징

  • 파이널(final) 키워드가 붙으면 해당 선언이 최종 상태가 되어 더 이상 수정할 수 없다.
  • 파이널 클래스는 상속이 불가능하고, 따라서 자식 클래스를 만들 수 없다.
  • 예를 들어, String 클래스는 파이널 키워드가 붙어 있어 상속이 불가능하다.
  • 파이널 메소드는 부모 클래스에서 선언된 인 경우, 자식 클래스에서 재정의(오버라이드)가 불가능하다.
  • 결론적으로, 파이널이 붙은 클래스와 메소드는 상속 및 재정의가 불가능하다.

Key term

상속

상속은 부모 클래스의 속성과 기능을 자식 클래스가 물려받는 것을 의미합니다. 예를 들어, 부모가 자식에게 잘 유전시킨다는 개념과 비슷합니다. 이렇게 하면 자식 클래스는 부모 클래스의 코드와 기능을 재사용할 수 있어 개발이 간편해집니다. 자바에서는 한 클래스만 상속을 받을 수 있습니다.

다형성

다형성은 같은 명령어가 다른 방식으로 동작할 수 있는 능력을 의미합니다. 예를 들어, '올리기'라는 동작이 상황에 따라 서로 다른 의미를 가질 수 있습니다. 자바에서는 메소드 이름이 같지만 서로 다른 방법으로 구현된 메소드를 사용합니다. 이를 통해 프로그래밍의 유연성을 높입니다.

추상 클래스

추상 클래스는 완전하지 않은 클래스입니다. 즉, 직접 객체를 만들 수 없고, 다른 클래스가 상속받아 사용해야 합니다. 마치 완성되지 않은 틀과 같아서, 구체적인 아이디어에 대해 기능을 정의할 수 있습니다. 필수적으로 자식 클래스에서 구현해야 할 메소드가 있을 수 있습니다.

인터페이스

인터페이스는 클래스가 구현해야 할 메소드의 청사진과 같습니다. 즉, 인터페이스에 정의된 메소드는 클래스에서 반드시 구현해야 합니다. 자바에서 인터페이스는 다중 상속을 지원하기 때문에 여러 클래스로부터 다양한 기능을 가져올 수 있습니다.

객체지향 프로그래밍

객체지향 프로그래밍은 프로그램을 특정 문제 해결을 위한 객체들의 상호작용으로 구성하는 방법입니다. 예를 들어, 실제 세계에서 각 개인이 역할에 따라 상호작용하는 것처럼, 코드에서도 객체들이 역할을 나누어 기능을 수행합니다.

부모 클래스

부모 클래스는 자식 클래스가 상속받는 클래스입니다. 기본적인 속성과 기능을 제공합니다. 이를 통해 자식 클래스는 직접 구현할 필요 없이, 부모 클래스의 기능을 지속적으로 활용할 수 있습니다.

자식 클래스

자식 클래스는 부모 클래스를 상속받아 만들어진 클래스입니다. 부모 클래스로부터 속성과 기능을 물려받아 자신만의 특징을 추가할 수 있습니다. 마치 부모의 성격과 외모를 물려받은 자녀와 같습니다.

메소드 오버라이드

메소드 오버라이드는 자식 클래스에서 부모 클래스의 메소드를 다시 정의하는 것입니다. 즉, 부모가 만든 메소드를 자식이 자신의 방식으로 '재작성'하는 과정입니다. 이를 통해 자식 클래스에 맞게 기능을 조정할 수 있습니다.

재사용성

재사용성은 이미 작성된 코드를 한 번 더 사용하는 능력을 의미합니다. 상속을 통해 부모 클래스의 기능을 자식 클래스가 활용함으로써, 코드 중복을 줄일 수 있습니다. 여러 번 같은 것을 쓰기보다, 한 번 잘 만들고 필요할 때마다 활용하는 것이 효율적입니다.

유지보수

유지보수는 코드 수정과 개선 작업을 포함합니다. 상속 구조를 가짐으로써, 부모 클래스의 수정이 자식 클래스에 자동으로 반영되어 유지보수가 간편해집니다. 즉, 한 곳에서만 변경해도 모든 관련 코드가 함께 변경해 줍니다.

필드

필드는 클래스의 속성을 의미합니다. 객체가 가지는 데이터 또는 상태와 같은 정보를 저장하는 역할을 합니다. 예를 들어, 자동차 클래스에서 '색상', '모델명' 같은 것이 그 자동차의 필드입니다.

메소드

메소드는 클래스가 수행할 수 있는 동작을 정의합니다. 마치 개인이 가진 능력이나 행동처럼, 객체가 무엇을 할 수 있는지를 알려줍니다. 예를 들어, 자동차 클래스에서 '주행하기', '정지하기'와 같은 메소드가 있을 수 있습니다.

접근 제한자

접근 제한자는 클래스, 필드, 메소드의 접근 범위를 설정합니다. 예를 들어, 'public'은 모든 클래스에서 접근할 수 있지만 'private'는 해당 클래스 내에서만 접근 가능합니다. 이는 보안과 은닉성을 위해 사용됩니다.

생성자

생성자는 객체가 생성될 때 호출되는 특별한 메소드입니다. 객체의 초기 값을 설정해주는 역할을 합니다. 예를 들어, 자동차 객체가 생성될 때 '모델'과 '색상'을 정해주는 것이 생성자의 주요 역할입니다.

파이널 클래스

파이널 클래스는 더 이상 상속될 수 없는 클래스를 의미합니다. 즉, 이 클래스는 더 이상 자식 클래스가 만들어질 수 없습니다. 예를 들면, 특정 기능을 고정하고 싶을 때 사용됩니다.

파이널 메소드

파이널 메소드는 오버라이드할 수 없는 메소드입니다. 즉, 이 메소드는 자식 클래스에서 변경할 수 없습니다. 이는 특정 기능의 변형 없이 항상 같은 방식으로 작동하게끔 보장합니다.

디폴트 접근 제한

디폴트 접근 제한은 클래스와 멤버가 접근 제한자가 명시되지 않았을 때 적용됩니다. 같은 패키지 안에서만 접근할 수 있는 제한입니다. 예를 들어, 특정 코드를 패키지 내부에서만 활용하고 싶을 때 쓸 수 있습니다.

오버라이드 규칙

오버라이드 규칙은 자식 클래스가 부모 클래스의 메소드를 재정의할 때 지켜야 할 규칙입니다. 부모 메소드와 시그니처가 동일해야 하고, 접근 제한자는 동일하거나 더 넓어야 하며, 새로운 예외를 던져서는 안됩니다.

객체 생성

객체 생성은 클래스를 바탕으로 실제 사용할 수 있는 인스턴스를 만드는 것이라는 의미입니다. 자바에서는 'new' 키워드를 사용하여 객체를 생성합니다. 이 객체는 클래스에 정의된 속성과 기능을 갖게 됩니다.

Java

Java는 객체지향 프로그래밍 언어로, 사용하기 쉽고 다양한 플랫폼에서 실행될 수 있도록 설계되었습니다. 이를 통해 한 번 코드를 작성하면 다양한 기기에서 사용할 수 있는 장점이 있습니다.

클래스 템플릿

클래스 템플릿은 객체를 생성하기 위한 청사진을 의미합니다. 즉, 클래스는 객체의 속성과 동작을 정의하는 틀입니다. 이를 통해 우리는 다양한 객체를 생성할 수 있습니다.
