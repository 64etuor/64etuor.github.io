5강. JAVA 문법(조건문과 반복문)

이번 강의는 자바 프로그래밍에서 조건문과 반복문의 중요성을 설명합니다. 조건문은 특정 조건에 따라 코드 블록의 실행 여부를 결정하며, 반복문은 특정 작업을 여러 번 수행하기 위해 사용됩니다. 이 강의를 통해 참가자는 조건문과 반복문의 기본 개념을 이해하고, 실습을 통해 프로그램의 흐름을 조절하는 방법을 배울 수 있습니다. 또한, 여러 조건을 동시에 다루는 if-else와 switch 문, 다양한 반복문(while, for 등)의 사용법도 익힐 수 있습니다.

1. ⚙️ 조건문과 반복문 개요

  • 조건문은 조건식에 따라 다르게 실행되는 문장으로, 조건식이 참일 때 해당 코드 블록이 실행되며, 거짓일 경우 블록을 건너뛴다.
  • 스위치문은 특정 변수의 값과 일치하는 경우 해당 코드를 실행하고, 일치하지 않을 경우 디폴트 코드를 실행한다.
  • 조건식의 결과에 따라 코드 블록의 실행 여부가 결정되고, 이 조건식에는 불리언 값이 사용될 수 있다.

2. 📜 조건문과 반복문의 기본 개념

  • 조건식이 참일 때 중괄호 블록이 실행되며, 단일 실행문이 있을 경우 중괄호는 생략 가능하다.
  • 실제 예제로 점수를 비교하여 90점 이상이면 "점수가 90점 이상입니다"라는 메시지를 출력하고, 그렇지 않으면 "점수가 90보다 작습니다"라는 메시지를 출력한다.
  • 자바 프로그램은 main 메소드에서 시작하여 위에서 아래로 실행되며, 흐름 제어가 가능하다.
  • 조건문은 주어진 조건이 참일 경우 특정 코드 블록을 실행하며, 조건이 거짓일 경우 건너뛴다.
  • if 문은 조건이 참일 경우 해당 블록을 실행하고, 조건이 틀리면 다음 코드 블록으로 이동하는 구조를 가졌다.

3. 📝 if-else 조건문 활용하기

  • if문을 사용하여 특정 조건을 만족할 때 실행할 코드를 작성할 수 있으며, 예를 들어, 스코어가 90 이상일 경우 "점수가 90보다 큽니다"를 출력한다.
  • else 블록은 상위 조건이 거짓일 경우 실행되며, 예를 들어, 스코어가 90 미만일 때 "점수가 90보다 작습니다"와 "등급은 B입니다"를 출력하게 된다.
  • 여러 조건을 차례로 체크하기 위해 if, else if, else를 사용할 수 있으며, 각각의 조건에 따라 다른 실행 블록이 선택된다.
  • else if를 사용하면 여러 조건을 논리적으로 결합하여 복잡한 조건식을 처리할 수 있으며, 예를 들어, 스코어가 80 이상이고 90 미만일 경우 "점수가 80에서 90입니다"와 "등급은 B입니다"를 출력할 수 있다.
  • 조건을 모두 확인한 후에도 해당하지 않을 경우 마지막 else 블록에서 기본값을 출력할 수 있으며, 예를 들어, 스코어가 70 미만일 경우 "점수는 72입니다"와 "등급은 D입니다"를 출력하는 형태이다.

4. 📊 조건문과 반복문의 활용 방식

  • 주어진 점수 75점은 90점보다 작고 80점보다 작으며 70점보다 크기 때문에 특정 조건문의 true 값을 가지게 된다.
  • 랜덤 숫자를 생성하는 과정에서 0에서 1 미만의 소수에 6을 곱해 1부터 6까지의 정수를 랜덤하게 생성할 수 있도록 한다.
  • 랜덤 생성된 정수에 따라 결과를 출력하는 방식으로, 예를 들어 1, 2, 3, 4, 5, 6번이 랜덤하게 나올 수 있다.
  • 반복문은 주로 세 가지 유형(for문, while문, do-while문)으로 구분되며, for문은 반복 횟수를 알고 있을 때 주로 사용된다.
  • 합계를 계산하려는 경우 1부터 5까지의 합을 구하기 위해 for문을 사용하며, 큰 값이 되면 처리하기 어려운 경우가 생길 수 있다.

5. 🔄 for문과 while문 사용법

  • for문은 초기화식 실행 후, 조건이 true일 때만 반복이 되어, 사용자가 설정한 범위 내에서 반복 동작을 수행한다.
  • 예를 들어, int i가 1부터 100까지 증가하는 과정을 통해 사용하는 for문으로 반복시킨다. 그리고 누적된 값은 썸이라는 변수에 할당된다.
  • while문은 조건식이 true인 동안 반복 활동을 수행하며,주로 조건식을 통해 반복 여부가 결정된다.
  • break문을 사용하면 특정 조건이 발생했을 때 반복문을 종료할 수 있으며, 특히 중첩된 반복문에서 활용 가능하다.
  • continue문은 반복문 내에서 특정 조건이 성립할 때, 그 이하의 코드를 건너뛰고 다음 반복으로 넘어간다.

5.1. for문의 작동 원리

  • for문은 초기화식이 먼저 실행되며, 이후 조건을 체크하게 된다. 조건이 true일 때만 내부 코드가 실행된다.
  • 실행 후에는 증감식을 통해 변수 값을 조정하고, 다시 조건을 평가하여 반복하게 된다.
  • 예를 들어, int i 변수를 1로 초기화하고 100까지 반복할 때, 매 반복마다 i를 누적하여 합계를 계산한다.
  • i 값이 100이 되면 조건이 false가 되어 반복이 종료되며, 최종 결과를 출력한다.
  • for문을 이해하기 위해서는 간단히 생각하고 반복의 동작 방식을 잘 살펴보아야 한다.

5.2. ️ JAVA 반복문 예제

  • 반복문을 사용하여 1부터 100까지의 합을 구하는 프로그램을 작성했다. 변수 썸에 누적하여 더하는 방식을 사용했다.
  • 반복문에서 i를 1로 초기화하며, 100 이하일 때까지 반복하고 i를 1씩 증가시키는 구조이다.
  • 1부터 100까지의 합이 5,050이라는 결과를 출력하는데, 이는 반복문 종료 후 이루어진다.
  • 다른 예로 플로트형 변수를 사용한 경우, 0.1부터 시작해 1.0보다 작은 값일 때까지 반복하여 값을 출력하는 프로그램도 소개했다.
  • 복합 대입 연산자를 활용하여 변수의 값을 누적할 수 있으며, 재사용 가능하다는 점도 강조되었다.

5.3. ️ 구구단 구현을 위한 중첩 반복문 설명

  • Java의 메인 메소드를 선언한 후 구구단을 만들기 위한 중첩 for문을 사용해야 한다.
  • int m 변수는 1부터 9까지의 값을 반복하며, 이 값은 각 단의 곱셈의 첫 번째 요소로 사용된다.
  • int n 변수는 1부터 9까지 증가하며 각 곱셈의 두 번째 요소로 기능한다.
  • for문은 처음부터 끝까지 반복되고, 출력은 System.out.print를 통해 이루어지며, 각 단의 결과를 표시한다.
  • 중첩된 for문을 통해 1단부터 9단까지 결과를 출력하는 구조로 구현된다.

5.4. 반복문의 조건식 및 사용법

  • 반복문은 조건식에 따라 반복 여부를 결정하며, 조건이 트루일 때 반복하고 펄스일 때 종료된다.
  • while문은 조건식이 트루인 동안 반복하며, 이 조건식은 비교 연산식을 사용하여 결과로 불린 타입의 값을 생성한다.
  • 예를 들어, i가 10 이하인 동안 반복하며, 반복 중 i를 출력하고 증감시켜야 무한 루프에 빠지지 않는다.
  • 또 다른 예에서, 1부터 100까지의 합을 누적하여 구하는 방식으로 사용되며, 초기식과 증감식을 바깥쪽에서 정의할 수 있다.
  • while문과 for문의 차이점은 주로 반복 구문 내에서 조건을 따지는 방식에 있다는 점이다.

5.5. break문과 continue문 사용법

  • break문은 반복문이나 스위치문에서 특정 영역을 빠져나갈 때 사용된다.
  • break문은 주로 이프문과 함께 사용되며, 특정 조건이 충족될 때 사용할 수 있다.
  • 중첩된 반복문에서 레이블을 사용하여 바깥쪽 반복문을 빠져나갈 수 있다.
  • 예를 들어, 주사위를 던져 6이 나올 때까지 반복하고, 6이 나왔을 때 break문을 통해 반복문을 빠져나가는 방식으로 활용된다.
  • continue문은 반복문에서 특정 조건에 해당될 경우 해당 반복의 나머지를 건너뛰고 다음으로 넘어가게 한다.

5.6. ️ 스위치 문에 대한 설명

  • switch문은 특정 변수의 값에 따라 해당하는 case를 찾아 블록을 실행하는 방식이다. 즉, 변수 값이 일치하는 case에서부터 실행된다.
  • 스위치문 내에서 break문을 만나게 되면, 해당 블록의 실행을 중단하고 스위치문을 빠져나간다.
  • 예를 들어, 주사위 값을 결정하기 위해 INT number를 생성하고 무작위로 1부터 6까지의 정수를 반환하는 스위치문을 만드는 예제가 있다.
  • 스위치문은 여러 케이스를 나열하여 각 케이스에 따라 실행할 코드를 작성할 수 있으며, 각 케이스 후에는 반드시 브레이크를 추가해야 한다.
  • 만약 break문이 없다면 스위치문의 모든 블록이 실행될 수 있으며, 이는 의도하지 않은 결과를 초래할 수 있다.

5.7. ️ 스위치문과 조건문 활용 예제

  • 스위치문을 사용하여 현재 시간에 따른 행동을 결정하는 예제에서, 시간이 9시일 경우 "회의를 합니다"로 출력되며 케이스 문이 없으면 디폴트로 "외근을 나갑니다"가 출력된다.
  • 대소문자를 구분하지 않기 위해 스위치문에 두 개의 케이스를 병합하여 처리하는 방식으로, 예를 들어 대문자 B는 "우수회원입니다"로 출력되는 특징이 있다.
  • 문자열 비교를 가능하게 하여 직책에 따라 급여를 출력하는 예제에서, "과장"인 경우 500만원이라고 출력되며, 기본값으로는 300만원이 설정된다.
  • 스위치문의 활용을 통해 조건을 간단하게 처리할 수 있으며, 브레이크 문이 없을 시 다음 케이스까지 실행되는 점이 강조된다.
  • 스위치문을 활용한 여러 예제를 통해 조건문을 효율적으로 사용하는 방법을 탐구하였다.

Key term

자바

자바는 객체 지향 프로그래밍 언어 중 하나로, 다양한 플랫폼에서 사용할 수 있도록 설계되었습니다. 즉, 자바로 작성한 프로그램은 여러 운영 체제에서 실행할 수 있다는 뜻이에요. 이 언어는 문법이 쉽고, 안정성이 높아 많은 개발자들이 사용합니다.

조건문

조건문은 프로그램이 주어진 조건에 따라 다른 행동을 하도록 도와주는 문장입니다. 예를 들어, 성적을 기준으로 합격하거나 불합격을 판단하는 것 같은 경우를 생각해 볼 수 있습니다. 조건문이 참이면 특정 블록의 코드가 실행되고, 거짓이면 다른 코드를 실행할 수 있어요.

반복문

반복문은 특정 작업을 여러 번 수행하기 위해 사용됩니다. 예를 들어, 구구단을 출력할 때 반복문을 사용하면 각 단을 쉽게 출력할 수 있습니다. 주어진 조건이 참인 동안 계속해서 작업을 반복하는 구조죠.

if문

if문은 조건이 참일 때 특정 코드를 실행합니다. 만약 조건이 거짓이면 다음 코드를 실행하게 됩니다. 예를 들어, '만약 점수가 90 이상이라면 A학점.'과 같은 조건을 줄 수 있어요.

switch문

switch문은 여러 조건 중 하나를 선택할 수 있게 도와줍니다. if문과 비슷하지만, 여러 경우를 효율적으로 처리할 수 있습니다. 특정 변수의 값에 따라 코드의 분기점을 쉽게 정의할 수 있죠.

for문

for문은 반복 횟수가 정해져 있을 때 사용합니다. 예를 들면, 1부터 10까지의 수를 출력할 때 유용해요. 시작 값, 종료 값, 증감식을 설정하여 정해진 범위만큼 반복할 수 있습니다.

while문

while문은 주어진 조건이 참인 동안 계속 반복합니다. 예를 들어, 사용자가 종료를 선택할 때까지 게임을 계속 진행하는 경우에 사용할 수 있어요. 조건이 참일 때만 해당 블록의 코드가 실행됩니다.

변수

변수는 데이터를 저장하는 공간입니다. 예를 들어, 학생의 점수를 저장하기 위해 'score'라는 변수를 만들면, 이 변수에 점수 값을 저장하고 필요할 때 꺼내 쓸 수 있어요.

메인 메소드

메인 메소드는 자바 프로그램의 시작점입니다. 프로그램을 실행할 때 제일 먼저 호출되는 부분이죠. 메인 메소드를 작성하지 않으면, 프로그램이 실행되지 않아요.

블록

블록은 중괄호 {}로 묶인 코드의 묶음입니다. 조건문이나 반복문 안에 쓰여서 그 안에 있는 코드가 어떤 조건이나 반복에 따라 실행되도록 나타냅니다.

true

true는 참(True)이라는 의미로, 조건문에서 조건이 만족했을 때를 나타냅니다. 즉, '조건이 맞다'는 뜻이에요.

펄스

펄스는 거짓(False)이라는 의미로, 조건문에서 조건이 만족하지 않았을 때를 나타냅니다. 즉, '조건이 맞지 않는다'는 뜻이에요.

비교 연산자

비교 연산자는 두 값을 비교하는 기호입니다. 예를 들어, >(크다)나 <(작다) 같은 것이죠. 이 연산자를 사용하면 조건이 참인지 거짓인지 판단할 수 있어요.

중괄호

중괄호는 {}로 표현되며, 코드의 블록을 정의하는데 사용됩니다. 여러 줄의 코드를 묶어줌으로써, 하나의 단위로 묶여서 조건이나 반복문 속에서 행동할 수 있게 합니다.

스코어

스코어는 점수라는 의미입니다. 프로그램에서는 학생의 점수나 게임의 점수를 나타낼 때 사용되죠. 예를 들어 시험 점수와 같은 수치를 저장할 수 있어요.

SOUT

SOUT는 System.out.println의 약어로, 콘솔에 내용을 출력할 때 사용하는 간단한 방법입니다. 즉, 화면에 무엇인가 보이게 할 때 쓰는 코드에요.

증감식

증감식은 변수를 늘리거나 줄이는 방법입니다. 예를 들어, i++는 'i의 값을 1 증가시키다'라는 의미죠. 이런 방식으로 반복문 내에서 변수를 조절할 수 있어요.

입력값

입력값은 사용자가 프로그램에 제공하는 데이터입니다. 이를 통해 프로그램은 사용자의 요구를 이해하고 처리할 수 있습니다.

랜덤

랜덤은 무작위라는 뜻입니다. 컴퓨터 프로그램에서 랜덤한 값을 생성하면, 예를 들어 주사위를 던지듯이 예측할 수 없는 결과를 만들어 낼 수 있어요.

선언

선언은 변수를 만들고 그 타입을 정의하는 것입니다. 예를 들어, int number;라고 작성하면, 숫자를 담을 수 있는 변수인 number를 선언한 거죠.

핸들링

핸들링은 프로그래밍에서 사건이나 오류를 다루는 방법을 의미합니다. 예를 들어, 잘못된 입력값이 들어왔을 때 그걸 처리하기 위해 어떻게 할지를 계획하는 것과 같아요.

흐름 제어문

흐름 제어문은 프로그램의 실행 경로를 조절하는 문장을 의미합니다. 조건문과 반복문이 여기에 해당하죠. 프로그램의 흐름을 제어함으로써 원하는 작업을 수행할 수 있도록 합니다.

디폴트

디폴트는 기본값이라는 뜻입니다. 주로 switch문에서 특정 값이 없는 경우 실행되는 코드를 지정할 때 사용되죠.

조건식

조건식은 참 또는 거짓을 반환하는 수식입니다. 조건문에서 어떤 행동을 할지를 결정하는 데 사용돼요.

휴먼 리더블

휴먼 리더블은 인간이 이해하기 쉬운 형태를 의미합니다. 코드나 데이터가 사람에게 쉽게 보이도록 구성된 것을 뜻해요.

제어문

제어문은 프로그램의 흐름을 제어하는 문장입니다. 조건문과 반복문이 이에 해당하며, 특정 상황에 따라 실행될 코드를 결정해주는 역할을 하죠.

유기적

유기적이라는 표현은 서로 관련이 있는, 또는 자연스러운 연결을 가진 상태를 의미합니다. 프로그램 내에서 여러 구성요소가 유기적으로 작동할 때 좋은 성능을 낼 수 있어요.

다중 조건

다중 조건은 여러 가지 조건을 동시에 고려하는 것을 의미합니다. 예를 들어, 성적 조건이 여러 개일 때 각각의 조건을 잘 조합하여 결과를 도출할 수 있죠.

구구단

구구단은 1부터 9까지의 숫자를 곱하는 수학적 작업입니다. 보통 학생들이 수학을 배우기 위해 암기하는 것으로, 반복문을 통해 쉽게 구현할 수 있어요.
