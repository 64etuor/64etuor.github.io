13강. JAVA 문법(컬렉션)

이 강의는 자바의 컬렉션 프레임워크에 대한 기초적인 내용을 다루고 있습니다. 컬렉션 프레임워크는 객체를 효율적으로 추가, 삭제및 검색할 수 있는 방법을 제공하며, 이 강의를 통해 그 구조와 사용법을 이해할 수 있게 됩니다. 각각의 컬렉션 유형인 list, set, map 등에 대한 설명 및 메소드사용법을 배운 후, 각 컬렉션의 특징과 차별점을 명확히 알게 될 것입니다. 이러한 기초 지식은 자바 프로그래밍을 위한 필수적인 바탕이 될 것입니다. 강의를 통해 자바 컬렉션의 활용법을 배우면, 실제 프로젝트 개발에 큰 도움이 될 것입니다.

1. 📚 컬렉션 프레임워크 개요

  • 컬렉션은 배열의 한계를 극복하기 위해 등장한 자료 구조로, 객체 저장을 지원한다.
  • 자바의 컬렉션 프레임워크는 다양한 인터페이스와 클래스를 통해 객체를 효율적으로 추가, 삭제, 검색할 수 있도록 제공하고 있다.
  • list, set, map등 여러 인터페이스와 그 구현 클래스들이 컬렉션 프레임워크의 핵심 구성 요소이다.

2. 📋 list 컬렉션의 특징

  • list 컬렉션은 배열과 유사하게 객체를 인스턴스로 관리하는 구조이다.
  • list의 저장 용량은 자동으로 증가하며, 배열과의 큰 차이점으로는 가변적인 크기 조정이 가능하다는 점이 있다.
  • 객체 저장시 자동으로 인덱스가 부여되며, 추가, 삭제, 검색을 위한 다양한 메소드들이 제공된다.
  • list컬렉션은 객체 자체가 아닌 객체의 참조 주소를 저장하여, 동일 객체의 중복 저장이 가능하다.
  • 또한, list컬렉션은 널 값도 저장할 수 있다.

3. 📚 list 컬렉션의 메소드 이해

  • list컬렉션에는 array list, 벡터, linked list가 포함되어 있으며, 이들을 구현하는 클래스들은 공통적인 인터페이스의 메소드를 갖고 있다.
  • 객체를 추가할 때는 add 메소드를 사용하여, 인덱스 값을 지정하면 해당 위치에 객체를 추가할 수 있다.
  • 수정할 때는 set 메소드를 이용하여 주어진 인덱스의 저장된 객체를 변경할 수 있다.
  • contains를 사용하여 특정 객체의 존재 여부를 판단할 수 있으며, get 메소드를 통해 특정 인덱스의 요소를 가져올 수 있다.
  • list는 비어있는지 확인하고, 요소 개수를 size 메소드로 확인하며, 모든 요소를 삭제할 때는 clear 메소드를 사용할 수 있다.

4. 📋 array list의 특징과 활용

  • array list는 list인터페이스의 대표적인 구현 클래스로, 특정 타입의 객체를 저장할 수 있도록 타입 파라미터를 지정해야 한다.
  • array list는 기본 생성자로 생성 시 10개의 객체를 저장할 수 있는 초기 용량을 가지며, 객체 수가 늘어나면 용량이 자동으로 증가한다.
  • 객체를 추가할 때 0번 인덱스부터 차례대로 저장되며, 특정 인덱스의 객체를 제거하면 뒤쪽 인덱스의 모든 요소가 당겨온다.
  • 중간에 많은 객체의 추가나 삭제가 일어난다면, 이로 인해 큰 연산이 발생할 수 있으며, 따라서 이 경우에는 linked list를 사용하는 것이 더 효율적일 수 있다.
  • array list는 마지막에만 요소를 추가할 때 좋은 성능을 발휘한다.

5. 📝 자바 컬렉션의 기초 및 활용

  • 새로운 프로젝트를 생성하고, 컬렉션이라는 이름으로 패키지를 만든다. 첫 자바 클래스로 ArrayList 예제를 생성하고 메인 메소드를 선언한다.
  • list는 자바 유틸리티의 list를 사용하며, 원하는 타입을 정의하고 변수명을 설정 후, 구현 클래스를 통해 list를 사용한다.
  • list에 값을 추가하고, 인덱스를 기반으로 요소를 조회하는 방법에 대해 설명한다. 또한, 포문을 사용하여 list의 모든 요소를 출력하는 방법을 소개한다.
  • list에서 요소를 삭제하는 방법과 함께, 향상된 for문을 통해 list를 순회하는 방법을 설명한다. <21,28-35>>
  • 벡터와 linked list를 비교하고, 각각의 장점과 사용 방법에 대해 설명한다. 벡터는 멀티스레드 환경에서 안전하고, linked list는 중간에 추가 및 삭제시 성능이 더 유리하다.
  • set 컬렉션은 순서를 유지하지 않으며, 중복을 허용하지 않는 특징이 있다. 여러 구현체가 있으며, iterator를 통해 set의 요소를 반복하며 접근하는 방법을 설명한다.
  • map은 키-값의 쌍으로 구성되며, 키의 유일성을 강조하고, 기본 사용법과 자주 사용되는 메소드를 설명한다.
  • 해시 map의 동작 방식과 예제를 통해 스튜던트 클래스를 정의하고, 해시 코드를 통해 중복 객체 생성을 방지하는 방법을 설명한다.
  • 전체적으로 자바의 컬렉션 프레임워크의 개요와 list, set, map의 특징을 정리하며, 인터페이스와 구현체의 다양성에 대해 언급한다.

5.1. 자바 컬렉션 프로젝트 생성 및 list 사용

  • 새로운 자바 프로젝트를 컬렉션이라는 이름으로 생성하고, src 폴더에 자바 클래스를 추가한다.
  • 패키지명을 섹션 01 그리고 클래스명을 array list 이그잼플로 설정하며, 메인 메소드를 선언한다.
  • 자바 유틸의 list를 사용하고 원하는 타입을 지정할 수 있으며, 변수명을 설정한다.
  • list에 여러 요소를 추가하는 방법으로 예를 들어 '자바', 'jdbc', '서블릿', 'jsp' 등을 추가하고 데이터베이스라는 요소를 두 번째 인덱스에 삽입한다.
  • 이후 list의 사이즈를 확인하고, list에서 두 번째 인덱스의 요소를 출력하며, 포문을 통해 list의 모든 요소를 루핑하여 출력하는 과정을 설명한다.

5.2. ️ list 요소 삭제 및 출력 과정

  • list에서 두 번째 인덱스에 위치한 요소를 삭제하고, 아이팟 티스트 객체를 찾아서 추가적인 삭제를 진행한다.
  • 향상된 포문을 사용하여 스트링 값을 가진 요소들을 list로 작성하고 가져온다.
  • 준비된 list를 출력하고, 추가한 값들이 잘 저장되어 있는지를 확인할 수 있다.
  • 데이터베이스는 인덱스 2번에 위치하며, 인덱스 2의 값이 삭제되면서 서블릿 JSP는 인덱스 1로 밀려난다.
  • 최종적으로 자바와 JDBC 두 가지 요소가 남게 되어 list의 상태를 출력한다.

5.3. 벡터와 linked list 비교

  • 벡터는 array list와 동일한 내부 구조를 가지지만, 멀티스레드 환경에서 안전하게 동작하는 특징이 있다.
  • array list는 동기화가 이루어지지 않아 멀티스레드에서 안전하지 않다.
  • linked list는 데이터를 보관하는 내부 구조가 다르며, 중간에 추가 및 삭제할 때 더 유리하게 작동한다.
  • linked list의 요소들은 인접 참조를 통해 연결되며, 특정 인덱스의 객체 제거 시 앞뒤 링크만 변경되므로 효율적이다.
  • 객체 삽입이 빈번한 경우, linked list는 array list보다 더 좋은 성능을 발휘한다.

5.4. set 컬렉션의 특성과 메소드

  • set 컬렉션은 저장 순서를 유지하지 않으며, 객체의 중복을 허용하지 않는다. 따라서 여러 개의 널 값이 있어도 하나만 저장된다.
  • set컬렉션은 수학의 집합 개념과 유사하며, 순서가 없고 중복이 허용되지 않는 특성을 가지고 있다.
  • 다양한 메소드가 제공되며, 이러한 메소드는 add, contains, isEmpty, iterator 등을 포함하고, 인덱스와 관련된 메소드는 없다.
  • iterator를 통해 요소를 반복적으로 추출할 수 있으며, hasNext, next, remove 메소드가 정의되어 있다.
  • iterator를 사용하지 않고도 향상된 for문을 사용하여 전체 객체를 간단히 가져올 수 있다.

5.5. ️ 해시set의 구현과 동작 이해

  • 해시set은 객체를 순서 없이 저장하고 동일한 객체는 중복 저장하지 않는다.
  • 해시 set의 중복 판단은 객체의 해시코드와 equals 메소드를 사용하여 진행된다.
  • 객체의 내용을 비교할 때, 이름과 나이가 같을 경우 동등한 객체로 판단하도록 equals 메소드를 오버라이드 한다.
  • 해시 set에 객체를 추가할 때 두 객체의 이름과 나이가 같으면 중복으로 인식되어 저장되지 않는다.
  • 해시 set의 객체 수는 중복되지 않는 경우에만 증가하며, 이 경우의 객체 수는 1로 유지된다.

5.6. ️ map 컬렉션의 개념 및 사용법

  • map은 키와 값으로 구성된 엔트리 객체를 저장하는 컬렉션이다. 각각의 엔트리는 키와 값의 쌍으로 나타나며, 이 키를 통해 특정 값을 접근할 수 있다.
  • 키는 유일해야 하며, 동일한 키로 값을 저장할 수 없다. 동일 키로 값을 추가하게 되면 이전 값은 삭제되고 마지막에 저장한 값만 남아있다.
  • map의 주요 메서드로는 put, get, remove, clear, isEmpty가 있다. 이 메서드를 사용하여 키와 값을 추가하거나 조회, 삭제할 수 있다.
  • map은 다양한 구현체를 가지고 있으며, 해시map, 트리map 등 여러 종류가 있다. 모든 구현체는 기본적으로 동일한 메서드를 가지고 있어, 기본적인 동작 원리를 이해하면 큰 문제가 없다.
  • map의 값을 반복적으로 접근하기 위해 iterator 또는 향상된 포문을 사용할 수 있다. entrySet 메서드를 통해 map엔트리를 set형태로 얻은 후, iterator로 루핑하여 키와 값을 가져온다.

5.7. ️ 해시 map 구현 및 기능 설명

  • 해시 map은 해시코드와 equals를 기반으로 동일한 키를 판단하며, 중복 값이 없는 것이 특징이다.
  • 키 타입으로는 주로 스트링을 사용하고, 문자열은 해시코드와 equals 메소드가 이미 재정의되어 있다.
  • 해시 map을 생성할 때는 키타입과 값 타입을 지정해야 하며, 기본 타입이 아닌 래퍼 타입을 사용해야 한다.
  • 객체에서 두 필드, 즉 sno와 name이 동일할 경우 동일 객체로 판단하기 위해 equals와 해시코드를 오버라이딩 한다.
  • 해시 map은 동일 키일 경우 하나의 엔트리만 존재하며, 이는 객체의 해시코드와 equals의 동작 방식 때문임을 보인다.

5.8. 컬렉션 프레임워크 개요

  • 컬렉션 프레임워크는 널리 알려진 자료구조를 통해 객체를 효율적으로 추가, 삭제, 검색할 수 있도록 설계되어 있다.
  • 자바에서는 유틸 패키지를 통해 컬렉션이 제공되며, 대표적으로 list, set, map이라는 세 가지 인터페이스가 있다.
  • 이들 인터페이스는 다양한 구현 클래스가 있으며, 기본적으로 정의되어 있는 추상 메소드를 이해하면 그 동작 원리를 파악할 수 있다.
  • 기초적인 내용을 반복 학습하는 것이 중요하며, 이를 통해 스프링 등 앞으로 배울 것들을 자연스럽게 연결할 수 있다.
  • 강의를 통해 배운 내용을 스스로 정리하고 반복 실습하는 노력이 필요하다는 점을 강조하고 있다.

Key term

컬렉션 프레임워크

자바에서 객체를 효율적으로 추가, 삭제 및 검색하기 위한 구조를 제공하는 시스템입니다. 쉽게 말해, 여러 종류의 상자(list, set, map 등)를 만들어서 거기에 데이터를 넣고 관리하는 방법이라고 생각하면 됩니다. 각 상자는 고유한 특징을 가지고 있어, 필요에 따라 다른 상자를 선택할 수 있습니다.

list

list는 객체를 관리하는 순서가 있는 컬렉션입니다. 이것은 배열과 비슷하지만, 크기가 자동으로 증가하고, 중복 저장도 가능합니다. 예를 들어, 학교에서 학생들의 성적을 순서대로 저장할 때 list를 사용하면 편리합니다.

set

set은 중복된 객체를 저장하지 않는 컬렉션입니다. 수학의 집합과 비슷하여, 여러 가지의 요소를 포함하되, 동일한 요소는 단 하나만 저장할 수 있습니다. 즉, 친구 목록에서 같은 친구는 한 번만 나오는 것과 같습니다.

map

map은 키-값 쌍으로 데이터를 저장하는 컬렉션입니다. 이는 마치 전화번호부에서 이름을 키로 하고 전화번호를 값으로 하는 식입니다. 각 키는 고유해야 하므로, 같은 이름의 친구가 두 명이라면, 구별할 수 없겠죠.

array list

array list는 동적 배열의 한 종류로, 객체를 저장하기 위한 list입니다. 사용자가 원하는 만큼 요소를 추가하고 삭제할 수 있으며, 인덱스를 사용하여 특정 요소에 접근할 수 있어요. 마치 여러 컵을 배열해 두고 필요한 컵을 빠르게 찾는 것과 같습니다.

linked list

linked list는 데이터를 요소별로 연결해 저장하는 list입니다. 중간에 요소를 추가하거나 삭제할 때 유리하게 작동합니다. 즉, 각각의 컵이 서로 연결되어 있어 하나를 빼도 다른 컵들이 그대로 남아있는 형태로 생각할 수 있습니다.

해쉬set

해쉬set은 요소를 중복 없이 저장하는 set의 한 종류로, 내부적으로 해시코드라는 방법으로 요소를 관리합니다. 해시set은 요소의 순서를 유지하지 않아서, 친구의 목록에서 중복 친구가 없도록 관리하는 느낌으로 생각하면 됩니다.

해시 map

해시 map은 키와 값으로 데이터를 저장하는 map의 하나로, 데이터의 접근속도가 매우 빠릅니다. 비유하자면, 친구의 이름(키)을 입력하면 그 친구의 번호(값)를 바로 찾을 수 있는 주소록과 같습니다.

메소드

메소드는 객체가 실행할 수 있는 기능으로, 예를 들어 '추가', '삭제', '검색' 등이 있습니다. 물건을 관리할 때, 어떤 행동을 마냥 비어있는 공간에서만 할 수 있는 것처럼, 메소드는 특정 객체에서만 사용됩니다.

공용 인덱스

인덱스는 list나 배열에서 각 요소의 위치를 지정하는 번호입니다. 즉, 0번 인덱스는 첫 번째 요소를 의미하며, 이것 덕분에 필요한 정보를 쉽게 찾을 수 있습니다.

중복 저장

중복 저장은 같은 내용을 여러 번 저장하는 것으로, list에서는 허용되지만, set에서는 불가능합니다. 친구 목록처럼 같은 친구의 이름을 여러 번 쓸 때 중복 저장이 필요할 수 있습니다.

객체 저장

객체 저장은 메모리에 데이터를 담는 행위입니다. 물건을 박스에 넣는 것처럼, 코드상에서 데이터를 다룰 수 있도록 돕습니다.

동기화

프로그래밍의 동기화는 여러 스레드가 데이터를 공유할 때 충돌을 막기 위해 안전하게 처리하는 것을 의미합니다. 마치 과제를 팀워크로 진행할 때, 서로 충돌하지 않게 조율하는 것과 비슷합니다.

자료구조

자료구조는 데이터를 저장하고조직하는 방법입니다. 예를 들어, 책장에 책을 양쪽에서 가로로 정렬해서 쉽게 찾을 수 있게 하는 방법과 같아요.

메모리 관리

메모리 관리는 데이터를 효율적으로 저장하고 회수하는 기술로, 사용하지 않는 데이터를 삭제하여 공간을 확보합니다. 쓰고 있는 책을 정리해서 빈 공간을 만드는 것처럼, 컴퓨터에서도 필요하지 않은 정보는 지워야 합니다.

추상 메소드

추상 메소드는 구체적인 구현 없이 정의된 메소드로, 이를 상속받은 클래스에서 특정 방식으로 구현해야 합니다. 마치 '운전하다'라는 동작을 글로 쓰지 않고, 각각의 사람에게 어떻게 운전해야 하는지 설명하는 것과 같습니다.

프레임워크

프레임워크는 소프트웨어의 구조 또는 뼈대로, 개발자가 필요한 기능을 더해 사용할 수 있는 기반입니다. 건물의 뼈대와 같아, 그 위에 벽이나 지붕을 세울 수 있는 것처럼요.

자바 유틸 패키지

자바 유틸 패키지는 자바에서 제공하는 다양한 기능들을 담은 도구 상자입니다. 프로그램 작성 시 도움이 되는 여러 메소드를 쉽게 사용할 수 있게 해주죠.

사이즈 메소드

사이즈 메소드는 컬렉션 안에 저장된 요소의 개수를 반환합니다. 즉, 여러 컵 중 지금 몇 개가 채워져 있는지를 확인하는 것과 같습니다.

컨테이너 메소드

컨테이너 메소드는 객체를 추가, 삭제 및 검색하기 위해 사용되는 메소드입니다. 이는 마치 물건을 넣고 꺼낼 수 있는 박스의 움직임을 제어하는 것과 같습니다.

iterator

iterator는 컬렉션의 요소를 하나씩 순회할 수 있게 해주는 도구입니다. 여러 컵이 나열되어 있을 때 한 컵씩 보고 싶다면, iterator를 사용해 접근하는 것과 같습니다.

타입 파라미터

타입 파라미터는 제너릭을 사용할 때 어떤 객체를 저장할지 정의하는 부분입니다. 나무 상자 안에 사과, 배 등이 들어간다면, 타입 파라미터는 그 상자가 무엇을 담을지 지정하는 역할을 합니다.

객체 참조

객체 참조는 변수에 객체의 주소를 저장하는 것으로, 해당 객체에 접근할 수 있게 해줍니다. 나무 상자의 위치를 기록해 두고 나중에 그 상자를 쉽게 찾는 것과 비슷합니다.

객체 추가

객체 추가는 컬렉션에 새로운 요소를 넣는 행위입니다. 쇼핑카트에 물건을 담는 것처럼, 원하는 데이터를 추가하는 작업입니다.

삭제

삭제는 컬렉션에서 특정 객체를 제거하는 작업입니다. 필요한 물건을 버리거나 불필요한 데이터를 지우는 것과 같습니다.

검색

검색은 컬렉션에서 특정 요소를 찾는 행위입니다. 마치 집안에서 원하는 물건을 찾아다니는 것처럼 필요한 정보를 찾는 것입니다.

배열의 단점

배열은 고정된 크기를 가지고 있어, 크기를 변경할 수 없다는 단점이 있습니다. 예를 들어, 친구가 갑자기 2명 더 생겨도, 배열의 크기를 바꾸지 못하면 기존 배열에 추가할 수 없습니다.

동적 크기

동적 크기는 컬렉션이 사용 중에 크기를 변경할 수 있음을 의미합니다. 사용자가 필요에 따라 추가하거나 삭제할 수 있어 유연하게 사용이 가능하죠.

학습용 예제

학습용 예제는 프로그래밍 개념을 이해하기 쉽게 하기 위해 만든 연습 코드입니다. 새로운 기술을 배울 때 적용해 볼 수 있는 좋은 방법이죠.

프로그래밍 기초

프로그래밍 기초는 기초적인 개념과 원리로, 모든 프로그래밍의 기본이 되는 내용들입니다. 이는 나무를 심기 전의 땅고르기와 같아 기본이 탄탄해야 이후의 작업도 잘 진행됩니다.
