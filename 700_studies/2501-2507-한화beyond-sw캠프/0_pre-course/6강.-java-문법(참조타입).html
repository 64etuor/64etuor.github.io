6강. JAVA 문법(참조타입)

이 강좌에서는 자바의 참조 타입에 대해 깊이 있게 설명합니다. 참조 타입은 기본 타입과는 달리 객체의 주소값을 저장하며, 메모리 관리 및 효율성에 있어 중요합니다. 특히, 힙 영역과 스택 영역의 개념을 명확히 하여 객체의 생성과 메모리의 관리 방식을 이해하는 데 도움을 줍니다. 또한, 참조 타입의 변수 간 비교 및 null 포인터 예외에 대한 처리를 통해 실제 프로그래밍에서 마주칠 수 있는 문제를 다루고 있습니다. 이 강좌는 자바 프로그래밍의 기초를 다지기에 적합한 내용으로 구성되어 있습니다.

1. 📚 참조 타입 개요

  • 참조 타입은 객체의 주소값을 저장하는 방식으로, 기본 타입과 구분된다. 기본 타입은 직접 값으로 저장되지만, 참조 타입은 객체의 크기가 일정치 않아 주소를 통해 저장된다.
  • 메모리 사용 구조에서 스택 영역은 기본 타입 변수를 저장하고, 힙 영역은 참조 타입 객체를 저장한다. 이 구조는 메모리 효율성을 높이며, 주소값을 통해 객체에 접근할 수 있게 한다.
  • JVM의 메모리 영역은 메소드 영역, 힙 영역, 스택 영역으로 나뉘며, 각 영역은 클래스 로드 시 정적 필드, 동적으로 생성된 객체, 메소드 호출 시의 데이터를 관리한다.

2. 📚 메소드 호출과 프레임 관리

  • 메소드가 호출되면 프레임이 추가되고, 메소드가 종료되면 해당 프레임이 제거된다.
  • JVM 스택 영역에서는 메소드가 호출될 때마다 프레임이 추가되며, 종료 시 해당 프레임이 제거되는 방식으로 동작한다.
  • 프레임 내부의 변수는 스택 영역에 저장되며, 기본 타입 변수의 값이 그대로 저장된다.
  • 코드 블록 실행 시 변수들이 쌓이고, 코드 블록을 빠져나갈 때 해당 변수들은 사라진다.
  • 참조 타입의 변수는 스택 영역에 힙 영역에 생성된 객체의 주소를 가진다. 여기서 배열도 객체의 일종으로 간주된다.

3. 📚 참조 변수와 객체 비교

  • 객체는 힙 영역에 저장되며, 참조 변수는 해당 객체의 주소값을 담고 있다.
  • 참조 변수는 메모리 주소값을 비교하여 객체가 동일한지를 판단하고, 이 때 주소가 같으면 true, 다르면 false를 반환한다.
  • 여러 참조 변수가 동일한 객체의 주소를 공유하면, 이를 통해 참조 타입 변수 간의 비교 연산 시 동일 객체인지 여부를 확인할 수 있다.
  • 참조 타입의 변수는 null(null) 값을 가질 수 있으며, 이는 해당 변수가 객체를 참조하지 않고 있음을 의미한다.
  • Null Pointer Exception은 참조 타입 변수가 null일 때 메소드를 호출하거나 필드를 참조할 경우 발생하는 오류이며, 이를 방지하기 위해 참조 변수가 객체를 참조하도록 수정해야 한다.

4. 🧩 문자열 리터럴과 참조 타입의 이해

  • string 타입은 기본 타입이 아니라 참조 타입으로, 이는 문자열의 길이가 가변적이기 때문에 힙 영역에서 관리된다.
  • 문자열 리터럴은 한 번 생성되면 공유되며, 여러 변수에 할당될 경우 동일한 힙 메모리 주소를 참조하게 된다.
  • 'new' 연산자를 사용할 경우 새로운 문자열 객체가 생성되어 서로 다른 주소값을 가지게 되므로, 동일한 내용의 문자열일지라도 비교 결과는 다르다.
  • equals() 메소드는 객체의 내용을 비교하여 문자열 값이 같은지를 확인하는 데 사용되며, 내용이 같으면 true, 다르면 false를 반환한다.
  • 기본 타입인 정수, 실수, 불리언은 null 값을 가질 수 없지만, string 변수는 null 값을 대입하여 객체가 없음을 나타낼 수 있다.

4.1. string 타입의 특성

  • string 타입은 기본 타입이 아닌 참조 타입으로 분류된다.
  • 텍스트의 길이를 알기 어려워 가변적인 특성을 가진다.
  • 문자열은 힙 영역에서 관리되며 객체로 취급된다.
  • 문자 단위의 크기는 예측 가능하지만 문자의 개수는 변동이 있을 수 있다.
  • 변수는 힙 영역에 대한 주소값을 저장하고, 문자열 리터럴은 힙 영역에 저장된다.
  • 동일한 문자열 리터럴의 경우, string 객체를 공유하는 구조를 가진다.

4.2. 문자열(string)의 특성 및 JVM 동작 방식

  • string은 변형되지 않는 객체로, 한 번 생성되면 읽기 전용으로 사용해야 한다.
  • 문자열이 공유되는 이유는 메모리에서 큰 문제가 발생하지 않기 때문이다.
  • JVM에서는 문자열 리터럴을 특정 메모리 영역에 생성하고, 이를 여러 곳에서 공유하여 사용한다.
  • 예를 들어, 동일한 문자열인 "홍길동"을 할당하면 같은 힙 메모리 주소값을 가지게 된다.
  • 두 개의 문자열이 동일한 주소값을 가진다면, 비교 시 결과가 같다고 나온다.

4.3. 'new' 연산자의 객체 생성 과정

  • 'new'라는 연산자는 객체를 생성하는 연산자로, 힙 영역에 새로운 객체를 생성하게 된다.
  • 동일한 리터럴 값을 가지더라도 'new'를 통해 생성된 객체들은 서로 다른 주소값을 갖기 때문에, 값이 같더라도 서로 다르다고 판단된다.
  • 예를 들어, '홍길동'이라는 리터럴 값을 가진 두 개의 string 객체가 생성되면, 이들은 동일한 주소값을 공유하여 'true' 값을 반환하지만, 'new' 연산자로 생성된 객체들은 서로 다른 주소값을 가져 'false'를 반환한다.
  • 'new' 연산자는 클래스를 생성하는 데 필요한 정보를 소괄호 안에 넣어 새로운 객체를 생성하며, 이 객체는 힙 메모리 영역에 할당된다.
  • 따라서 동일한 문자열 리터럴 값을 가지더라도 'new'를 사용하여 생성된 객체들은 서로 다른 메모리 공간을 갖는다.

4.4. String 타입의 비교와 equals 메소드

  • equals 메소드는 서로 다른 메모리 위치에 있는 문자열의 의미가 동일한지 비교할 때 사용된다.
  • 이 메소드는 입력받은 문자열과의 비교 결과를 true 또는 false 값으로 반환하고, 값이 같으면 true, 다르면 false를 반환한다.
  • 일반적인 비교 연산자(==)는 메모리 주소를 비교하며 동일한 객체임을 확인하는 데 사용된다.
  • equals 메소드는 문자열 리터럴 값 비교에 사용되며, string 클래스에 한정되어 적용된다.
  • 메모리 영역을 새롭게 갖는 new 연산자를 사용할 경우, 두 객체는 서로 다른 메모리 위치를 갖게 되므로 참조가 다르다는 결과가 출력된다.

4.5. 참조 타입의 null 값 사용

  • string 변수에 초기 값을 null로 대입할 수 있다는 점이 언급되었다.
  • 기본 타입인 정수, 실수, 불리언 타입은 null 값을 가질 수 없다고 말한다.
  • null 값은 string 변수가 참조하는 객체가 없음을 의미한다.

5. 🗑️ 가비지 컬렉터와 메모리 관리

  • 가비지 컬렉터는 JVM 내에서 참조를 잃은 객체를 자동으로 회수하여 메모리를 관리하는 역할을 한다.
  • 메모리는 별도의 관리 없이 가비지 컬렉터에 의해 자동으로 제거되므로 프로그래머의 부담이 줄어든다.
  • 참조 변수가 null 값을 갖게 되면 해당 객체는 더 이상 참조되지 않으며, 이 경우 가비지 컬렉터가 메모리를 회수하게 된다.
  • 이번 강의에서는 참조 타입과 그에 따른 메모리 관리 방법에 대해 설명하였다.

Key term

자바

자바는 프로그래밍 언어 중 하나로, 객체 지향 언어입니다. 즉, 개발자가 소프트웨어를 만들 때 '객체'라는 개념을 사용하여 코드 구조를 설계합니다. 자바는 강한 타입 체크를 통해 안전성을 높이고, 다양한 플랫폼에서 코드를 실행할 수 있도록 해줍니다.

참조 타입

참조 타입은 객체의 주소값을 저장하는 변수를 의미합니다. 이는 기본 타입과 달리 실제 데이터를 직접 가지고 있지 않고, 데이터의 위치를 가리킵니다. 즉, 참조 타입 변수는 특정 객체를 찾는 '지도' 역할을 합니다.

기본 타입

기본 타입은 자바에서 단순한 데이터 유형을 말합니다. 예를 들어, 정수(int), 실수(double), 문자(char), 논리값(boolean) 등이 이에 해당합니다. 이들은 메모리에서 직접 값을 가지고 있습니다.

메모리 관리

메모리 관리는 컴퓨터 프로그램의 자료를 저장하는 방법을 지정합니다. 자바는 메모리를 자동으로 관리할 수 있는 기능을 제공하여, 사용자가 복잡하게 메모리를 다루지 않아도 되도록 돕습니다.

힙 영역

힙 영역은 프로그램이 실행될 때 동적으로 생성되는 객체가 저장되는 공간입니다. 자바에서 객체를 생성하면, 그 객체는 힙 영역에 저장됩니다. 많은 객체가 생성되면 이 공간이 커질 수 있습니다.

스택 영역

스택 영역은 프로그램의 실행 중 일시적인 데이터를 저장하는 공간입니다. 메서드가 호출되면 그 메서드에 필요한 정보가 저장되고, 메서드가 끝나면 이 정보는 지워집니다. 마치 책상을 정리하는 것과 같다고 볼 수 있습니다.

객체

객체는 데이터와 기능을 함께 묶은 정보 단위입니다. 예를 들어, '자동차' 객체는 차의 색상, 모델 등 정보와 엔진을 START하는 기능을 포함할 수 있습니다.

주소값

주소값은 메모리에서 데이터의 위치를 나타내는 숫자입니다. 예를 들어, 집 번호처럼 특정 데이터가 어디에 있는지를 알려줍니다.

null 포인터 예외

null 포인터 예외는 코드에서 사용하려고 하는 객체가 존재하지 않을 때 발생하는 오류입니다. 예를 들어, '없다'는 것을 인식하지 못하고 그 객체에 접근하려고 하면 에러가 발생합니다.

가비지 컬렉터

가비지 컬렉터는 프로그램에서 더 이상 사용하지 않는 메모리를 자동으로 정리하는 기능입니다. 이로 인해 메모리 누수 없이 깔끔하게 메모리를 사용할 수 있게 돕습니다.

프리미티브 타입

프리미티브 타입은 자바의 기본 데이터 유형으로, int, double, char 같은 단순한 데이터 타입을 포함합니다. 이들은 실제 값을 직접 저장합니다.

string

string은 문자열을 표현하는 데이터 타입으로, 문자들의 집합입니다. 예를 들어, '안녕하세요'는 하나의 string으로 연속된 문자를 담고 있습니다.

배열

배열은 여러 개의 같은 타입 데이터를 하나의 변수에 저장할 수 있게 해주는 데이터 구조입니다. 예를 들어, 5개의 정수를 연속적으로 저장할 수 있습니다.

클래스

클래스는 객체를 만들기 위한 청사진입니다. 객체의 속성과 동작을 정의하며, 이를 바탕으로 여러 객체를 생성할 수 있습니다.

메서드

메서드는 객체가 수행할 수 있는 기능으로, 특정 작업을 수행하는 코드 블록입니다. 예를 들어, 자동차 객체의 '주행하다'라는 메서드는 자동차가 움직이도록 합니다.

변수

변수는 데이터를 저장하는 공간으로, 이름을 가지고 있습니다. 이 공간에 값이 저장되면, 훗날 해당 이름으로 그 값을 쉽게 참조할 수 있습니다.

비교 연산자

비교 연산자는 두 값을 비교하여 결과를 반환하는 연산자입니다. 예를 들어, '=='는 두 값이 같은지를 비교합니다.

equals 메서드

equals 메서드는 두 객체의 내용을 비교하는 기능을 가지고 있습니다. 이 메서드를 사용하면 두 객체가 같은 데이터를 가지고 있는지 판단할 수 있습니다.

동일 객체

동일 객체는 메모리에 저장된 같은 주소를 가진 객체를 의미하며, 같은 객체는 항상 같은 데이터를 가지고 있습니다.

프레임

프레임은 호출된 메서드에 대한 정보를 저장하는 공간으로, 메서드가 실행될 때마다 새 프레임이 만들어지고 메서드가 끝나면 제거됩니다.

메모리 주소

메모리 주소는 컴퓨터의 메모리에서 특정 위치를 나타내는 값입니다. 주소값을 알고 있으면 그 위치에 접근할 수 있습니다.

JVM

JVM(Java Virtual Machine)은 자바 프로그램을 실행하는 가상의 컴퓨터입니다. 자바로 작성된 코드를 이해하고 실행시켜줍니다.

string 리터럴

string 리터럴은 직접 입력된 문자열 값을 의미합니다. 예를 들어, 'hello'라고 적으면 이 자체가 하나의 string 리터럴입니다.
