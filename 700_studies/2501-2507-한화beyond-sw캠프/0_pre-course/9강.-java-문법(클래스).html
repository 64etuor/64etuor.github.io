9강. JAVA문 법(클래스) |

이번 강의는 클래스와 객체지향 프로그래밍의 핵심 개념을 배우는 것입니다. 자바에서 클래스를 사용하여 객체를 만들고, 객체의 속성( 필드)과 동작( 메소드)를 정의하는 방법을 설명합니다. 클래스는 객체의 설계도로서 역할을 하며, 객체는 메모리에서 식별 가능한 인스턴스입니다. 이 과정을 통해 객체지향 프로그래밍의 기본 원칙을 이해할 수 있습니다. 실습을 통해 직접 클래스를 생성하고 속성과 메소드를 구현하는 경험을 얻을 수 있습니다.

1. 🧑‍💻 클래스와 객체 지향 프로그래밍의 구조

  • 객체는 식별 가능한 속성을 가진 실체이며, 객체지향 프로그래밍에서는 이를 필드와 메소드로 구현한다.
  • 클래스는 객체를 만들기 위한 설계도로, 클래스에서 new 연산자를 사용하여 객체를 생성한다.
  • 객체는 메모리에서 고유하게 식별되는 것으로, 참조 변수를 통해 객체의 메모리주소를 참조할 수 있다.
  • 클래스는 필드, 생성자, 메소드로 구성되어 있으며, 필드는 객체의 데이터를 저장하는 변수의 역할을 한다.
  • 클래스내부와 외부에서 필드에 접근하여 데이터를 읽거나 변경할 수 있으며, 초기값을 주지 않으면 기본값이 설정된다.

1.1. ️ 객체지향 프로그래밍과 클래스의 개념

  • 클래스는 객체지향 프로그래밍에서 객체를 모델링하기 위한 기본 단위로, 속성과 동작을 가지게 된다.
  • 객체는 물리적으로 존재하거나 추상적으로 생각할 수 있는 식별 가능한 존재로, 속성과 동작으로 구성된다.
  • 객체의 속성은 필드로 표현되며, 동작은 메소드를 통해 구현된다.
  • 자바를 통해 인간이나 자동차와 같은 객체를 모델링할 수 있으며, 이들 각각의 속성과 행동이 프로그래밍적으로 표현된다.
  • 클래스는 객체를 정의하는 템플릿으로, 파일을 생성할 때 클래스형태로 구성된다.

1.2. ️ 클래스와 객체의 관계

  • 클래스를 이제는 객체를 만들기 위한 설계도로 사용하게 된다.
  • 인스턴스란 클래스로부터 만들어진 객체를 의미하며, 객체와 같은 의미이다.
  • 객체는 컴퓨터 프로그래밍에서 메모리상에서 존재하며, 개별적으로 구별 가능해야 한다.
  • 객체지향 프로그래밍의 단계에서 클래스는 객체를 만들기 위한 설계도를 제공한다.
  • 'new'라는 연산자를 통해 클래스로부터 객체를 생성한 후, 그 객체를 변수에 할당하여 활용할 수 있다.

1.3. 클래스명 작성 규칙

  • 클래스명은 식별자 작성 규칙에 따라 작성되어야 하며, 이는 변수의 명명 규칙과 동일하다.
  • 클래스명 스타일은 Pascal Case로, 첫 글자를 대문자로 시작하고, 그 다음 단어의 첫 글자도 대문자로 작성한다.
  • 단어 사이를 대문자로 이어주는 형식으로 작성되며, 이어지는 단어는 소문자로 표현한다.
  • 클래스이름은 자바파일의 이름과 동일해야 하며, 이를 잊지 말아야 한다.
  • 클래스를 생성할 때는 필드와 메서드를 포함해야 하며, 객체는 new 키워드를 사용하여 생성한다.

1.4. 객체 생성 및 클래스 활용

  • 클래스명을 사용하여 'new' 연산자를 통해 메모리영역에 객체를 생성하고, 생성된 객체의 메모리주소가 return된다.
  • 생성된 객체는 참조변수에 담겨 해당 주소값을 통해 접근할 수 있다.
  • 객체생성 후, 참조변수는 특정 클래스의 레퍼런스 주소를 참조하여 사용된다.
  • 클래스는 실행을 위한 용도와 라이브러리 클래스로서의 용도를 가지며, 예제로 학생 객체를 생성하는 과정이 설명된다.
  • 실행 클래스메인 메소드를 통해 외부에서 객체를 생성하고 가시적으로 출력하는 방법을 시연한다.

1.5. ️ 클래스 멤버의 구성 요소

  • 클래스멤버는 필드, 생성자, 메소드의 세 가지로 구성된다.
  • 필드는 객체의 데이터가 저장되는 장소로, 변수처럼 동작한다.
  • 생성자는 객체가 생성되는 시점, 즉 new 연산자를 만났을 때 실행되는 부분이다.
  • 메소드는 객체가 동작할 때 사용되며, 객체가 생성된 이후에 호출할 수 있다.

1.6. 필드의 개념과 활용

  • 필드는 객체에 고유 데이터 및 상태 정보를 저장하는 역할을 한다.
  • 자동차 객체의 예에서 속성들은 클래스에 정의되며, 이를 통해 객체가 만들어진다.
  • 필드는 변수 선언과 유사하며, 클래스내부에서 초기 값을 주거나 생략할 수 있다.
  • 객체에 접근할 수 있는 방법으로 클래스외부에서도 필드에 접근이 가능하다.
  • 예시로 Car 클래스를 만들어 속성을 정의하고, 이를 통해 객체생성을 실습할 수 있다.

2. 🏗️ 클래스와 생성자 이해하기

  • 더블 타입의 필드, INT 배열, 그리고 스트링 타입의 레퍼런스 필드를 생성하여 다양한 타입의 필드를 만들어 보았다.
  • 클래스의 생성자는 new 연산자로 호출되며, 객체초기화와 필드값 설정을 담당한다.
  • 디폴트 컨스트럭터는 생성자선언이 생략되었을 때 자동으로 추가되며, 반환 타입 없이 클래스이름과 소괄호로 구성된다.
  • 생성자는 여러 개를 오버로딩할 수 있으며, 매개변수를 통한 필드초기화가 가능하다.
  • 객체생성 시, 필드에 초기화 할 값을 파라미터로 전달해야 하며, 이는 this 키워드를 통해 필드와 파라미터의 구분이 가능하다.

2.1. ️ 다양한 필드 타입 만들기 및 출력

  • 더블 타입 필드를 생성하고, INT 배열과 스트링 타입 레퍼런스 필드도 추가로 만들어 보았다.
  • 필드이름은 밸류로 설정하고, 실행 클래스에서 객체를 생성하여 값들을 출력하기로 하였다.
  • 다양한 타입의 필드를 하나씩 출력하며, 바이트, 숏, INT, 그리고 불타입을 계속해서 조회해 보았다.
  • 필드값을 출력한 결과, 숫자 타입은 모두 0으로 초기화되어 있었고, 불타입은 퍼즐 값을 가지는 모습을 확인하였다.

2.2. ️ 클래스 생성자 이해하기

  • 클래스 생성자는 'new' 연산자로 호출되는 중괄호 블록이며, 객체초기화를 담당한다.
  • 객체초기화는 필드에 값을 넣거나 메소드를 호출하여 객체사용 준비를 하는 과정이다.
  • 생성자가 성공적으로 실행되면, 힙 영역에 객체가 생성되고 객체의 주소가 반환된다.
  • 디폴트 생성자는 생성자가 명시되지 않을 때 자동으로 추가되며, 기본적으로 퍼블릭으로 생성된다.
  • 생성자가 인수를 받지 않을 경우, 빈 생성자로부터 객체를 생성할 수 있다.

2.3. 클래스와 생성자 이해하기

  • 인텔리제이에서 클래스를 생성할 때 기본 생성자를 포함하여 필드를 정의하고 초기화하는 방법을 설명한다.
  • 생성자는 클래스내에서 클래스이름을 사용하여 선언하며, 매개변수를 통해 필드를 초기화하는 코드가 필요하다.
  • 생성자가 여러 개인 경우, 이를 오버로딩이라고 하며 매개변수의 타입이나 개수에 따라 메소드를 구별할 수 있다.
  • 기본 생성자가 없으면 객체를 생성할 수 없는 에러가 발생하며, 클래스는 최소 하나의 생성자를 가져야 한다.
  • 필드초기화 시 매개변수이름과 동일한 경우가 많고, 이를 구별하기 위해 this 키워드를 사용하여 객체의 필드를 지정할 수 있다.

2.4. ️ 생성자 오버로딩 예제 설명

  • 생성자 오버로딩을 이해하기 위해 여러 개의 생성자를 만드는 예제를 살펴본다.
  • 자바 클래스'칼'을 만들고 필드를 음악, 회사, 모델, 컬러, 맥스 스피드로 선언한다.
  • 기본 생성자를 생성하고, 외부에서 받아온 모델과 컬러 값으로 초기화하는 생성자를 추가로 만든다.
  • 총 네 가지 생성자를 만들어 기본 생성자와 모델, 컬러를 받는 생성자를 구현하고, 회사 값을 제외한 나머지 세 가지 필드를 외부로부터 값으로 초기화한다.
  • 클래스생성 이후 추가적인 클래스를 만들 필요가 있다.

2.5. 다양한 생성자 활용 및 객체 생성

  • 카 객체를 기본 생성자로 만들었고, 이후 S 아웃을 통해 출력 실습을 진행했다.
  • 두 번째 객체인 카투는 자가용 모델 값을 설정하는 생성자를 사용하여 모델 필드를 초기화하였다.
  • 세 번째 객체는 자가용 모델과 빨강 색상 값을 설정하고, 이에 대한 정보를 출력해볼 예정이다.
  • 네 번째 객체는 택시라는 이름으로 생성하고, 검정 색상 및 최대 속도 200으로 초기화하였다.
  • 이 과정에서 다양한 생성자의 활용 사례를 통해 객체생성의 이해를 도왔다.

3. ✨ 생성자 오버로딩과 메소드 구조

  • 다양한 생성자를 제공하기 위해 파라미터의 타입, 개수, 선언된 순서가 달라야 하며, 매개변수이름 변경만으로는 오버로딩할 수 없다.
  • 생성자에서 다른 생성자를 호출할 때 this 키워드를 사용하며, 이는 생성자의 첫 번째 위치에만 올 수 있다.
  • 메소드선언부는 return 타입, 메소드 이름, 매개변수를 포함하며, 메소드블록은 실행 코드가 위치한다.
  • 반환 값이 없는 메소드는 return 타입이 void로 정의되며, 최대 하나의 값을 반환할 수 있다.
  • 메소드를 호출할 때 매개변수타입이 다르면 컴파일 에러가 발생할 수 있으며, 자동 형변환이 가능하다.

3.1. 생성자 오버로딩과 디스 키워드

  • 다양한 생성자를 만들기 위해 파라미터를 다르게 정의하고, 이를 통해 생성자를 오버로딩할 수 있다.
  • 생성자 오버로딩은 매개변수의 타입, 개수, 선언된 순서를 기준으로 하며, 매개변수이름을 변경하는 것은 해당되지 않는다.
  • 두 개의 스트링 값만을 가진 경우는 오버로딩으로 간주되지 않는다.
  • 다른 생성자를 호출할 때는 디스라는 키워드를 사용하며, 소괄호를 통해 호출할 수 있다.
  • 디스 키워드는 생성자의 첫 번째 위치에만 사용될 수 있다.

3.2. 클래스 생성자와 중복 제거

  • 개에 대한 예제를 통해 새로운 클래스를 만들고, 다양한 필드와 메소드를 추가하는 방법을 설명하고 있다.
  • 생성자를 사용하여 모델 값을 입력받고, 추가적으로 컬러와 최대 속도를 설정하는 것도 포함된다.
  • 생성자내에서 this를 활용하여 다른 생성자를 호출하며, 중복을 줄이는 방식을 설명한다.
  • 실행 클래스를 생성하여 객체를 만들고, 다양한 값을 입력한 후 결과를 출력해본다.
  • 생성자호출 방식을 통해 중복된 코드 작성을 줄일 수 있음을 부각하고 있다.

3.3. ️ 클래스의 메소드 이해하기

  • 클래스의 메소드는 객체의 동작을 규정하는 부분으로, 공통적으로 갖는 함수이다.
  • 메소드선언부는 return 타입, 메소드 이름, 매개변수로 구성되며, 실행 블록도 포함된다.
  • 메소드는 최대 하나의 return 값을 가지고 있으며, 경우에 따라 return 값이 없을 수도 있다.
  • return 타입이 없는 경우, void로 명시하며 이 경우에는 반환할 값이 없으므로 변수가 필요 없어진다.
  • 메소드예시로 칼큘레이터 클래스내에서 파워온, 플러스, 디바이드, 파워 오프 메소드를 선언하고 설명한다.

3.4. 메인 클래스와 메소드 실행 과정

  • 메인 클래스를 생성하기 위해 카일 큐레이터라는 클래스를 만들었다.
  • 마이 칼크라는 변수명을 만들어서 뉴 카이큐레이터로 계산기를 생성하였다.
  • 파워 메소드를 호출하고, 플러스 메소드를 통해 5와 6의 값을 전달하여 결과를 출력하였다.
  • 바이트 값을 가지고 디바이드 메소드를 호출하여 10과 4 값을 전달하고, 자동 형변환이 일어난다.
  • 파워 오프 메소드를 호출하여 전원을 켜고, 출력 결과를 확인한 후 프로그램이 종료된다.

3.5. 메소드 명명 규칙과 매개변수 설명

  • 메소드의 명령 규칙은 변수명 짓는 것과 동일하다.
  • 따라서, 메소드의 이름을 지을 때는 변수명 짓듯이 하면 된다.
  • 메소드의 매개변수 선언은 소괄호 안쪽에서 이루어진다.
  • 소괄호 안에는 외부로부터 필요한 데이터를 입력받아 코드 블록에서 활용할 수 있는 값이 포함된다.

4. ⚙️ JAVA 메소드 매개변수와 return 처리

  • 잘못된 매개변수를 사용하면 컴파일 에러가 발생하고, 예를 들어 INT 대신 더블 타입의 값을 사용할 경우 에러가 난다.
  • 매개변수의 개수를 모르면, 매개변수를 배열 타입으로 선언하여 사용할 수 있으며, 예를 들어 INT 배열로 사용할 수 있다.
  • 메소드를 호출할 때는 배열을 만들어서 뉴 연산자를 사용해 직접 전달할 수 있다.
  • 메소드를 실행하는 과정에서 가변적인 매개변수를 받아올 수 있어 여러 개의 값을 배열 형태로 넘길 수 있다.
  • return 타입은 메소드선언에서 정해진 타입과 일치해야 하며, void return 타입의 메소드도 return문만으로 종료할 수 있다.

4.1. ️ 매개변수 처리 및 배열 활용

  • 잘못된 매개값 사용 시 컴파일 에러가 발생하여, 예를 들어 INT 타입의 매개변수에 더블 타입의 값을 전달하면 에러가 나게 된다.
  • 매개변수의 개수를 모를 경우 배열 타입으로 매개변수를 선언하여 사용할 수 있다.
  • 메소드인자는 배열 형태로 받을 수 있으며, 이를 호출할 때 new 연산자를 통해 직접 배열을 생성하여 전달할 수 있는 방법도 있다.
  • 예를 들어, INT 타입의 배열을 사용하는 메소드를 만들고, 포문을 통해 값을 더해주는 구조를 구현할 수 있다.
  • 가변적인 INT 타입의 매개변수를 받아서, 동일한 방식으로 합산 후 값을 반환하는 메소드를 만들 수 있다.

4.2. ️ 클래스와 메소드 사용 예시

  • 두 개의 메소드값을 갖는 클래스가 생성되었으며, 이를 사용하기 위해 실행 클래스를 만들 필요가 있다.
  • 메인 메소드에서 객체를 생성하고, 이를 변수에 담아 사용할 수 있도록 하였다.
  • 정수 배열을 생성하여 값을 1, 2, 3으로 설정하고, 이 배열을 메소드에 넘겨준다.
  • 여러 개의 값을 배열 형태로 메소드에 넘겨줄 수 있으며, 이를 통해 결과를 출력할 수 있다.
  • 가변 파라미터를 사용하여 필요한 만큼 값을 넘겨줄 수 있고, 그에 따른 결과 역시 출력될 수 있다.

4.3. 메소드의 매개변수와 return 관련 사항

  • 메소드의 매개변수는 베어를 생성하지 않고 값에 목록만 남겨주는 방식으로 선언된다.
  • 호출 시 여러 개의 값은 컴마로 구분하여 넘겨줄 수 있다.
  • 메소드의 return은 예약어 'return'을 통해 작성하며, 넘겨줄 값은 한 개만 가능하다.
  • return하는 값의 타입은 메소드선언부에서 작성한 return 타입과 일치해야 하며, 값이 없을 경우는 void를 사용하여 return할 수 있다.
  • return문은 메소드의 실행을 종료시키고, 메모리에서 반환되는 과정을 포함한다.

4.4. 가스 관리 클래스 구현

  • 인트 가스 필드를 만들어 가스 값을 저장한다.
  • 셋 가스 메소드는 INT 타입의 가스 값을 받아 해당 필드에 초기화하는 기능을 수행한다.
  • 이즈 레프트 가스 메소드는 가스 값이 0일 경우 "가스가 없습니다"라는 메시지를 출력하고 거짓을 반환한다.
  • 가스 값이 0이 아닐 경우 "가스가 있습니다"라는 메시지를 출력하고 참을 반환하며, 이 정보를 제공한다.
  • 런 메소드는 무한루프를 돌며, 가스가 0보다 클 때 "달립니다"라는 메시지와 가스 잔량을 출력하고 가스를 감소시키며, 가스가 0일 경우 "멈춥니다"라는 메시지와 가스 잔량을 출력한다.

5. 🔍 JAVA 클래스 메소드와 멤버

  • 메소드 호출은 객체내외부의 호출에 따라 실행되며, 내부 호출은 메소드이름만으로 가능하고 외부 호출은 객체참조 변수를 통해 이루어진다.
  • 메소드 오버로딩은 동일한 이름의 메소드를 매개변수의 타입, 개수, 순서가 다르게 정의하여 사용할 수 있게 하며, 이를 통해 다양한 매개변수를 처리할 수 있다.
  • 인스턴스 멤버는 객체가 생성된 후에만 사용할 수 있는 필드와 메소드로, 객체각각의 상태를 담는다.
  • 정적 멤버는 클래스에 고정된 멤버로, 객체생성 없이도 사용할 수 있으며, 클래스이름과 점 연산자로 접근이 가능하다.
  • 파이널 필드는 초기 값을 한 번만 설정할 수 있는 특성을 가지며, 이후 수정이 불가능하다. 이를 통해 불변의 데이터를 만들 수 있다.

5.1. 카 객체와 메소드 실행

  • return을 통해 작업을 끝내는 메커니즘을 설명하고 있다.
  • 카 객체를 담을 변수를 생성하고, 메인 메소드를 포함하는 카 이그제밍플을 작성한다.
  • '가스' 값을 설정하기 위해 5라는 값을 넣어주는 과정이 포함된다.
  • 가스 스테이터스를 확인하기 위한 변수를 만들고, 이에 따라 적절한 출력을 수행한다.
  • 가스 상태에 따라 가스를 주입할 필요가 있는지 확인하고, 그에 맞는 메시지를 출력하는 구조로 이루어져 있다.

5.2. 가스 주입 메소드 실행 과정

  • 실행 과정 중 가스가 있는지 확인하여 출력이 이루어진다.
  • 런 메소드가 실행되면, 가스가 0이 아닐 때 달리게 된다.
  • 가스가 0에 도달하면, 멈춘다는 메시지가 출력된다.
  • 가스가 없는 경우에는 가스를 주입해주세요라는 메시지가 추가 출력된다.

5.3. ️ 메소드 호출의 이해

  • 메소드 호출은 클래스내외부의 호출에 의해 실행되는 방식이다.
  • 내부 호출의 경우는 메소드이름을 단순히 사용하고, 외부 호출은 클래스로부터 객체를 생성후 참조 변수를 통해 메소드를 호출해야 한다.
  • return 값이 없는 메소드는 변수 없이 간단히 사용할 수 있다.
  • 예시로, 두 개의 인트 값을 받아 더하는 플러스 메소드와 평균을 계산하는 에버리지 메소드를 선언하여 return하는 방법을 설명한다.
  • 마지막으로 객체외부에서 메소드를 호출할 때, 참조 변수에서 객체를 생성하고 점(.)으로 메소드를 호출하는 방법을 사용한다.

5.4. ️ 메소드 오버로딩의 이해

  • 메소드 오버로딩은 같은 이름의 메소드를 여러 개 정의할 수 있는 기능이다. 이는 return 타입과 매개변수의 조합에 따라 유일하게 식별되기 때문이다.
  • 메소드 오버로딩을 사용할 때, 매개변수의 타입, 개수, 순서를 다르게 설정해야 한다.
  • 자바에서는 매개변수의 형 변환을 통해 다양한 타입의 값을 전달하고, 해당 매개변수에 맞는 메소드를 자동으로 호출할 수 있다.
  • 예를 들어, 두 개의 서로 다른 매개변수를 가진 메소드를 정의하여 메소드 오버로딩을 구현할 수 있다.
  • 메소드 오버로딩이 이루어질 때, 매개변수의 이름은 달라도 매개변수의 타입, 개수, 순서가 일치해야 메소드 오버로딩으로 인식되지 않으므로 주의해야 한다.

5.5. 자바 메소드와 클래스 멤버의 이해

  • 메소드를 정의할 때는 return 타입, 메소드 이름, 매개변수를 선언해야 한다.
  • void는 return 값이 없는 메소드의 return 타입을 나타내며 반드시 기술해야 한다.
  • 매개변수는 메소드호출 시 제공되는 값으로, 메소드블록 실행시 사용된다.
  • return문은 메소드의 return 값을 지정하거나 메소드실행을 종료하는 데 사용된다.
  • 클래스의 인스턴스 멤버는 객체생성 후 사용 가능하며, 정적 멤버는 클래스명에 도티 연산자를 사용하여 접근할 수 있다.

5.6. ️ 인스턴스 멤버와 정적 멤버

  • 인스턴스 멤버는 객체가 생성된 후 사용할 수 있는 필드와 메소드를 의미하며, 이를 통해 객체의 상태를 관리한다.
  • 정적 멤버는 static이라는 예약어를 통해 생성되며, 객체 없이도 사용 가능한 필드와 메소드를 제공한다.
  • 정적 필드와 메소드는 클래스의 고정된 멤버로, 클래스로더가 클래스를 로딩할 때 메모리의 메소드영역에 적재된다.
  • 객체마다 다를 수 있는 필드값은 인스턴스 필드로, 그렇지 않은 값은 정적 필드로 선언하는 것이 바람직하다.
  • 정적 메소드는 인스턴스 필드와 인스턴스 메소드를 사용할 수 없으며, 이는 메인 메소드에도 동일하게 적용된다.

5.7. ️ 싱글톤 패턴의 개요

  • 싱글톤 패턴은 전체 프로그램에서 단 하나의 객체만 생성되도록 보장하는 코딩 기법이다.
  • 싱글톤을 설계할 때는 클래스 외부에서 객체를 생성할 수 없도록 하기 위해 프라이빗 생성자를 사용해야 한다.
  • 자신에 대한 객체를 가지기 위해 정적 필드와 getInstance() 메소드를 선언하여, 이 메소드를 통해 객체를 가져올 수 있게 해야 한다.
  • 생성자를 프라이빗으로 설정하고 내부 클래스에서만 사용할 수 있도록 하여 외부에서 객체를 만들 수 없게 해야 한다.
  • 싱글톤 패턴을 통해 생성된 객체는 메모리에서 동일한 객체를 바라보는 특성이 있으며, 이를 통해 유일한 객체를 프로그램 내에서 사용할 수 있다.

5.8. 파이널 필드와 상수의 특성

  • 파이널 필드는 초기화된 후 프로그램 실행 중에 변경할 수 없는 값을 가지며, 이는 프로그래밍에 있어 중요한 개념이다.
  • 파이널 필드는 필드 선언 시 직접 초기화를 하거나, 생성자를 통해 외부 데이터로 초기화할 수 있다.
  • 파이널 변수는 초기화 후에는 값을 변경할 수 없으므로, 예를 들어 "p1"의 네이션이나 SSN 값을 변경하려 할 경우 에러가 발생한다.
  • 정적 파이널 필드는 관례적으로 상수로 사용되며, 이름은 모두 대문자로 작성하는 것이 일반적이다.
  • 파이널 필드는 인스턴스에 한번 초기화된 데이터 변경이 불가능하게 하며, 이로 인해 불변의 값 생성이 가능하다.

Key term

자바

자바는 객체지향 언어로, 프로그램을 작성할 때 주로 클래스와 객체를 사용하여 문제를 해결합니다. 쉽게 말해, 자바는 여러 가지 정보를 관리하는 데 사용하는 도구입니다.

클래스

클래스는 객체를 만들기 위한 설계도 같은 것입니다. 예를 들어, '자동차'라는 클래스를 만들면, 이 클래스는 자동차의 속성(예: 색상, 모델)과 동작(예: 주행하기, 정지하기)을 정의합니다.

객체지향 프로그래밍

객체지향 프로그래밍은 객체를 중심으로 프로그램을 구성하는 프로그래밍 방법론입니다. 현실 세계의 사물처럼 객체를 만들고 이를 통해 다양한 동작을 수행하는 방식입니다.

인스턴스

인스턴스란 클래스를 통해 생성된 실제 데이터입니다. 마치 '자동차'라는 클래스에서 '내 자동차'라는 특정 자동차를 만들 때, 이 특정 자동차가 바로 인스턴스입니다.

필드

필드는 클래스 안의 변수로, 객체의 속성을 나타냅니다. 예를 들어, '자동차' 클래스의 필드는 색상, 속도와 같은 정보를 저장하는 역할을 합니다.

메소드

메소드는 클래스 안에 정의된 함수로, 객체가 수행할 수 있는 동작을 정의합니다. 예를 들어, '자동차' 클래스의 메소드로는 '운전하기'와 '정지하기'가 있을 수 있습니다.

생성자

생성자는 객체가 생성될 때 호출되는 특별한 메소드입니다. 주로 객체의 초기값을 설정하는 역할을 하며, 클래스 이름과 동일한 이름을 가져야 합니다.

객체

객체는 클래스의 실제 구현체로, 메모리에서 생성되고 사용됩니다. 간단히 말해, 클래스에서 만들어진 데이터 단위를 의미합니다.

모델링

모델링은 현실 세계의 사물이나 개념을 이해하고 이를 바탕으로 클래스를 설계하는 과정입니다. 마치 설계도를 그려서 실제 건물을 짓는 것과 비슷합니다.

객체 생성

객체 생성은 클래스에 정의된 내용을 바탕으로 실제 객체를 메모리에서 만드는 과정입니다. 이를 위해 'new'라는 키워드를 사용합니다.

파라미터

파라미터는 메소드에 전달되는 인자값을 의미합니다. 예를 들어, '덧셈'이라는 메소드에서 두 숫자를 더하는 경우, 이 두 숫자가 파라미터입니다.

오버로딩

오버로딩은 같은 이름의 메소드가 여러 개 정의되는 것을 뜻합니다. 이때, 메소드를 구별하기 위해 파라미터의 개수나 타입이 달라져야 합니다.

레퍼런스 변수

레퍼런스 변수는 객체의 메모리 주소를 저장하는 변수입니다. 마치 주소로 집을 파악하는 것처럼, 객체에 대한 정보를 간접적으로 다룬다고 생각하면 됩니다.

디폴트 생성자

디폴트 생성자는 클래스 내에서 사용자 정의 생성자가 없을 때, 자바가 자동으로 만들어주는 생성자입니다. 이는 매개변수 없이 객체를 생성할 때 사용됩니다.

메모리

메모리는 프로그램이 실행되는 공간을 의미합니다. 클래스와 객체는 프로그램이 메모리에 로드될 때 생성됩니다.

식별자

식별자는 변수나 메소드, 클래스의 이름을 지칭합니다. 이름을 통해 코드를 이해하기 쉽게 해주며, 각 이름은 고유해야 합니다.

기본값

기본값은 변수가 초기화되지 않았을 때 자동으로 부여되는 값입니다. 예를 들어, 숫자형 변수는 0으로 초기화됩니다.

상태 데이터

상태 데이터는 객체의 현재 상태를 나타내는 데이터를 의미합니다. 예를 들어, 자동차의 현재 속도나 기름 양과 같은 정보를 포함할 수 있습니다.

자바 파일

자바 파일은 자바 코드를 저장하는 파일로, 일반적으로 .java 확장자를 가집니다. 이 파일는 사용자 정의 클래스를 포함할 수 있습니다.

명명 규칙

명명 규칙은 변수를 만들 때 따라야 할 규칙입니다. 예를 들어, 자바에서는 변수 이름은 소문자로 시작하고, 여러 단어를 연결할 때는 카멜 케이스를 사용합니다.

매개변수

매개변수는 메소드에 입력으로 주는 값으로, 메소드가 실행되는 동안 사용됩니다. 따라서 정해진 타입에 맞는 값이 필요합니다.
